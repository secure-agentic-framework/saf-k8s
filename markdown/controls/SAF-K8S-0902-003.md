# SAF-K8S-0902-003 - Inference request validation and input sanitization

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.2 - Inference Server and Model Serving Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Inference endpoints must validate and sanitize all incoming requests through adversarial input detection, payload size limits, request format enforcement, and injection attack prevention to protect inference servers from malicious inputs.

## Objective

Protect inference servers from malicious inputs that exploit model behavior or crash the server through adversarial examples, oversized payloads, malformed requests, or injection attacks.

## Rationale

Inference APIs accept untrusted input that can contain adversarial examples, oversized payloads, malformed requests, or injection attacks designed to manipulate model behavior or crash the server. Without input validation, these attacks succeed silently. Input validation at the API gateway and inference preprocessing layer provides defense in depth.

## Scope

Applies to all inference request processing paths on Kubernetes, including HTTP/gRPC inference APIs, batch prediction endpoints, and streaming inference channels. Covers input preprocessing pipelines. Excludes model-internal robustness (adversarial training) which is addressed in KA 9.3. Response filtering is covered by SAF-K8S-0902-009.

## Enforcement

- Primary Enforcement Point: Inference API gateway request validation policies and inference server input preprocessing configuration.
- Implementation Surface: API gateway and ingress controller request validation policies (payload size limits, schema enforcement), inference server input preprocessing configuration, adversarial input detection middleware, and prompt injection detection for LLM endpoints.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Application Security
