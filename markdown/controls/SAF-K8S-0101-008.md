# SAF-K8S-0101-008 - API server audit policy coverage and event detail

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must enable API server audit logging with an audit policy that captures at least Metadata-level events for all resource types, Request-level events for sensitive resources (including Secrets, ConfigMaps, RBAC bindings, and TokenReviews), and RequestResponse-level events for authentication and authorization decisions.

## Objective

Ensure that API server activity is captured at the right audit verbosity so investigators can reconstruct sensitive requests and decision points without blind spots.

## Rationale

Audit logging fails as a forensic control when the policy omits high-value resources or records them at insufficient detail. This control isolates the audit policy design obligation from backend storage so teams can separately verify event coverage and verbosity tuning.

## Scope

Applies to all production Kubernetes clusters, including managed services where the customer controls or can verify API server audit policy settings.

## Enforcement

- Primary Enforcement Point: Audit Policy Definition
- Implementation Surface: Audit policy YAML file, kube-apiserver audit policy configuration, and managed service audit policy settings.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Compliance
