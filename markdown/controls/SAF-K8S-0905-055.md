# SAF-K8S-0905-055 - Admission-time rejection of unapproved external model source references

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must deploy admission controllers that reject pods or workloads that reference external models from sources not present on the approved external model source allowlist.

## Objective

Prevent workload admission when model source references do not match the approved external model source allowlist.

## Rationale

Runtime network controls alone do not prevent a workload from being created with an unapproved model source reference. Admission-time validation blocks these workloads before they enter protected environments and provides explicit policy enforcement at creation time.

## Scope

Applies to production and protected Kubernetes namespaces that consume external models. Covers admission controller validation of model source URLs, image references, registry namespaces, or equivalent identifiers against the approved source allowlist. Excludes quarantine and vetting workflow obligations, which remain covered by SAF-K8S-0905-044 and SAF-K8S-0905-045.

## Enforcement

- Primary Enforcement Point: Admission controller policy evaluation at workload creation time.
- Implementation Surface: Admission controllers such as Kyverno or Gatekeeper rejecting unapproved external model sources, policy definitions validating source references against approved allowlists, and rejected workload events for attempted policy bypasses.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Governance
