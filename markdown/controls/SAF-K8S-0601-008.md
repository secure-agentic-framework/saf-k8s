# SAF-K8S-0601-008 - CI/CD build-time container image vulnerability scanning

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

All container images must undergo automated vulnerability scanning during the CI/CD build stage before they are pushed to any registry, and images with unmitigated critical or high-severity vulnerabilities must be blocked from promotion unless a documented risk acceptance is approved.

## Objective

Catch known vulnerabilities early in the image lifecycle by enforcing build-time scanning and promotion gates before vulnerable images enter shared registries.

## Rationale

Build-time scanning is the earliest scalable point to stop vulnerable images from propagating downstream. If critical findings are not blocked before push, the same vulnerable image can spread across registries, environments, and workloads. This is a distinct control claim from post-push and runtime exposure monitoring.

## Scope

Applies to all container images built for production and pre-production Kubernetes workloads, including AI/ML images with native CUDA, cuDNN, NCCL, and framework-specific dependency chains.

## Enforcement

- Primary Enforcement Point: CI/CD Build Stage
- Implementation Surface: CI pipeline scanning tools, build promotion gates, and pre-push risk acceptance workflow.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
