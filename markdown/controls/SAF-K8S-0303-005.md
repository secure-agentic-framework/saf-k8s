# SAF-K8S-0303-005 - MAC profile pre-enforcement audit-mode validation

- Domain: D03 - Workload and Pod Security
- Knowledge Area: 3.3 - Mandatory Access Controls
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must test all generated seccomp, AppArmor, or SELinux profiles in audit or complain mode (seccomp SCMP_ACT_LOG, AppArmor complain, SELinux permissive) before activating enforcement mode, and must confirm that no legitimate workload operations are denied during a defined observation period.

## Objective

Prevent production outages caused by over-restrictive MAC profiles by validating that the generated profile does not block any legitimate workload operation before enforcement mode is activated.

## Rationale

Activating a generated MAC profile directly in enforcement mode can cause immediate production outages if the profiling phase missed any legitimate operations. Audit mode (SCMP_ACT_LOG for seccomp, complain mode for AppArmor, permissive mode for SELinux) logs denied operations without blocking them, allowing operators to verify that no legitimate workload operations would be blocked before committing to enforcement.

## Scope

Applies to all generated MAC profiles prior to their first enforcement-mode deployment and after any significant profile change. The observation period must be long enough to capture infrequent legitimate operations such as periodic health checks, garbage collection syscalls, and scheduled backup operations.

## Enforcement

- Primary Enforcement Point: Seccomp SCMP_ACT_LOG mode, AppArmor complain mode, and SELinux permissive mode, monitored via audit logs and alerting before enforcement mode is activated.
- Implementation Surface: Seccomp SCMP_ACT_LOG action for non-blocking violation logging, AppArmor complain mode profile deployment, SELinux permissive domain configuration, audit log monitoring for denied-but-allowed operations, and sign-off workflow requiring zero legitimate denials over a defined observation window.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Application Development teams
