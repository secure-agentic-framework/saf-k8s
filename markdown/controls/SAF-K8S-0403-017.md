# SAF-K8S-0403-017 - AI platform key-domain hierarchy and envelope encryption architecture

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations operating AI platforms must implement a structured key hierarchy using envelope encryption with distinct key domains for training data encryption, model artifact encryption, model signing, inference payload protection, and other platform cryptographic functions, with HSM-backed or equivalently protected root keys and domain-specific rotation schedules.

## Objective

Ensure AI platform cryptographic design separates key domains by function so compromise of one domain does not automatically expose every other protected asset.

## Rationale

AI platforms protect multiple asset classes with different trust assumptions. A structured envelope-encryption hierarchy with domain-specific keys and protected root-of-trust material prevents a single compromised encryption key from collapsing the full cryptographic boundary for training, model, and inference systems.

## Scope

Applies to AI platform cryptographic architecture for training data, model artifacts, model signing, checkpoints, and inference payload protection. Covers key hierarchy design, root-key protection, and domain-specific rotation scheduling.

## Enforcement

- Primary Enforcement Point: KMS and HSM architecture design for AI platform key hierarchy and root-of-trust management.
- Implementation Surface: Cloud KMS key hierarchy, HSM-backed root keys, envelope encryption flows, model registry encryption configuration, training data encryption integration, model signing infrastructure, and per-domain rotation schedules.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; ML Engineering; Key Management/Cryptography
