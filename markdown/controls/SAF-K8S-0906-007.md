# SAF-K8S-0906-007 - Integrity verification at each training data transformation stage

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.6 - Training Data Integrity and Poisoning Defense
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must enforce integrity checks at each data transformation stage in the training data pipeline to detect tampering or corruption before altered data propagates downstream.

## Objective

Detect tampering or corruption at each transformation stage in training pipelines before poisoned or corrupted data reaches model training.

## Rationale

Without integrity checks at transformation stages, an attacker can inject poisoned or corrupted data at any point in the pipeline and the corruption propagates undetected through downstream transformations into the trained model.

## Scope

Applies to all training data pipelines on Kubernetes from ingestion through feature engineering to model training input. Covers stage-level integrity verification, hash or manifest comparison, and validation hooks at data transformation steps. Excludes end-to-end provenance recordkeeping.

## Enforcement

- Primary Enforcement Point: Integrity verification hooks at pipeline transformation stages.
- Implementation Surface: Integrity check logs at each transformation stage, pipeline configuration showing validation hooks, manifest or hash verification between pipeline stages, and alerting or rejection records for failed stage-integrity checks.
- Owner Primary: Data Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
