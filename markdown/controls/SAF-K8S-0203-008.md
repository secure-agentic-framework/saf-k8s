# SAF-K8S-0203-008 - Pod metadata endpoint network path blocking

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.3 - Host OS and Kernel Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Cloud-hosted Kubernetes clusters must implement network-level controls that prevent pod workloads from directly accessing cloud metadata endpoints, including host-network and local-routing paths where applicable.

## Objective

Block pod reachability to cloud metadata services so SSRF, host-network abuse, or compromised workloads cannot retrieve node-scoped cloud credentials through metadata endpoints.

## Rationale

Even when cloud metadata services support hardened modes, pod workloads may still attempt to reach metadata endpoints through routable local addresses. This control isolates the network-path blocking obligation so teams can independently verify that pod traffic cannot reach metadata services.

## Scope

Applies to cloud-hosted Kubernetes clusters where pods can otherwise route to metadata endpoints such as 169.254.169.254. Covers NetworkPolicy, host firewall, iptables, and CNI controls used to block metadata paths from pod networks.

## Enforcement

- Primary Enforcement Point: Node and cluster networking controls for pod credential access paths.
- Implementation Surface: NetworkPolicy or host-level filtering for metadata endpoints and validation tests confirming pods cannot reach metadata services.
- Owner Primary: Cloud Infrastructure
- Owner Supporting: Platform Engineering; Security Engineering
