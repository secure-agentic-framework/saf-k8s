# SAF-K8S-0606-011 - CI build-time security gate enforcement

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

CI pipelines must enforce mandatory build-time security gates including SAST, DAST where applicable, SCA scanning, secret scanning with push protection, and dependency review before artifact promotion. Pipelines must block artifacts that fail any mandatory gate unless an approved exception exists.

## Objective

Prevent vulnerable code, leaked credentials, and unsafe dependencies from progressing through CI by enforcing automated build-time security gates and promotion blocking.

## Rationale

Security defects and hardcoded secrets are cheapest and most reliably stopped before artifacts leave the build stage. Mandatory CI gates and enforced promotion blocking are a distinct obligation from freshness or repository posture checks, and they directly determine whether unsafe artifacts are allowed to move downstream.

## Scope

Applies to all CI pipelines producing deployable artifacts for Kubernetes clusters. DAST is required for externally facing services; SAST, SCA, secret scanning, and dependency review are required for all pipelines.

## Enforcement

- Primary Enforcement Point: CI Build Gate
- Implementation Surface: CI pipeline gate configuration, SCM push-protection settings, dependency review bots, and artifact promotion policies.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Development Teams
