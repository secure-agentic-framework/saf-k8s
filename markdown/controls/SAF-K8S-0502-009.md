# SAF-K8S-0502-009 - Kubernetes CNI IPAM capacity sizing

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.2 - CNI Plugins and Pod Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

IP address management for Kubernetes networking must be configured with adequate pod, service, and node CIDR capacity so IP pools support workload scheduling and service operation without avoidable exhaustion.

## Objective

Maintain sufficient IP address capacity for workload scheduling and service operation by correctly sizing pod, service, and node CIDR allocations.

## Rationale

Undersized pod, service, or node CIDR ranges cause scheduling failures and service disruption. Separate sizing governance from monitoring so capacity design can be independently reviewed and validated.

## Scope

Applies to all Kubernetes clusters. Covers pod CIDR, service CIDR, and node CIDR range sizing, utilization monitoring, exhaustion thresholds, and alerting for IP address pool depletion.

## Enforcement

- Primary Enforcement Point: CNI plugin IPAM capacity planning and allocation design.
- Implementation Surface: CNI plugin IPAM configuration for pod, service, and node CIDR allocation sizing, pool reservation strategy, and operational review of remaining address capacity.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Security Operations
