# SAF-K8S-0503-013 - Ingress web application firewall integration and request filtering

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.3 - Ingress, Egress, and DNS Hardening
- Maturity: Practitioner
- Classification: baseline

## Statement

Ingress controllers protecting internet-facing services must integrate with a web application firewall that applies request filtering rules appropriate to the exposed application risk.

## Objective

Reduce exploit delivery and malicious request exposure at the ingress layer by filtering abusive or malformed traffic before it reaches backend services.

## Rationale

TLS alone does not stop injection payloads, malformed requests, or exploit probes. Separating WAF integration from rate limiting makes request filtering and abuse throttling independently auditable.

## Scope

Applies to internet-facing ingress controllers and to internal ingress paths where risk assessment requires request filtering or abuse throttling. Covers WAF integrations, request filtering rules, and per-route or per-client rate limiting.

## Enforcement

- Primary Enforcement Point: Ingress request filtering controls.
- Implementation Surface: WAF policy configuration, managed rule attachments, custom request filtering rules, and enforcement telemetry for blocked or challenged traffic.
- Owner Primary: Network Engineering
- Owner Supporting: Application Security; Platform Engineering
