# SAF-K8S-0205-004 - eBPF-based kernel-level network policy enforcement

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.5 - kube-proxy and Node Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Clusters using eBPF-based networking (Cilium, Calico eBPF mode) must configure kernel-level network policy enforcement, validate kernel version compatibility for required eBPF features, and verify that network policies are correctly translated to eBPF programs in the kernel data path.

## Objective

Leverage eBPF's kernel-level network policy enforcement capabilities to provide high-performance, granular network segmentation that operates directly in the kernel data path.

## Rationale

eBPF-based networking provides superior performance and more granular network policy enforcement than iptables-based approaches by operating directly in the kernel data path. Kernel-level enforcement eliminates iptables rule scaling limitations and enables identity-aware, L7-capable network policies. However, eBPF features depend on specific kernel versions, and misconfiguration can silently degrade to permissive behavior. Explicit validation of policy enforcement ensures that declared network segmentation is actually applied at the kernel level.

## Scope

Applies to all clusters using eBPF-based CNI plugins or kube-proxy replacements. Clusters using traditional iptables-based networking are exempt. Managed Kubernetes services offering eBPF-based networking must document provider-managed configuration. Kernel version requirements for eBPF features must be documented and validated.

## Enforcement

- Primary Enforcement Point: CNI plugin configuration and deployment with validated kernel version compatibility.
- Implementation Surface: CNI plugin eBPF mode configuration (Cilium, Calico), kernel version requirements documentation, network policy enforcement verification tests, eBPF datapath validation tools, and kube-proxy replacement configuration.
- Owner Primary: Platform Engineering
- Owner Supporting: Network Engineering
