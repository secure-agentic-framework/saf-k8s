# SAF-K8S-0702-026 - Pipeline data classification taxonomy definition

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must define and maintain a data type classification taxonomy for pipeline data flows so data sensitivity and handling categories are consistently understood across AI pipeline resources and stages.

## Objective

Ensure pipeline data sensitivity categories are explicitly defined so downstream controls can reason about data type and sensitivity consistently.

## Rationale

Classification-based controls depend on a stable, shared taxonomy. Without a defined data classification model, downstream policy, audit, logging, and handling decisions become inconsistent and teams cannot reliably distinguish data sensitivity across pipeline stages.

## Scope

Applies to all data pipeline workloads, including ETL jobs, preprocessing steps, feature engineering pipelines, training jobs, and model-serving data paths. Covers taxonomy definition for pipeline data types and sensitivity categories.

## Enforcement

- Primary Enforcement Point: Data governance taxonomy definition and governance process.
- Implementation Surface: Organizational data classification taxonomy documentation, sensitivity category definitions, and governance records establishing how pipeline data types are classified.
- Owner Primary: Data Governance
- Owner Supporting: Platform Engineering; Security Engineering
