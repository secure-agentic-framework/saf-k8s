# SAF-K8S-0905-032 - Safe model format allowlist and unsafe deserialization blocking

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must enforce an allowlist of approved safe model formats for production loading operations and must block or quarantine models that use unsafe deserialization mechanisms such as Python pickle or unsafe torch.load patterns.

## Objective

Prevent code execution through unsafe model serialization by restricting production model intake to approved safe formats and blocking unsafe deserialization paths.

## Rationale

Pickle-based and similar unsafe model formats can execute attacker-controlled code during deserialization. Safe format allowlisting and explicit blocking of unsafe deserialization paths remove this execution path before a model reaches training or serving infrastructure.

## Scope

Applies to all model loading operations on Kubernetes, including inference servers, training pipelines loading pretrained weights, and fine-tuning jobs. Covers safe format allowlisting, unsafe-format blocking, and deserialization safeguards in production paths. Excludes deeper structural and metadata validation of model files, which is addressed separately.

## Enforcement

- Primary Enforcement Point: Model loading policy, admission or promotion gates, and runtime deserialization configuration.
- Implementation Surface: Model format allowlist policy, admission controllers rejecting pickle-based or otherwise disallowed model formats, inference and training configuration enforcing safe deserialization settings such as weights_only, and CI or promotion gates blocking unsafe deserialization usage.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; ML Engineering
