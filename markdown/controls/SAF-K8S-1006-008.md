# SAF-K8S-1006-008 - NIST SP 800-218A AI/ML profile alignment for Kubernetes AI development

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.6 - Compliance and Governance
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must map their Kubernetes and AI/ML development security practices to the NIST SP 800-218A AI/ML-specific community profile, identifying AI-focused adaptations for training data integrity, model provenance, and ML pipeline security.

## Objective

Ensure Kubernetes AI development practices align to the AI/ML-specific secure development expectations in NIST SP 800-218A so training data, model provenance, and ML pipeline risks are addressed explicitly.

## Rationale

Baseline SSDF interpretations do not fully address AI/ML-specific development risks such as treating training data as source, training runs as builds, model weight integrity as software integrity, and ML pipeline security as part of the development system. Explicit SP 800-218A alignment is required to demonstrate AI-focused secure development coverage.

## Scope

Applies to all software development and ML engineering teams building or deploying AI workloads on Kubernetes. Covers the NIST SP 800-218A AI/ML community profile and its AI-specific secure development adaptations.

## Enforcement

- Primary Enforcement Point: AI/ML secure development profile documentation and AI-specific CI/CD compliance checks.
- Implementation Surface: NIST SP 800-218A task alignment documentation, AI-specific adaptations for training data as source, training runs as builds, model weight integrity as software integrity, alignment mapping to SAF-K8S controls for training data integrity, model provenance, and ML pipeline security, and remediation documentation for AI-specific secure development gaps.
- Owner Primary: GRC/Compliance
- Owner Supporting: AI/ML Engineering; DevOps/Platform Engineering; Security Architecture
