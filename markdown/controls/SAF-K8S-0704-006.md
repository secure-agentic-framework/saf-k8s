# SAF-K8S-0704-006 - Cloud provider contingency plans for managed Kubernetes services

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.4 - Cloud Provider Security Integration
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must maintain cloud provider contingency plans for managed Kubernetes services that address provider lock-in, service disruption, and data portability requirements.

## Objective

Ensure operational continuity and data portability in the event of cloud provider service disruptions, relationship changes, or migration requirements.

## Rationale

Without cloud provider contingency plans, organizations face operational paralysis if a managed Kubernetes service experiences extended outages, pricing changes, or service deprecation. Data portability gaps can make migration prohibitively expensive or technically infeasible. Plans must be reviewed annually and after significant provider service changes.

## Scope

Applies to all managed Kubernetes services (EKS, GKE, AKS). Cloud provider contingency plans must cover data export, workload migration, and alternative deployment targets. Plans must be reviewed annually and after significant provider service changes.

## Enforcement

- Primary Enforcement Point: Documented cloud provider contingency plans with periodic validation exercises.
- Implementation Surface: Cloud provider portability assessment documentation (cloud-specific vs. cloud-agnostic component mapping), training data and model artifact export procedures to portable formats, and disaster recovery procedures for managed Kubernetes service outages including etcd backup restoration.
- Owner Primary: Security Engineering
- Owner Supporting: Cloud Platform Engineering; Platform Engineering; Legal; Procurement
