# SAF-K8S-0101-016 - API server audit log backend delivery and durable storage

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must direct API server audit logs to a durable backend, either a rotated log file or an approved webhook destination such as a SIEM, so audit events remain available and searchable after restart or backend disruption.

## Objective

Ensure API server audit records are delivered to a durable storage or forwarding destination that preserves availability for investigation and monitoring.

## Rationale

Audit event selection is ineffective if records are lost during restart, disk pressure, or forwarding failure. Durable local rotation or approved webhook delivery to downstream log infrastructure preserves the audit trail through control-plane failures and operational churn.

## Scope

Applies to all production Kubernetes clusters, including managed services exposing customer-configurable audit export settings. Covers kube-apiserver audit log path, rotation settings, webhook export configuration, and downstream durable ingestion.

## Enforcement

- Primary Enforcement Point: API server audit backend configuration and downstream durable ingestion path.
- Implementation Surface: API server manifest or managed-service audit export settings, log rotation configuration, webhook destination configuration, and SIEM or log platform ingestion controls.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Compliance
