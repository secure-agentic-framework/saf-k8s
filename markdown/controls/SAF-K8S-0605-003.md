# SAF-K8S-0605-003 - Kubewarden WebAssembly-based admission policies

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.5 - Admission Control
- Maturity: Practitioner
- Classification: baseline

## Statement

Kubernetes clusters using Kubewarden must deploy WebAssembly-based admission policies that enforce organizational security requirements, must source policies from trusted and verified policy registries, and must validate policy module signatures before deployment. Custom policies must be developed following secure coding practices and tested in a sandboxed environment before production deployment.

## Objective

Enforce Kubernetes admission policies using Kubewarden's WebAssembly-based policy engine, leveraging its language-agnostic policy development model and sandboxed execution environment.

## Rationale

Kubewarden policies execute in a WebAssembly sandbox, providing stronger isolation than native admission webhook code. However, without signature verification on policy modules, an attacker could inject malicious policies that modify or approve non-compliant workloads. Without trusted registry sourcing and pre-deployment testing, policy defects or supply chain attacks on the policy modules themselves could compromise the admission control layer.

## Scope

Applies to Kubernetes clusters where Kubewarden is the selected admission controller. Covers policy sourcing, signature verification, and custom policy development. Organizations using OPA/Gatekeeper or Kyverno exclusively may document this control as not applicable.

## Enforcement

- Primary Enforcement Point: Kubernetes Kubewarden policy server admission webhook.
- Implementation Surface: Kubewarden policy server admission webhook deployment, WebAssembly-based policy modules compiled from Rust/Go/other languages, Kubewarden policy hub (ghcr.io/kubewarden/policies/) for sourcing trusted policies, kwctl verify CLI for policy module signature verification, ClusterAdmissionPolicy and AdmissionPolicy custom resources, sandboxed Wasm execution environment for policy runtime, and custom policy development with secure coding practices and pre-deployment sandbox testing.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering
