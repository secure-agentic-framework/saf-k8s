# SAF-K8S-0505-002 - LoadBalancer, NodePort, and ExternalIP restriction policies

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.5 - API Server and Service Exposure Protection
- Maturity: Practitioner
- Classification: baseline

## Statement

Admission control policies must prevent the creation of LoadBalancer and NodePort type Services and the use of the spec.externalIPs field unless explicitly approved, ensuring that no Kubernetes service is unintentionally exposed to external networks through these mechanisms.

## Objective

Prevent accidental or unauthorized external exposure of cluster services by enforcing policy-based controls on service types and fields that create external network endpoints or intercept external traffic.

## Rationale

Any developer with permission to create a Service can expose an internal workload to the internet by specifying type LoadBalancer or NodePort. The spec.externalIPs field poses an additional risk: it allows a Service to claim an arbitrary IP address, enabling man-in-the-middle interception of traffic intended for other destinations (CVE-2020-8554). Without admission-time enforcement, detection of unintended exposure relies on post-deployment scanning, leaving a window of exposure.

## Scope

Applies to all production Kubernetes clusters. Admission policies must cover Service creation, Service type modification, and spec.externalIPs field usage. Approved exceptions must be tracked in a service exposure registry with periodic review. Managed Kubernetes environments must verify that admission policies apply before cloud load balancer provisioning.

## Enforcement

- Primary Enforcement Point: Admission controller policies (OPA/Gatekeeper, Kyverno) with constraints on Service type field values and spec.externalIPs, enforced at both creation and update time. The DenyServiceExternalIPs feature gate (stable since 1.25) can disable externalIPs at the API server level.
- Implementation Surface: Admission controller policies (Kyverno ClusterPolicy, OPA Gatekeeper ConstraintTemplates) denying Service type LoadBalancer, NodePort, and spec.externalIPs field usage, DenyServiceExternalIPs feature gate (API server --feature-gates=DenyServiceExternalIPs=true, stable since 1.25), approval annotation workflows for exception management, service exposure registry with periodic review, API server audit log detection rules for unauthorized LoadBalancer/NodePort/externalIP creation, and cluster-wide kubectl audit scripts for unapproved externally exposed services.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Application Development
