# SAF-K8S-0604-004 - VEX (Vulnerability Exploitability eXchange) publication

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.4 - SBOMs and Vulnerability Intelligence
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must produce and consume VEX (Vulnerability Exploitability eXchange) documents to communicate the exploitability status of vulnerabilities identified in container image SBOMs. VEX statements must classify each vulnerability as exploitable, not exploitable, under investigation, or fixed, with justification. VEX data must be integrated into vulnerability management workflows to prioritize remediation and reduce false-positive noise.

## Objective

Reduce vulnerability management noise and improve remediation prioritization by providing contextual exploitability information that distinguishes actionable findings from false positives.

## Rationale

Vulnerability scanners produce high volumes of findings, many of which are not exploitable in the specific deployment context (e.g., a library is present but the vulnerable code path is unreachable). Without VEX, security teams waste remediation effort on non-exploitable findings while genuinely exploitable vulnerabilities are deprioritized. VEX documents provide the contextual signal needed for accurate risk-based prioritization.

## Scope

Applies to all container images with generated SBOMs deployed in production clusters. VEX documents must be produced for internally built images and consumed from vendors for third-party images. Integration with vulnerability management platforms is required for production workloads.

## Enforcement

- Primary Enforcement Point: Vulnerability management platform configuration and CI/CD pipeline vulnerability assessment stage.
- Implementation Surface: OpenVEX format documents attached as OCI artifacts alongside image SBOMs, VEX exploitability status classifications (exploitable, not_affected, under_investigation, fixed) with justification fields, vulnerability management platform VEX data ingestion configuration (Trivy, Grype, Snyk integration), CI/CD pipeline vulnerability assessment stage VEX consumption, and vendor VEX document collection and correlation workflows for third-party images.
- Owner Primary: Security Engineering
- Owner Supporting: DevSecOps / Release Engineering; ML Platform Engineering
