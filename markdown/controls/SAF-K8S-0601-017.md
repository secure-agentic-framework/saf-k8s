# SAF-K8S-0601-017 - Multi-stage build and stripped runtime image minimization

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

All container images deployed to production clusters must use multi-stage builds to exclude build-time tooling from runtime images and must remove unnecessary packages and utilities before promotion.

## Objective

Reduce runtime container attack surface by ensuring production images exclude build-time tooling and unnecessary packages.

## Rationale

Multi-stage builds and runtime stripping are independently auditable build-hardening obligations: they prevent compilers, shells, package managers, and other non-runtime content from reaching production images even when the selected base image is already approved.

## Scope

Applies to all container images built or deployed in production and pre-production Kubernetes clusters, regardless of workload type.

## Enforcement

- Primary Enforcement Point: CI/CD build stage and image inspection controls that verify runtime image composition before promotion.
- Implementation Surface: Dockerfiles, build pipelines, OCI image build configurations, and layer inspection tooling that confirm build-time tooling and unnecessary packages are absent from production runtime images.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering
