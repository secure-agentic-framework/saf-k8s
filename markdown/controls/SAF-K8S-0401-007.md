# SAF-K8S-0401-007 - GPU resource governance permission boundaries

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.1 - Role-Based Access Control (RBAC)
- Maturity: Foundational
- Classification: ai-specific

## Statement

Organizations must restrict GPU resource management permissions, including GPU quota requests, scheduling priority overrides, and GPU metrics access, to explicitly authorized roles so that users who deploy or operate AI workloads cannot unilaterally change GPU allocation or observability boundaries without approved authorization.

## Objective

Prevent GPU resource abuse, cost overruns, and unfair scheduling by separating routine AI workload operation from privileged GPU quota, priority, and observability controls.

## Rationale

GPU-enabled environments introduce privileged controls that can materially affect cost, availability, and fairness across AI workloads. If model deployers or tenant operators can independently raise GPU quotas, override scheduling priority, or access privileged GPU telemetry, they can starve other workloads or bypass shared governance intended to protect cluster stability and multi-tenant fairness.

## Scope

Applies to all clusters running AI and ML workloads with GPU resources. Covers RBAC for GPU ResourceQuota objects, PriorityClass resources, and protected GPU monitoring or metrics endpoints. Exceptions require documented risk acceptance.

## Enforcement

- Primary Enforcement Point: RBAC restrictions on GPU quota objects, scheduling priority controls, and GPU metrics endpoint authorization.
- Implementation Surface: GPU ResourceQuota RBAC restrictions, PriorityClass resource RBAC controls, GPU metrics endpoint access policies, and identity provider group-to-role mapping configurations for privileged GPU administration boundaries.
- Owner Primary: Identity and Access Management
- Owner Supporting: Platform Engineering; ML Engineering; Security Engineering
