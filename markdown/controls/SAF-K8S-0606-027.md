# SAF-K8S-0606-027 - GitOps repository access restriction and least-privilege deploy credentials

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

GitOps deployment repositories must enforce repository access controls and least-privilege GitOps service account, deploy-key, or repository credential access so only authorized identities can read or modify deployment sources at the minimum required scope.

## Objective

Protect GitOps deployment sources from unauthorized access and over-privileged repository credentials.

## Rationale

Even with signed commits and branch protection, an attacker who can access the repository or abuse over-privileged GitOps credentials can still expose or alter deployment sources. Repository access restriction and least-privilege deploy credentials are independently auditable because they govern who can reach deployment sources and with what scope.

## Scope

Applies to Git repositories used as deployment sources for Flux or Argo CD in staging and production environments. Covers repository access controls and GitOps repository credentials, deploy keys, or service accounts.

## Enforcement

- Primary Enforcement Point: Git repository access settings and GitOps repository credential scope.
- Implementation Surface: Repository access controls in GitHub, GitLab, or Bitbucket; least-privilege deploy keys or service accounts for Flux or Argo CD; and repository credential scope limiting GitOps controllers to required read paths and actions.
- Owner Primary: Platform Engineering
- Owner Supporting: DevSecOps / Release Engineering; Security Engineering
