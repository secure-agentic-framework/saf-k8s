# SAF-K8S-1003-004 - MITRE ATT&CK for Containers coverage mapping and gap analysis

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.3 - Threat Modeling Methodologies
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must map their Kubernetes AI infrastructure defenses to the MITRE ATT&CK Threat Matrix for Containers and identify coverage gaps across tactics and techniques relevant to AI workload attack scenarios.

## Objective

Validate defensive coverage against known container and Kubernetes attack techniques and surface ATT&CK-aligned coverage gaps that require remediation.

## Rationale

MITRE ATT&CK for Containers provides a structured view of adversary behavior against containerized environments. Without explicit coverage mapping, security teams cannot tell which tactics and techniques are already mitigated, which are only partially detected, and where meaningful blind spots remain for AI infrastructure.

## Scope

Applies to all Kubernetes clusters hosting AI workloads. Coverage mapping must include infrastructure-level and AI workload-level defensive coverage and must be refreshed when ATT&CK matrix versions or core AI infrastructure architecture materially change.

## Enforcement

- Primary Enforcement Point: Detection engineering review process and ATT&CK coverage mapping documentation.
- Implementation Surface: MITRE ATT&CK Threat Matrix for Containers coverage mapping documentation, tactic-by-tactic and technique-level coverage analysis, AI workload-specific attack path mapping, gap identification across ATT&CK tactics, remediation planning for uncovered techniques, and periodic review aligned with ATT&CK matrix updates.
- Owner Primary: Security Architecture
- Owner Supporting: Detection Engineering; Security Operations; AI/ML Engineering
