# SAF-K8S-0102-024 - etcd backup repository access audit logging

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must generate auditable records for all access to etcd backup repositories and restore media.

## Objective

Ensure access to etcd backup repositories is observable through complete audit logging of backup and restore-media access activity.

## Rationale

Least-privilege backup access controls are difficult to enforce and investigate without a durable record of who accessed backup repositories or restore media. Dedicated access logging preserves the evidentiary trail needed to detect misuse and reconstruct backup-related activity.

## Scope

Applies to all production clusters. Covers object storage access logging, repository audit trails, and restore-media access records for etcd backup repositories.

## Enforcement

- Primary Enforcement Point: Backup repository access logging
- Implementation Surface: Backup repository access logs, object storage audit events, restore-media access records, and centralized retention of those records.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Backup and Recovery Operations
