# SAF-K8S-0902-006 - LLM serving configuration security

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.2 - Inference Server and Model Serving Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

LLM inference servers must enforce secure configuration of LLM-specific serving subsystems including KV cache memory isolation between concurrent requests, continuous batching queue depth and priority controls, and prefix caching tenant isolation to prevent cross-request information leakage and resource exhaustion through serving-layer misconfigurations.

## Objective

Prevent cross-request data leakage and resource abuse through secure configuration of LLM-specific serving subsystems (KV cache, continuous batching, prefix caching) that have no equivalent in traditional inference servers.

## Rationale

LLM inference servers (vLLM, TGI, Triton with vLLM backend) introduce serving subsystems unique to autoregressive generation: KV caches that persist attention state across tokens, continuous batching that interleaves multiple requests on a single GPU, and prefix caching that shares computed prefixes across requests. Without memory isolation on KV caches, one request may read residual attention state from a prior request. Without continuous batching controls, a single long-generation request can monopolize the batch slot and starve other requests. Without prefix caching tenant isolation, cached prefixes from one tenant's system prompt can leak to another tenant's inference path in multi-tenant deployments.

## Scope

Applies to all LLM-specific inference servers deployed on Kubernetes, including vLLM, TGI, and Triton with LLM backends. Covers KV cache configuration, continuous batching parameters, and prefix caching settings. Does not apply to non-generative model serving (image classification, embedding models) where these subsystems are absent.

## Enforcement

- Primary Enforcement Point: LLM inference server configuration (vLLM engine args, TGI serving parameters), Kubernetes resource limits for KV cache memory, and serving framework tenant isolation settings.
- Implementation Surface: LLM inference server configuration parameters (vLLM engine args: max_model_len, gpu_memory_utilization, max_num_seqs; TGI serving parameters), KV cache memory isolation and zeroing configuration, continuous batching queue depth limits and priority-based preemption settings, prefix caching tenant-aware cache key configuration, per-request generation deadline settings, and memory profiling tooling for KV cache leakage verification.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Platform Engineering
