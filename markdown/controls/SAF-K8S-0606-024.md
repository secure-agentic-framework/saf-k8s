# SAF-K8S-0606-024 - Policy-as-code and runtime configuration integrity governance

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Runtime settings and policy-as-code definitions must be stored in integrity-protected configuration code repositories, with authorized modification controls preventing unauthorized changes to network policies, RBAC, pod security, admission rules, and other runtime governance settings.

## Objective

Ensure security-critical runtime configuration remains tamper-evident, reviewable, and protected from unauthorized modification.

## Rationale

Even when infrastructure templates are scanned, attackers can still weaken the runtime environment by changing policy-as-code definitions or runtime governance settings if those repositories lack integrity protection. Protecting policy code and runtime configuration is therefore independently auditable from IaC scanning.

## Scope

Applies to policy-as-code and runtime configuration repositories used to define Kubernetes security settings in staging and production environments, including RBAC, network policy, admission policy, and pod security definitions.

## Enforcement

- Primary Enforcement Point: Integrity controls on policy-as-code repositories and runtime configuration sources.
- Implementation Surface: Integrity-protected configuration code repositories, signed or otherwise verified policy changes, protected change workflows for runtime settings, and auditability for modifications to policy-as-code and security configuration definitions.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
