# SAF-K8S-0205-005 - eBPF program integrity verification and loading monitoring

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.5 - kube-proxy and Node Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Nodes running eBPF-based networking must validate that eBPF programs are loaded from trusted and integrity-verified sources, restrict eBPF capabilities to authorized components, and monitor eBPF program loading events to detect unauthorized kernel-level network manipulation.

## Objective

Ensure that eBPF programs executing with kernel-level privileges are trustworthy and that unauthorized eBPF program loading is detected, preventing attackers from using rogue eBPF programs to bypass network policies or exfiltrate data.

## Rationale

eBPF programs execute with kernel-level privileges, meaning a compromised or malicious eBPF program can intercept, modify, or redirect any network traffic on the node without application-layer visibility. Without integrity verification and loading monitoring, an attacker who gains sufficient node access could load rogue eBPF programs to exfiltrate data, bypass network policies, or create covert communication channels, all invisible to traditional network monitoring tools.

## Scope

Applies to all nodes running eBPF-based CNI plugins or other eBPF networking components. eBPF capability restrictions (CAP_BPF, CAP_SYS_ADMIN) must be enforced for all non-CNI workloads. Managed Kubernetes services must document provider-managed eBPF program integrity controls.

## Enforcement

- Primary Enforcement Point: Runtime monitoring of bpf() syscalls and eBPF program loading events, combined with container image integrity verification for CNI components.
- Implementation Surface: eBPF capability restriction (CAP_BPF, CAP_SYS_ADMIN denial for non-CNI workloads), eBPF program loading detection (auditd bpf syscall rules, Falco/Tetragon rules), eBPF program source integrity verification (container image digest pinning for CNI), and eBPF verification tools (bpftool prog list).
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Network Engineering
