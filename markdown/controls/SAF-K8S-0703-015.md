# SAF-K8S-0703-015 - Chaos engineering validation for AI resource governance controls

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.3 - Resource Governance and Priority
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must conduct periodic chaos engineering tests against GPU resource controls, queue management, auto-scaling limits, and recovery procedures to validate that AI resource governance controls function correctly under realistic failure and saturation scenarios.

## Objective

Validate AI resource governance controls through controlled failure injection so resource exhaustion, queue backlogs, and GPU failure conditions are discovered and corrected before production incidents occur.

## Rationale

Resource governance controls are only trustworthy if they are exercised under realistic stress and failure modes. Chaos engineering reveals whether queue bounds, PriorityClass behavior, disruption budgets, recovery procedures, and autoscaling controls actually preserve resilience under GPU faults, OOM conditions, and scheduler saturation.

## Scope

Applies to cloud-hosted Kubernetes clusters with AI workloads and periodic resilience testing programs. Covers test schedules, failure injection targeting GPU devices and queues, automated resilience verification, and lessons-learned feedback into guardrail tuning and recovery procedures.

## Enforcement

- Primary Enforcement Point: Chaos engineering test schedules and recovery validation workflows for AI resource governance controls.
- Implementation Surface: Chaos engineering tooling targeting GPU device failure, training job OOM, scheduler saturation, queue backlog scenarios, PDB and PriorityClass validation under fault conditions, automated resilience test pipelines, and documented recovery procedures for each exercised failure mode.
- Owner Primary: SRE
- Owner Supporting: Platform Engineering; Security Engineering; FinOps
