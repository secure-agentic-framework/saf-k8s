# SAF-K8S-0802-007 - Device plugin registration authentication monitoring and rogue plugin detection

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.2 - GPU Driver, Library, and Toolkit Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Device plugin registration with the kubelet must be authenticated and continuously monitored, and unauthorized or rogue GPU device plugin registration or interface abuse attempts must be detected and blocked.

## Objective

Ensure only approved GPU device plugins can register with the kubelet and that rogue plugin behavior or manipulated health reporting is detected before it can influence GPU allocation decisions.

## Rationale

Even if socket-directory access is restricted, organizations still need assurance that only authorized plugins register and that abnormal registration, fake device advertisement, or manipulated health reporting is detected. This is a separate obligation from filesystem protection because it focuses on registration integrity and runtime detection of rogue plugin behavior.

## Scope

Applies to all Kubernetes nodes where GPU device plugins register with the kubelet. Covers plugin registration events, kubelet authorization or attestation of approved plugins, and monitoring of device health reporting or rogue registration behavior.

## Enforcement

- Primary Enforcement Point: Device-Plugin Registration Integrity Monitoring
- Implementation Surface: kubelet device plugin registration logs, approved plugin inventory and attestation, runtime security tooling for rogue registration detection, and monitoring of device health reporting callbacks or anomalous plugin behavior.
- Owner Primary: Platform Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
