# SAF-K8S-0102-020 - etcd certificate maximum validity period enforcement

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters must enforce organization-defined maximum validity periods for etcd server, peer, and API server-to-etcd client certificates through certificate issuance policy or equivalent configuration controls.

## Objective

Reduce key-compromise exposure by constraining etcd-related certificates to approved maximum validity windows.

## Rationale

Certificate rotation loses much of its security value when certificate validity windows remain excessively long. Enforcing bounded validity periods limits the blast radius of certificate compromise and ensures etcd-related identities are re-issued on a deliberate cadence aligned to organizational policy.

## Scope

Applies to all self-managed Kubernetes clusters using etcd. Covers certificate lifetime policy and issuance profile configuration for etcd server, peer, and API server-to-etcd client certificates.

## Enforcement

- Primary Enforcement Point: Certificate issuance policy enforcement
- Implementation Surface: CA or issuer policy configuration, certificate profile templates, and periodic certificate inventory review for etcd-related identities.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering
