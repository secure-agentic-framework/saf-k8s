# SAF-K8S-0904-001 - Pipeline orchestrator hardening

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.4 - AI Pipeline Orchestration and Experimentation Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

ML pipeline orchestrators (Kubeflow Pipelines, Argo Workflows, Airflow, Tekton) must enforce RBAC for pipeline definition creation and modification, execute pipeline steps in isolated security contexts, and manage secrets through Kubernetes-native or external secret stores rather than embedding credentials in pipeline definitions.

## Objective

Prevent unauthorized pipeline creation or modification, contain the blast radius of compromised pipeline steps, and protect secrets used by pipeline stages from exposure.

## Rationale

ML pipelines orchestrate the end-to-end workflow from data ingestion through model deployment. Without RBAC on pipeline definitions, any authenticated user can create or modify pipelines to exfiltrate data or deploy malicious models. Without execution isolation, a compromised pipeline step can access resources belonging to other steps or namespaces. Secrets embedded in pipeline definitions are exposed in pipeline logs, version history, and API responses.

## Scope

Applies to all ML pipeline orchestration platforms deployed on Kubernetes, including Kubeflow Pipelines, Argo Workflows, Airflow on Kubernetes, and Tekton. Covers pipeline definition RBAC, step execution contexts, and secret management. Excludes CI/CD pipelines for application code unless they execute ML workloads.

## Enforcement

- Primary Enforcement Point: Pipeline orchestrator RBAC configuration, Kubernetes namespace and pod security policies for pipeline execution, and external secret store integrations (Vault, AWS Secrets Manager, Kubernetes Secrets with encryption).
- Implementation Surface: ML pipeline orchestrator RBAC configuration (Kubeflow Pipelines namespace-scoped profiles, Argo Workflows WorkflowTemplate restrictions, Airflow DAG-level RBAC, Tekton Pipeline/Task RBAC), per-step pod security contexts with isolated service accounts, external secret store integrations (HashiCorp Vault sidecar injection, AWS Secrets Manager, Kubernetes External Secrets Operator), GitOps pipeline definition deployment with PR review requirements, and CI credential scanning for hardcoded secrets in pipeline definitions.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; DevOps Engineering
