# SAF-K8S-0104-019 - Kubernetes CVE risk prioritization framework

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.4 - CIS Benchmarks and Patch Management
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations must maintain a documented Kubernetes CVE prioritization framework that classifies vulnerabilities by severity, exploitability, exposure, and cluster-specific impact to drive timely triage decisions.

## Objective

Ensure remediation effort is focused first on the Kubernetes vulnerabilities with the highest real risk to the organization's clusters and workloads.

## Rationale

Awareness of Kubernetes CVEs is insufficient if teams treat every disclosure equally. A documented prioritization framework makes triage consistent and risk-informed by accounting for exploitability, exposed attack surface, tenant impact, and operational context so the most dangerous vulnerabilities are addressed first.

## Scope

Applies to all production and non-production clusters. Covers Kubernetes core component CVEs, container runtime CVEs, and critical add-on vulnerabilities where prioritization must consider CVSS, exploitability, exposed components, tenant impact, and provider patch timelines.

## Enforcement

- Primary Enforcement Point: Vulnerability triage governance
- Implementation Surface: Documented CVE prioritization workflow, severity and SLA criteria, exploitability or exposure scoring inputs, and triage decision records for Kubernetes-related vulnerabilities.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Incident Response
