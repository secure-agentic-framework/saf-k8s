# SAF-K8S-0901-003 - Checkpoint security

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.1 - Distributed Training Workload Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Model checkpoints and optimizer state must be stored with encryption at rest, verified for integrity before restoration, and protected by access controls that restrict read and write operations to authorized training jobs and personnel.

## Objective

Protect model checkpoints from tampering, unauthorized access, and corruption to ensure that training can resume from a known-good state and that checkpoint data containing model weights is not exfiltrated.

## Rationale

Model checkpoints contain the full model weights and optimizer state at a point in time, representing significant intellectual property and compute investment. Without encryption, checkpoints can be stolen from shared storage. Without integrity verification, a tampered checkpoint can inject backdoors into the model when training resumes. Without access controls, unauthorized users can read or overwrite checkpoint data.

## Scope

Applies to all model checkpoint and optimizer state files written during training to persistent volumes, object storage, or network file systems on Kubernetes. Covers both scheduled checkpoints and fault-recovery checkpoints. Includes checkpoint files in all formats (PyTorch .pt, TensorFlow SavedModel, ONNX).

## Enforcement

- Primary Enforcement Point: Storage class encryption configuration, persistent volume access policies, and training job service account RBAC bindings for checkpoint storage paths.
- Implementation Surface: Kubernetes StorageClass encryption-at-rest configuration, PersistentVolume and PVC access mode settings (ReadWriteOnce, ReadOnlyMany), RBAC bindings for checkpoint storage service accounts, SHA-256 hash manifests for checkpoint integrity verification, object storage encryption (S3 SSE, GCS CMEK), checkpoint retention and lifecycle policies, and application-level checkpoint encryption tooling for PyTorch .pt, TensorFlow SavedModel, and ONNX formats.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Storage Engineering; Security Engineering
