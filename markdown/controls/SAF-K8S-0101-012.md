# SAF-K8S-0101-012 - API server TLS enforcement

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must enforce TLS 1.2 or later with organization-approved cipher suites for all API server communication paths, including client-facing and back-end connections, using certificates issued by a trusted CA.

## Objective

Protect API server communication paths from eavesdropping, tampering, and impersonation by enforcing strong TLS on every connection path.

## Rationale

The API server communicates with external clients and internal cluster components across multiple connection paths. Weak or absent TLS exposes credentials, cluster state, and control-plane operations to network attackers and intermediary systems.

## Scope

Applies to all API server communication paths: client to API server (front-end), API server to etcd, API server to kubelet, API server to aggregated API servers, and webhook backends. For managed Kubernetes services, applies to customer-configurable TLS settings and to verification that provider-managed connections meet organizational TLS requirements.

## Enforcement

- Primary Enforcement Point: API Server Configuration
- Implementation Surface: Cluster provisioning templates, PKI configuration, managed-service TLS settings, and periodic TLS configuration assessment.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; PKI/Certificate Management
