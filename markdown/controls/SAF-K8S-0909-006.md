# SAF-K8S-0909-006 - Vector index lifecycle management

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.9 - RAG Infrastructure Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Vector indexes used for RAG must implement lifecycle controls including versioned index snapshots, integrity verification of index state, controlled rebuild and migration procedures, and secure deletion of retired indexes and their associated embedding data.

## Objective

Ensure that vector indexes can be verified for integrity, rolled back to known-good states after incidents, rebuilt when embedding models change, and securely deleted when retired.

## Rationale

Vector indexes are mutable data structures that change as documents are added, updated, or deleted. Without lifecycle controls, there is no way to detect silent corruption, roll back to a known-good state after a poisoning incident, or ensure that deleted documents are actually removed from the index. Embedding model changes require full index rebuilds; mixing embeddings from different model versions creates retrieval quality and security issues.

## Scope

Applies to all vector indexes used for RAG on Kubernetes. Covers index snapshots, integrity verification, rebuild procedures, migration between embedding model versions, and secure deletion. Excludes general database backup and recovery (see D7).

## Enforcement

- Primary Enforcement Point: Vector database snapshot and backup configuration, index integrity verification jobs, and storage lifecycle policies for vector index volumes.
- Implementation Surface: Vector database snapshot and backup configuration with immutable storage, snapshot retention policies, periodic integrity verification jobs (vector count validation, metadata consistency, cryptographic checksums or Merkle trees), documented and tested full index rebuild procedures with recovery time measurement, embedding model version-aware migration requiring full rebuild, secure deletion verification (compaction for soft deletes, PV secure erase or crypto-shredding for retired indexes), and storage lifecycle policies for vector index volumes.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Infrastructure Engineering; Security Engineering
