# SAF-K8S-0303-006 - MAC profile iterative refinement cycle

- Domain: D03 - Workload and Pod Security
- Knowledge Area: 3.3 - Mandatory Access Controls
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must iteratively refine MAC profiles through successive audit-mode test cycles, incorporating any legitimate operations observed as denied into revised profile versions, and must require at least two consecutive clean audit cycles before activating enforcement mode.

## Objective

Ensure MAC profiles are complete and accurate for all workload code paths, including infrequent paths such as health checks, garbage collection, and scheduled operations, before enforcement is activated, by requiring systematic iterative refinement until successive clean cycles confirm profile completeness.

## Rationale

A single audit-mode observation cycle may not exercise all legitimate workload code paths, particularly infrequent operations such as periodic health checks, garbage collection syscalls, checkpoint saves, or emergency error-handling paths. Requiring iterative refinement across multiple cycles ensures that profiles are complete before enforcement is activated, preventing operational disruptions caused by profiles that are accurate for common paths but miss edge cases.

## Scope

Applies to all generated MAC profiles before enforcement mode activation and after any workload update that could introduce new syscalls or file access patterns. The iterative cycle must be repeated whenever the workload binary, its dependencies, or the underlying runtime environment changes significantly.

## Enforcement

- Primary Enforcement Point: Profile version control and change management process, with iterative cycle completion verified before enforcement mode is approved in the deployment pipeline.
- Implementation Surface: Version-controlled profile files with change history documenting additions between refinement iterations, iterative refinement workflow across model load, training epoch, checkpoint, inference, and health check code paths, cycle completion tracking, and deployment pipeline gates requiring documented clean audit cycles before enforcement.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Application Development teams
