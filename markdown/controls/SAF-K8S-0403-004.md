# SAF-K8S-0403-004 - Secret rotation and expiration enforcement

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations must enforce secret rotation on a defined schedule, set expiration policies for all secrets, and implement revocation procedures for compromised credentials to limit the useful lifetime of any individual secret.

## Objective

Limit the useful lifetime of any individual secret through enforced rotation and expiration, reducing the window of opportunity for credential-based attacks.

## Rationale

Secrets that are never rotated remain valid indefinitely, giving attackers unlimited time to exploit stolen credentials. Without expiration enforcement, accumulated secrets create an ever-growing attack surface. Enforced rotation and expiration ensure that even if a credential is compromised, it becomes invalid within a bounded time window.

## Scope

Applies to all secret types across all clusters: API keys, database credentials, TLS private keys, service account tokens, and cloud provider credentials. Covers rotation schedules, expiration policies, and revocation runbooks.

## Enforcement

- Primary Enforcement Point: External secrets management rotation policies and credential expiration enforcement in secret stores.
- Implementation Surface: External secrets manager rotation policies (Vault dynamic secret TTLs, AWS Secrets Manager rotation lambdas, GCP Secret Manager rotation schedules), credential expiration enforcement in secret stores, and incident response runbooks for credential revocation.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Application Development
