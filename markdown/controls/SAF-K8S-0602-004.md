# SAF-K8S-0602-004 - Notary v2 trust policy and signing identity governance

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.2 - Image Signing and Admission Enforcement
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations using Notary v2 (notation) must configure trust policies that specify authorized signing identities and trusted certificate chains for notation signature validation.

## Objective

Establish governed trust policy boundaries for which Notary v2 signing identities are authorized to sign deployable artifacts.

## Rationale

Trust policy governance is an independent control claim from signature storage or admission verification. Without explicit notation trust policies and approved signing identity chains, verification can succeed against unauthorized publishers or unmanaged certificate roots.

## Scope

Applies to all container registries and Kubernetes clusters where Notary v2 is the designated or supplementary signing mechanism. Includes environments using Azure Container Registry, AWS ECR, or other registries with native notation support. Organizations exclusively using Sigstore/cosign may document this control as not applicable.

## Enforcement

- Primary Enforcement Point: Notation trust policy configuration and signing identity trust-root management.
- Implementation Surface: trustpolicy.json configuration, trusted signing identity definitions, certificate chain specifications, organizational PKI support for notation signing identities, and documented trust policy governance for Notary v2.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; DevSecOps / Release Engineering
