# SAF-K8S-0905-052 - Approved external model source allowlist definition and maintenance

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must define and maintain a curated allowlist of approved external model sources using documented approval criteria for permitted repositories, publishers, or registry namespaces.

## Objective

Restrict external model ingestion to a governed set of explicitly approved sources through defined allowlist entries and documented approval criteria.

## Rationale

Without a defined allowlist and explicit source-approval criteria, engineers can source pre-trained models and checkpoints from arbitrary public repositories, including typosquatted or impersonated publishers distributing malicious artifacts under trusted names.

## Scope

Applies to all external model sources accessed from Kubernetes infrastructure. Covers approved-source criteria, repository URL and publisher allowlists, and governance ownership for deciding which external sources are permitted. Excludes periodic revalidation and stale-source review activities.

## Enforcement

- Primary Enforcement Point: Approved-source policy and allowlist definition process.
- Implementation Surface: Approved external model source allowlists covering repository URLs, organization identifiers, or registry namespaces, with documented approval criteria and governance ownership for deciding which external sources may be used.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Governance
