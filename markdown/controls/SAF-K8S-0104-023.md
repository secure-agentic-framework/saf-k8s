# SAF-K8S-0104-023 - Stage-based approval and risk assessment for production feature gates

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.4 - CIS Benchmarks and Patch Management
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters must govern alpha- and beta-stage feature gate enablement through explicit approval and documented security risk assessment before those gates are allowed in production.

## Objective

Ensure unstable or incompletely assessed feature gates do not enter production clusters without stage-appropriate security review and approval.

## Rationale

Alpha and beta feature gates change cluster behavior in ways that may be unstable, insufficiently reviewed, or unsupported in production. Stage-based approval and documented risk assessment prevent experimental feature behavior from bypassing security review.

## Scope

Applies to all production clusters and all alpha- or beta-stage feature gates on API server, controller-manager, scheduler, kubelet, kube-proxy, and managed service surfaces where feature gate behavior can be influenced or must be tracked.

## Enforcement

- Primary Enforcement Point: Cluster change approval and security review processes governing production feature gate enablement.
- Implementation Surface: Change approval workflows for production feature gates, documented security risk assessments for alpha or beta gates, prohibition of unapproved alpha gates, and production configuration scans verifying only approved feature gates are active.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Architecture Review
