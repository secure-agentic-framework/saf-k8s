# SAF-K8S-0403-020 - Kubernetes Secrets KMS key least-privilege access policy

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Foundational
- Classification: baseline

## Statement

KMS keys used to protect Kubernetes Secrets must enforce least-privilege access to encryption and decryption operations.

## Objective

Ensure only approved Kubernetes control-plane or managed encryption identities can use the KMS keys protecting Kubernetes Secrets.

## Rationale

Using an external KMS backend is insufficient if the KMS key policy is over-broad. This control isolates least-privilege key governance so teams can independently verify that only approved identities may encrypt or decrypt Kubernetes Secrets.

## Scope

Applies to all production Kubernetes clusters. Covers key policies, grants, IAM bindings, and periodic access review for KMS keys protecting Kubernetes Secrets. External KMS provider integration remains governed by SAF-K8S-0403-019.

## Enforcement

- Primary Enforcement Point: External KMS key policies, grants, and IAM bindings for Secrets-protecting keys.
- Implementation Surface: Cloud KMS or HSM key policies, grants, IAM roles, and service-account bindings used by the Kubernetes control plane or managed encryption service.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Key Management/Cryptography; Cloud Platform Engineering
