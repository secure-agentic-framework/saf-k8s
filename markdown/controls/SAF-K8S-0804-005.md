# SAF-K8S-0804-005 - Attestation-conditioned enclave key release

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.4 - Confidential Computing for AI Workloads
- Maturity: Advanced
- Classification: ai-specific

## Statement

Encryption keys protecting models and data within confidential computing enclaves must be managed through a hardware-rooted key management system with key release conditioned on successful TEE attestation.

## Objective

Ensure cryptographic keys for model decryption and data access are released only to verified TEE environments that present approved attestation evidence.

## Rationale

If model or data encryption keys are released without attestation-conditioned policy checks, a compromised host or operator can retrieve the keys and decrypt protected assets outside the enclave. This obligation is distinct from sealed storage because it governs live key release decisions made by the KMS at access time.

## Scope

Applies to confidential AI workloads where encrypted models or data require key release into a TEE enclave. Covers hardware-rooted KMS integration, attestation-conditioned release policies, and audit of key release decisions tied to attestation results.

## Enforcement

- Primary Enforcement Point: Attestation-Conditioned KMS Release Policy
- Implementation Surface: Hardware-rooted key management systems, attestation-conditioned key release policies, audit correlation between attestation verification and key release, and enclave model decryption workflows using KMS-mediated release.
- Owner Primary: Security Engineering
- Owner Supporting: Key Management; ML Platform Engineering; GPU Infrastructure Engineering
