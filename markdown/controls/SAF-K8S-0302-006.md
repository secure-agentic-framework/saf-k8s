# SAF-K8S-0302-006 - No-new-privileges execution enforcement

- Domain: D03 - Workload and Pod Security
- Knowledge Area: 3.2 - Security Contexts and Capabilities
- Maturity: Foundational
- Classification: baseline

## Statement

All containers must set allowPrivilegeEscalation to false so the runtime applies the no_new_privs flag, and admission controls must deny pods, init containers, and ephemeral debug containers that permit privilege escalation unless a documented exception with compensating controls is approved.

## Objective

Prevent container processes from gaining additional privileges after start through setuid, setgid, or capability-raising execve paths.

## Rationale

The no_new_privs execution flag blocks privilege-raising binaries and exec transitions even when an attacker reaches code execution inside the container. Without this control, a compromised workload can convert a limited foothold into higher-privilege execution through setuid binaries or similar runtime escalation paths.

## Scope

Applies to all pods and containers in production clusters, including init containers and ephemeral debug containers.

## Enforcement

- Primary Enforcement Point: Pod Security Admission (Restricted level) and policy engine rules validating securityContext.allowPrivilegeEscalation for all container types.
- Implementation Surface: Pod and container securityContext.allowPrivilegeEscalation settings and admission validation for no_new_privs enforcement.
- Owner Primary: Application Development teams
- Owner Supporting: Platform Engineering; Security Engineering
