# SAF-K8S-0701-003 - Encryption at rest for persistent volumes

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.1 - Persistent Storage Security
- Maturity: Foundational
- Classification: baseline

## Statement

All persistent volumes must be encrypted at rest using cloud provider-managed encryption, dm-crypt/LUKS, or storage-level encryption mechanisms, with encryption keys managed through a dedicated key management service and rotated according to organizational key rotation policy.

## Objective

Protect data stored on persistent volumes from unauthorized access in the event of physical media theft, improper disk decommissioning, or unauthorized access to underlying storage infrastructure.

## Rationale

Persistent volumes may contain sensitive training data, model weights, inference results, and configuration secrets. Without encryption at rest, an attacker who gains access to the underlying storage media or cloud storage layer can read data directly, bypassing all Kubernetes access controls. Regulatory frameworks such as PCI DSS and HIPAA require encryption at rest for sensitive data.

## Scope

Applies to all persistent volumes across all Kubernetes clusters, regardless of data classification. Covers cloud provider-managed encryption (e.g., AWS EBS encryption, GCE PD encryption, Azure Disk encryption), node-level dm-crypt/LUKS, and storage-backend-native encryption. Exceptions for ephemeral scratch volumes must be documented with justification.

## Enforcement

- Primary Enforcement Point: StorageClass encryption parameters, cloud provider volume encryption settings, node-level disk encryption configuration, and admission control policies that reject PVCs bound to unencrypted StorageClasses.
- Implementation Surface: Cloud provider storage encryption settings (AWS EBS encryption, GCP Persistent Disk encryption, Azure Disk encryption), cloud KMS services (AWS KMS, GCP Cloud KMS, Azure Key Vault) for customer-managed encryption keys (CMK/CMEK), on-premises block-level encryption (dm-crypt/LUKS), storage-backend-native encryption (Ceph, Longhorn), StorageClass encryptionConfiguration parameters, key rotation policy automation, HashiCorp Vault or Thales CipherTrust for on-premises key management, and LUKS header validation tooling.
- Owner Primary: Platform Engineering
- Owner Supporting: Cloud Platform Engineering; Security Engineering
