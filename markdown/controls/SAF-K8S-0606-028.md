# SAF-K8S-0606-028 - GitOps commit signing and protected deployment branch governance

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

GitOps deployment repositories must require signed commits on deployment branches and enforce branch protection rules including pull request approval and required status checks before reconciled changes can reach controlled environments.

## Objective

Ensure changes to Git-based deployment sources are cryptographically attributable and pass protected review gates before reconciliation.

## Rationale

Repository access restriction alone does not ensure that changes are reviewable, attributable, and policy-compliant. Signed commits and protected deployment branches are independently auditable because they govern how authorized changes are validated and approved before GitOps reconciliation occurs.

## Scope

Applies to Git repositories used as deployment sources for Flux or Argo CD in staging and production environments. Covers commit signature verification, pull request review, and protected deployment branch policy.

## Enforcement

- Primary Enforcement Point: Deployment branch governance and commit verification policy.
- Implementation Surface: Commit signature verification on GitOps deployment branches, protected branch rules with pull request approval and required status checks, and review workflows governing changes before GitOps reconciliation.
- Owner Primary: Platform Engineering
- Owner Supporting: DevSecOps / Release Engineering; Security Engineering
