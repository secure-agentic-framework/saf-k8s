# SAF-K8S-0404-002 - TLS provisioning for webhooks, API aggregation, and internal services

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.4 - Certificate Management
- Maturity: Practitioner
- Classification: baseline

## Statement

All admission webhooks, API aggregation layer endpoints, and internal service-to-service communication endpoints must be provisioned with valid TLS certificates signed by a trusted cluster CA, and the API server must be configured to validate these certificates.

## Objective

Ensure that all internal Kubernetes control plane extensions and service communication channels are protected by authenticated TLS, preventing man-in-the-middle attacks on webhook calls, API aggregation, and inter-service traffic.

## Rationale

Admission webhooks receive all API requests matching their rules and can modify or reject them. An attacker who can intercept webhook traffic via a man-in-the-middle attack could manipulate admission decisions to allow malicious workloads or block legitimate ones. API aggregation layer endpoints extend the Kubernetes API surface. Without proper TLS, these critical control plane communication paths are vulnerable to interception and tampering.

## Scope

Applies to all ValidatingWebhookConfigurations, MutatingWebhookConfigurations, and APIService resources in all clusters. Covers both built-in webhooks (e.g., Pod Security Admission) and third-party webhooks (OPA Gatekeeper, Kyverno, operator webhooks). Internal service endpoints that serve as backends for API aggregation are also in scope.

## Enforcement

- Primary Enforcement Point: Webhook and APIService configurations specifying caBundle or cert-manager Certificate resources, with TLS certificate validity verified by the API server during webhook and aggregation calls.
- Implementation Surface: ValidatingWebhookConfiguration and MutatingWebhookConfiguration resources (caBundle field, cert-manager CA injection annotations), APIService resources for aggregation layer endpoints, cert-manager Certificate resources for webhook and aggregation TLS, API server TLS verification settings for webhook and aggregation calls, and audit tooling verifying absence of --insecure-skip-tls-verify flags and failurePolicy: Ignore on security-critical webhooks.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering
