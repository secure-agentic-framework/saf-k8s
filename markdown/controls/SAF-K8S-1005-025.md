# SAF-K8S-1005-025 - Kubernetes forensic evidence acquisition for container, node, audit, and network artifacts

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.5 - Incident Response for Kubernetes
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must establish forensic evidence acquisition procedures for Kubernetes incidents, including container or node state capture, audit log preservation, and network flow capture for collected infrastructure evidence.

## Objective

Preserve complete Kubernetes infrastructure evidence from incidents through documented acquisition procedures that capture volatile container, node, audit, and network artifacts before they are lost.

## Rationale

Kubernetes evidence is highly ephemeral: pod filesystems disappear, node state changes quickly, and network flows are not retained by default. Without prepared acquisition procedures, responders can lose critical infrastructure evidence before investigation begins.

## Scope

Applies to Kubernetes clusters hosting AI workloads where forensic readiness is required. Covers container-level, node-level, audit-log, and network-level evidence acquisition for Kubernetes infrastructure incidents.

## Enforcement

- Primary Enforcement Point: Forensic toolkit deployment and evidence acquisition procedures for Kubernetes infrastructure artifacts.
- Implementation Surface: Container and node snapshot tooling, immutable audit log export, network flow capture, metadata preservation commands, and infrastructure evidence repository handling used during Kubernetes incident response.
- Owner Primary: Security Operations
- Owner Supporting: Digital Forensics; AI/ML Engineering; Legal
