# SAF-K8S-0702-024 - Tenant namespace-scoped RBAC boundary enforcement

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must enforce tenant boundaries at the Kubernetes API layer by restricting tenant users and service accounts to namespace-scoped RBAC bindings and by reviewing grants to prevent cross-namespace or shared-cluster privilege expansion.

## Objective

Prevent cross-tenant API access by enforcing namespace-scoped RBAC boundaries for tenant identities.

## Rationale

RBAC determines who can act through the Kubernetes API. If tenant users or service accounts receive cross-namespace or cluster-wide grants, they can bypass tenant isolation even when runtime network boundaries exist. Namespace-scoped RBAC enforcement is therefore a distinct tenant-boundary control.

## Scope

Applies to all multi-tenant Kubernetes clusters. Tenant users and service accounts must be bound only to approved namespace-scoped roles unless an explicitly governed shared-service exception exists.

## Enforcement

- Primary Enforcement Point: RoleBinding and ClusterRoleBinding review for cross-namespace or cluster-wide grant detection.
- Implementation Surface: Namespace-scoped RoleBindings, restricted ClusterRoleBindings, service account binding reviews, and recurring RBAC audit evidence confirming tenant identities remain scoped to their namespaces.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Network Engineering
