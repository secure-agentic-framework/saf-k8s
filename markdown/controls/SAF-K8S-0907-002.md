# SAF-K8S-0907-002 - Training data privacy controls

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.7 - Feature Store Security and Data Access Controls
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Training data must be subject to PII handling procedures, data classification, anonymization or pseudonymization where required, and data access audit trails that record who accessed which datasets and when.

## Objective

Protect personal and sensitive information within training datasets through classification, handling controls, and comprehensive access auditing to support regulatory compliance and breach investigation.

## Rationale

Training datasets frequently contain PII, proprietary information, or data subject to regulatory requirements. Without data classification, sensitive data is treated identically to public data, lacking appropriate protections. Without anonymization, PII can be memorized by models and exposed through inference. Without access audit trails, organizations cannot demonstrate compliance with data protection regulations or investigate unauthorized data access during security incidents.

## Scope

Applies to all training datasets on Kubernetes, including raw data, processed data, and feature data containing PII or sensitive information. Covers data classification, PII handling, anonymization, and access audit trails. References Appendix A: AI Data Privacy Governance for consent management, privacy notices, and individual data rights. Excludes inference-time data controls.

## Enforcement

- Primary Enforcement Point: Data classification tooling, anonymization pipeline configuration, and storage and compute platform access audit logging settings.
- Implementation Surface: Data classification tooling and labels (public, internal, confidential, restricted), PII detection and scanning at data ingestion, anonymization and pseudonymization pipelines with k-anonymity/l-diversity/t-closeness validation, automated residual PII scanning, data access audit logging (principal, dataset ID, version, timestamp) shipped to centralized SIEM, and storage platform access audit configuration for PersistentVolumes, object storage, and feature stores.
- Owner Primary: Data Engineering
- Owner Supporting: Privacy Engineering; Security Engineering; Compliance
