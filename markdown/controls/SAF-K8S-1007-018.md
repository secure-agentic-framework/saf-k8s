# SAF-K8S-1007-018 - Deprecated Kubernetes API usage inventory and migration tracking for AI workloads

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.7 - Cluster Lifecycle and Asset Inventory
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must maintain an inventory of deprecated Kubernetes APIs used by cluster workloads, operators, custom resources, Helm charts, and manifests, and must notify responsible teams with tracked migration plans and completion status before production upgrades proceed.

## Objective

Discover deprecated Kubernetes API usage in AI environments and drive accountable migration completion before removed APIs break production workloads during upgrades.

## Rationale

Upstream deprecation awareness alone is insufficient unless teams know exactly which workloads and controllers still depend on deprecated APIs. Inventorying that usage and tracking migration completion prevents removed APIs from causing upgrade failures in AI platforms.

## Scope

Applies to all Kubernetes clusters scheduled for version upgrades. Discovery must cover AI workload namespaces, operator deployments, Helm chart templates, and custom resource definitions used by AI platforms and services.

## Enforcement

- Primary Enforcement Point: Deprecation scanning tools in CI/CD pipelines, manifest and cluster scan workflows, and migration tracking dashboards.
- Implementation Surface: API deprecation scanning tools such as kubent, pluto, kubepug, or kubectl deprecations; manifest and Helm chart scanning for deprecated APIs; CRD deprecated field detection; deprecated API inventory generation; team notification workflows; and migration completion tracking before upgrade approval.
- Owner Primary: Platform Engineering
- Owner Supporting: AI/ML Engineering; DevOps; Security Engineering
