# SAF-K8S-0104-020 - Kubernetes supported version window compliance

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.4 - CIS Benchmarks and Patch Management
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must run a Kubernetes version that remains within the project or provider active support window.

## Objective

Keep clusters inside supported version boundaries so security patches remain available and vendor support is not lost.

## Rationale

Unsupported Kubernetes versions stop receiving security patches and provider support, leaving clusters exposed to known vulnerabilities without an endorsed remediation path. Enforcing supported-version compliance keeps clusters on versions that still receive security fixes and compatible vendor support.

## Scope

Applies to all production clusters. Covers Kubernetes control-plane and worker node version selections, including managed-service versions subject to provider support windows.

## Enforcement

- Primary Enforcement Point: Supported-version governance
- Implementation Surface: Cluster version inventory, support-window tracking, provider lifecycle bulletins, and periodic compliance checks against supported minor versions.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Change Management
