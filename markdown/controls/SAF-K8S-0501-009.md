# SAF-K8S-0501-009 - Model download path isolation from training data paths

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.1 - Network Policies
- Maturity: Foundational
- Classification: ai-specific

## Statement

Model download paths must be isolated from training data paths through distinct network policy and egress controls so that external model retrieval cannot contaminate training data flows.

## Objective

Separate software supply-chain ingress from training data ingress to prevent malicious or unreviewed model artifacts from traversing the same paths as training data sources.

## Rationale

Model download traffic and training data traffic serve different trust domains and attack surfaces. Isolating them reduces the chance that a compromised artifact source, registry, or download mirror can influence or masquerade as a trusted training data path.

## Scope

Applies to AI training and model preparation environments that retrieve external model artifacts, base models, weights, or dependencies while also accessing training data stores.

## Enforcement

- Primary Enforcement Point: Egress and path-isolation controls on AI training environments and artifact retrieval components.
- Implementation Surface: Network policies, egress controls, and path validation between model registries, artifact mirrors, training data stores, and model preparation workloads.
- Owner Primary: Network Engineering
- Owner Supporting: ML Platform Engineering
