# SAF-K8S-0606-032 - GitOps drift detection and automated resync or notification

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

GitOps controllers must detect runtime configuration deviations from Git-declared state and automatically resync or notify on drift in production environments.

## Objective

Detect divergence between declared and running cluster state and ensure unauthorized runtime changes are surfaced and either automatically corrected or escalated through alerting.

## Rationale

If runtime drift is not detected and corrected or escalated promptly, unauthorized or accidental changes can persist in production and silently weaken security controls. This is a separate operational obligation from monitoring controller reconciliation health itself.

## Scope

Applies to production Kubernetes clusters managed through GitOps. Covers runtime drift detection, self-heal or resync behavior, and alerting or notification when deployed state diverges from approved Git state.

## Enforcement

- Primary Enforcement Point: GitOps drift detection settings and remediation or alert workflows.
- Implementation Surface: Argo CD auto-sync and self-heal settings, Flux drift detection behavior, controller-generated drift alerts, automated resync execution, and notification workflows for unauthorized runtime changes.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; SRE / Operations
