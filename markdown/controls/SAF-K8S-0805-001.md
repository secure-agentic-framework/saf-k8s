# SAF-K8S-0805-001 - GPU telemetry collection and anomaly detection

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.5 - GPU Workload Auditing and Monitoring
- Maturity: Practitioner
- Classification: ai-specific

## Statement

All GPU-equipped Kubernetes nodes must run DCGM Exporter or equivalent telemetry agents collecting NVML metrics for GPU utilization, memory usage, temperature, power consumption, ECC error counts, and clock throttling, with anomaly detection rules configured to identify error injection, abnormal utilization patterns, and hardware degradation.

## Objective

Establish comprehensive GPU observability that enables detection of hardware anomalies, unauthorized workloads, and potential security incidents through continuous collection and analysis of GPU-specific telemetry.

## Rationale

GPU workloads lack the observability depth of CPU workloads in standard Kubernetes monitoring. Without GPU-specific telemetry, cryptomining on expensive training GPUs goes undetected, hardware errors silently corrupt training results, thermal throttling degrades model training without alerting, and error injection attacks that manipulate GPU computation are invisible. DCGM and NVML metrics provide the GPU-specific signals necessary for security monitoring that standard Kubernetes metrics do not cover.

## Scope

Applies to all Kubernetes nodes with NVIDIA GPUs or equivalent accelerators. Covers DCGM Exporter DaemonSet deployment, NVML metric collection, Prometheus or equivalent metric storage, and alerting rule configuration. Non-NVIDIA accelerators (AMD, Intel) must use equivalent vendor telemetry tools. Managed GPU services must verify provider telemetry coverage against organizational requirements.

## Enforcement

- Primary Enforcement Point: DCGM Exporter DaemonSet configuration, Prometheus scrape targets and recording rules, alerting rule definitions for GPU anomaly detection, and GPU monitoring dashboard configuration.
- Implementation Surface: DCGM Exporter DaemonSet deployment on GPU nodes, NVML metrics collection (DCGM_FI_DEV_GPU_UTIL, DCGM_FI_DEV_FB_USED, DCGM_FI_DEV_ECC_SBE_VOL_TOTAL, DCGM_FI_DEV_POWER_USAGE, DCGM_FI_DEV_GPU_TEMP, DCGM_FI_DEV_CLOCK_THROTTLE_REASONS), Prometheus scrape targets and recording rules for GPU metrics, Prometheus alerting rules for GPU anomaly detection (utilization spikes, ECC errors, thermal events, power anomalies), GPU monitoring dashboards (Grafana), and per-workload-type baselines for training, inference, and idle patterns.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Observability Engineering; Security Operations; GPU Infrastructure Engineering
