# SAF-K8S-0103-017 - Controller-manager and scheduler insecure-port disablement and non-public health-metrics exposure

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must disable insecure serving ports for the controller-manager and scheduler, and must not expose health or metrics endpoints on network-accessible interfaces unless explicitly required and protected by authentication and approved access controls.

## Objective

Eliminate unauthenticated control-plane exposure by disabling insecure ports and constraining health and metrics endpoints to non-public or explicitly protected access paths.

## Rationale

Historically, controller-manager and scheduler components exposed insecure or weakly protected HTTP endpoints for health checks, metrics, and diagnostics. If these endpoints are reachable on network-accessible interfaces, attackers can harvest operational data, probe control-plane health, or exploit misconfigured monitoring paths. Disabling insecure ports and restricting health or metrics exposure reduces this risk.

## Scope

Applies to all self-managed Kubernetes clusters. Covers deprecated `--port` / `--address` handling, secure health and metrics exposure configuration, and any approved authenticated path used for monitoring integration. For managed Kubernetes services, this is provider-managed and should be documented in the shared responsibility model.

## Enforcement

- Primary Enforcement Point: Controller-manager and scheduler endpoint exposure hardening
- Implementation Surface: Controller-manager and scheduler static pod manifests or systemd unit configurations, metrics and health endpoint exposure settings, CIS Benchmark scanning, and network reachability assessment.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering
