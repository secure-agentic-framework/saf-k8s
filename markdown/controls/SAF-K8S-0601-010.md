# SAF-K8S-0601-010 - Artifact retention period and lifecycle enforcement

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must define and enforce retention policies for container images, Helm charts, and ML model artifacts in registries, including minimum retention periods by artifact type and protection of artifacts referenced by running deployments from garbage collection.

## Objective

Ensure software release artifacts remain available long enough for rollback, forensics, and audit while preventing uncontrolled registry growth.

## Rationale

Without explicit retention periods and deployment-aware garbage-collection protection, organizations can lose the exact artifacts needed to investigate incidents or roll back safely, or accumulate unbounded artifact storage with no lifecycle discipline.

## Scope

All container registries, Helm chart repositories, ML model registries, and OCI artifact stores used by the organization for release artifact storage.

## Enforcement

- Primary Enforcement Point: Registry retention and lifecycle policy enforcement for release artifacts.
- Implementation Surface: Registry lifecycle rules, garbage-collection protections for deployed digests, tag immutability and retention policies, and automated cleanup tooling for container, chart, and model artifacts.
- Owner Primary: Platform Engineering
- Owner Supporting: DevSecOps / Release Engineering; ML Platform Engineering
