# SAF-K8S-0503-006 - Cloud load balancer security group configuration

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.3 - Ingress, Egress, and DNS Hardening
- Maturity: Practitioner
- Classification: baseline

## Statement

Cloud load balancer security groups and firewall rules must be configured to limit source IP ranges to approved networks for all load balancer resources provisioned by Kubernetes Services.

## Objective

Ensure that cloud-provisioned load balancers restrict network-level access to approved source IP ranges, preventing access from unintended network segments even when internal load balancer annotations are correctly applied.

## Rationale

Without cloud security group restrictions, even internal load balancers may be reachable from unintended network segments within the cloud environment. Kubernetes Service manifests control the Kubernetes-level exposure, but cloud-level security groups govern the actual network reachability of the provisioned load balancer resource. Misalignment between Kubernetes configuration and cloud security groups creates gaps in access control.

## Scope

Applies to all cloud-provisioned load balancer resources in production clusters. Security groups must be reviewed for both internet-facing and internal load balancers. Applies to AWS, GCP, and Azure managed Kubernetes environments.

## Enforcement

- Primary Enforcement Point: Cloud provider security groups and firewall rules on load balancer resources.
- Implementation Surface: Cloud security groups and firewall rules on load balancer resources (AWS Security Groups on ELB/NLB, GCP firewall rules on forwarding rules, Azure NSGs on load balancer frontends), infrastructure-as-code templates enforcing security group configuration, and audit scripts validating security group source IP ranges against approved network lists.
- Owner Primary: Cloud Platform Engineering
- Owner Supporting: Network Engineering; Security Engineering
