# SAF-K8S-0801-014 - MPS and time-slicing side-channel risk assessment

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Advanced
- Classification: ai-specific

## Statement

Clusters using NVIDIA MPS or GPU time-slicing must assess side-channel leakage risks for shared GPU deployments and document whether timing, contention, or memory access pattern channels create unacceptable cross-workload exposure.

## Objective

Provide technical assurance that shared GPU deployments have undergone explicit side-channel risk assessment before multi-tenant or mixed-sensitivity use is permitted.

## Rationale

Even when memory remnants are prevented, shared GPU deployments may still expose timing, contention, cache, or memory access pattern channels. This control is separate from remnant prevention because it requires threat-informed assessment of leakage mechanisms that are not resolved solely by clearing memory between workload transitions.

## Scope

Applies to any Kubernetes cluster using NVIDIA MPS or GPU time-slicing. Covers side-channel assessment activities, documented findings, and risk characterization for shared GPU deployments.

## Enforcement

- Primary Enforcement Point: Shared-GPU Side-Channel Assessment
- Implementation Surface: Threat-informed side-channel assessment processes, shared GPU leakage testing, and documented findings or risk determinations for MPS and time-sliced GPU deployments.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Data Protection Office
