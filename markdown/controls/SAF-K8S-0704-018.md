# SAF-K8S-0704-018 - Break-glass cloud identity governance for Kubernetes administration

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.4 - Cloud Provider Security Integration
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must restrict cluster-admin-equivalent or break-glass cloud identities to tightly governed emergency access procedures with explicit approval, control, and review.

## Objective

Ensure emergency cloud identities capable of privileged Kubernetes administration are tightly governed and not available as uncontrolled standing access paths.

## Rationale

Break-glass access remains a high-risk backdoor when permanently available or weakly governed. Emergency cloud identities need explicit governance so they are available when required but do not become a routine or opaque path to cluster-admin-equivalent access.

## Scope

Applies to managed Kubernetes services and any cloud-authenticated break-glass, vendor-admin, or other emergency identities that can obtain cluster-admin-equivalent Kubernetes access.

## Enforcement

- Primary Enforcement Point: Emergency access governance and privileged identity control processes for cloud-to-cluster administration.
- Implementation Surface: Break-glass account procedures, multi-person approval workflows for emergency privileged access, access review evidence, and controls restricting standing availability of cluster-admin-equivalent cloud identities.
- Owner Primary: Cloud Platform Engineering
- Owner Supporting: Security Engineering; Platform Engineering
