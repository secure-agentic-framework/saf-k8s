# SAF-K8S-0703-022 - GPU abuse pattern monitoring and detection for Kubernetes AI workloads

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.3 - Resource Governance and Priority
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must monitor GPU-equipped Kubernetes environments for cryptomining, resource squatting, personal workloads, and other unauthorized GPU usage patterns using telemetry, workload fingerprinting, and anomaly detection rules that surface suspicious non-sanctioned GPU consumption for review.

## Objective

Detect unauthorized GPU consumption patterns early enough to prevent sustained waste, tenant impact, or concealed malicious activity on scarce accelerator resources.

## Rationale

Unauthorized GPU usage usually appears first as anomalous utilization, suspicious workload fingerprints, or repeated non-sanctioned consumption patterns. A dedicated detection control is needed so cryptomining, personal workloads, and abuse of shared accelerator capacity are surfaced before they become entrenched or hide broader compromise.

## Scope

Applies to all GPU-equipped nodes and GPU-consuming workloads in Kubernetes clusters. Monitoring must cover GPU utilization anomalies, workload fingerprinting, and alert generation for suspected non-sanctioned consumption patterns.

## Enforcement

- Primary Enforcement Point: GPU telemetry monitoring, workload fingerprinting, and anomaly detection rules for suspected GPU abuse.
- Implementation Surface: GPU telemetry pipelines, anomaly detection rules, workload fingerprinting logic, cryptomining signatures, and alert generation for suspicious GPU workload behavior.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; ML Engineering
