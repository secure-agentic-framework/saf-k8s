# SAF-K8S-0502-007 - Kubernetes pod IP anti-spoofing enforcement and validation

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.2 - CNI Plugins and Pod Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

IP spoofing protection must be enabled and validated so pods cannot forge source IP addresses to bypass network policies or impersonate other workloads.

## Objective

Prevent network-layer identity spoofing that could circumvent policy enforcement or allow a compromised pod to impersonate another workload.

## Rationale

IP spoofing allows a malicious pod to impersonate another workload's IP address, bypassing NetworkPolicy rules and undermining trust in network-layer identity. Separating anti-spoofing enforcement from IPAM capacity governance makes it possible to audit datapath enforcement and spoofing test validation directly.

## Scope

Applies to all Kubernetes clusters. Covers anti-spoofing enforcement for all CNI configurations and validation on managed and self-managed clusters that forged source IP packets are dropped.

## Enforcement

- Primary Enforcement Point: CNI anti-spoofing enforcement at the pod virtual network interface or datapath.
- Implementation Surface: CNI anti-spoofing enforcement at the veth pair or eBPF datapath, managed or self-managed CNI configuration, and spoofing test validation confirming forged source IP packets are dropped.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Security Operations
