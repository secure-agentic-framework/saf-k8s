# SAF-K8S-0601-015 - Inference image minimal composition with GPU runtime-only dependencies

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Inference-serving images must use distroless or multi-stage build outputs that retain only the runtime libraries required for serving, including GPU and CUDA compatibility libraries when needed, while excluding compilers, package managers, and debugging toolchains.

## Objective

Minimize inference-serving image contents while preserving only the runtime dependencies necessary for model serving and accelerator compatibility.

## Rationale

Inference containers are high-value targets because they hold model-serving code, runtime credentials, and access to accelerators. Multi-stage and distroless outputs limit the software supply-chain and post-exploitation surface while preserving only the libraries required to serve models safely on GPU-capable platforms.

## Scope

Applies to production and pre-production inference-serving images, including GPU-enabled images that require documented CUDA or accelerator runtime libraries.

## Enforcement

- Primary Enforcement Point: Build pipeline controls governing the final inference image contents before registry publication and deployment.
- Implementation Surface: Multi-stage Dockerfile outputs, distroless base images, runtime library chain verification, and build validation proving only required inference and GPU runtime libraries remain.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: ML Platform Engineering; Platform Engineering
