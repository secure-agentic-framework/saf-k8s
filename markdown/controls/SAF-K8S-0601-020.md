# SAF-K8S-0601-020 - Container registry authentication and role-based authorization

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Container registries must enforce authenticated access for all push and pull operations and apply role-based authorization restricting write access to authorized CI/CD service accounts, release engineers, and other approved identities.

## Objective

Prevent unauthorized image upload, overwrite, or pull operations by enforcing authenticated and role-scoped registry access.

## Rationale

Authentication and authorization are independently auditable registry access controls: they ensure only approved human and non-human identities can push, pull, or administer registry content, preventing malicious image upload or tampering by unauthorized actors.

## Scope

Applies to all container registries used for storing production, staging, and development images, including model-serving image registries and OCI artifact stores. Public mirror registries must be proxied through an authenticated pull-through cache.

## Enforcement

- Primary Enforcement Point: Container registry authentication configuration and registry RBAC or IAM access policies.
- Implementation Surface: Container registry authentication settings, registry RBAC policies, robot or service account identities, repository access scope controls, and cloud registry IAM bindings for push and pull operations.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
