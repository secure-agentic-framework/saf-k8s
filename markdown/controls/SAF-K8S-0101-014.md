# SAF-K8S-0101-014 - API server authorization mode baseline enforcement

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must enable Node and RBAC authorization and must not use ABAC or AlwaysAllow in the API server authorization mode chain.

## Objective

Ensure API server authorization decisions follow an auditable, least-privilege baseline that avoids legacy or unsafe authorization modes.

## Rationale

Authorization modes determine how every API request is evaluated. Node and RBAC provide scoped, auditable authorization paths, while ABAC and AlwaysAllow create hard-to-review or outright unsafe authorization behavior that can grant unintended access.

## Scope

Applies to all production clusters and managed Kubernetes services where authorization mode configuration is customer-visible or customer-selectable.

## Enforcement

- Primary Enforcement Point: API server authorization mode configuration and authorization review.
- Implementation Surface: API server authorization mode flags or managed-service settings, removal of ABAC policy files, prohibition of AlwaysAllow, and review of Node plus RBAC authorization configuration.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Identity and Access Management
