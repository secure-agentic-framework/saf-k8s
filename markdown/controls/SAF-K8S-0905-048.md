# SAF-K8S-0905-048 - OCI model artifact digest pinning in deployment and promotion workflows

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Models stored and promoted through OCI registries must be referenced by immutable digests in deployment and promotion workflows so that reviewed model artifacts cannot be silently replaced through mutable tag resolution.

## Objective

Ensure that model promotion and deployment workflows consume digest-pinned OCI artifact references rather than mutable tags that can later resolve to different artifacts.

## Rationale

Mutable tags let a deployment or promotion workflow resolve to a different artifact than the one that was reviewed. Digest pinning binds the workflow to a specific reviewed model artifact and prevents silent substitution during rollout, replication, or rollback.

## Scope

Applies to deployment manifests, promotion metadata, release records, and policy checks that reference OCI-backed model artifacts. Covers digest-pinned references and validation that mutable-tag references are not used for approved model deployments.

## Enforcement

- Primary Enforcement Point: Deployment manifests, promotion metadata, and policy checks that validate model artifact references.
- Implementation Surface: Deployment and promotion manifests referencing models by digest (@sha256:...), CI or admission policy checks rejecting mutable-tag model references, and release records that preserve the reviewed digest for each model version.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; DevOps Engineering
