# SAF-K8S-0902-010 - Infrastructure-layer prompt injection classification and instruction boundary enforcement

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.2 - Inference Server and Model Serving Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

LLM serving infrastructure must classify incoming prompts for prompt injection indicators, enforce system, developer, and user instruction hierarchy, and validate prompt boundary integrity before model execution through controls that operate independently of the model's own safety training.

## Objective

Provide infrastructure-layer defense in depth against prompt injection attacks and prompt-boundary manipulation before user content reaches the model context.

## Rationale

Model-level safety behavior is probabilistic and can be bypassed through prompt manipulation. Infrastructure-layer prompt classifiers and boundary validators operate outside the model context window, making them harder to override with hostile prompts and better suited to enforce system or developer prompt precedence.

## Scope

Applies to LLM inference endpoints deployed on Kubernetes. Covers prompt preprocessing, prompt injection classification, instruction hierarchy enforcement, and system or user prompt boundary validation. Excludes output-side token filtering, which is addressed separately.

## Enforcement

- Primary Enforcement Point: Inference preprocessing middleware or sidecar containers and LLM serving framework role-separation configuration.
- Implementation Surface: Prompt preprocessing sidecars or middleware deploying prompt injection classifiers, serving framework role hierarchy settings, API gateway schema validation for system, developer, and user prompt boundaries, and red team bypass testing for prompt override attempts.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Application Security
