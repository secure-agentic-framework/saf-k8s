# SAF-K8S-0101-015 - API server webhook authorizer endpoint trust controls

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Webhook authorization may be enabled only for approved endpoints that are authenticated, integrity-protected, and reviewed as trusted external authorization dependencies.

## Objective

Ensure any external webhook authorizer used by the API server is treated as a trusted security dependency with authenticated and protected transport.

## Rationale

Webhook authorizers extend API access decisions to external services. If those endpoints are unauthenticated, weakly protected, or poorly governed, they create a hidden authorization bypass or integrity failure in the control plane.

## Scope

Applies to production clusters that enable API server webhook authorization, including managed clusters where webhook authorization backends are customer-managed.

## Enforcement

- Primary Enforcement Point: Webhook authorizer endpoint configuration and trust validation.
- Implementation Surface: Webhook authorizer configuration, endpoint TLS settings, authenticated backend identity, certificate validation, and periodic review of approved webhook authorization dependencies.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Identity and Access Management
