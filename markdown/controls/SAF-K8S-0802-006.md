# SAF-K8S-0802-006 - Device plugin socket directory access restriction and unauthorized socket access monitoring

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.2 - GPU Driver, Library, and Toolkit Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Device plugin Unix domain socket directories must be protected with restrictive filesystem permissions and admission controls, and unauthorized processes or pods must be prevented from mounting or accessing the kubelet device-plugin socket path.

## Objective

Protect the kubelet device-plugin socket directory from unauthorized access so rogue pods or local processes cannot tamper with GPU allocation interfaces or observe plugin communication paths.

## Rationale

The device-plugin socket directory is a trust boundary on GPU nodes. Restrictive permissions and hostPath controls reduce the chance that unapproved workloads can access or interfere with device plugin sockets, while monitoring unauthorized socket access provides evidence that the boundary remains enforced.

## Scope

Applies to all Kubernetes nodes where GPU device plugins are registered with the kubelet. Covers the /var/lib/kubelet/device-plugins directory, device plugin Unix domain sockets, and controls preventing unauthorized hostPath access to that directory.

## Enforcement

- Primary Enforcement Point: Device-Plugin Socket Directory Access Control
- Implementation Surface: Filesystem permissions on /var/lib/kubelet/device-plugins and Unix domain sockets, admission control policies restricting hostPath mounts to the device-plugins directory, and host runtime monitoring for unauthorized socket access or creation.
- Owner Primary: Platform Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
