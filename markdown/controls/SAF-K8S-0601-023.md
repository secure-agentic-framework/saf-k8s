# SAF-K8S-0601-023 - Image pull credential automatic rotation and expiry reduction

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Image pull credentials for authenticated registries must be rotated automatically to reduce the exposure window for leaked or stale registry credentials.

## Objective

Reduce the usable lifetime of leaked registry pull credentials by enforcing automatic rotation and credential refresh workflows.

## Rationale

Credential rotation is a distinct operational obligation from secret distribution. Even correctly stored pull secrets become a long-lived liability if they are not refreshed or expired automatically, especially in shared clusters and long-running workloads.

## Scope

Applies to Kubernetes clusters and CI/CD workflows that use authenticated registries for pulling images. Covers image pull secret storage, synchronization from external secret managers, and automatic credential rotation.

## Enforcement

- Primary Enforcement Point: Image pull credential rotation automation and secret refresh workflows.
- Implementation Surface: Automatic rotation configuration, secret refresh automation, registry credential renewal workflows, and logs or records showing pull secrets are rotated without manual long-lived reuse.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
