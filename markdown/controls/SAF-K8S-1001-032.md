# SAF-K8S-1001-032 - Cluster-wide Kubernetes and AI log source coverage

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must deploy and maintain cluster-wide collection coverage for node-level, container, managed-service, and GPU-specific log sources across Kubernetes environments hosting AI workloads so relevant logs are captured before they are lost from ephemeral execution surfaces.

## Objective

Ensure Kubernetes, managed-service, node, container, and GPU log sources are covered by collection mechanisms across all environments hosting AI workloads.

## Rationale

Distributed Kubernetes and AI platforms emit security-relevant logs from ephemeral pods, nodes, managed control planes, and GPU subsystems. Without comprehensive source coverage, critical evidence can be lost during node recycling, pod eviction, or provider-managed platform transitions before centralized analysis can occur.

## Scope

Applies to production and pre-production Kubernetes clusters hosting AI workloads. Covers node-level log agents, provider-generated cluster log sources, container and kubelet paths, and GPU driver or device log capture coverage. Excludes the downstream centralized aggregation or export of already collected logs into enterprise logging platforms.

## Enforcement

- Primary Enforcement Point: Cluster-wide collector deployment and log source coverage verification.
- Implementation Surface: DaemonSet-based log collectors on all nodes including GPU nodes with taint tolerations, provider-generated cluster log source enablement, container and kubelet log collection paths, and GPU driver or device log capture coverage.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; Infrastructure Engineering
