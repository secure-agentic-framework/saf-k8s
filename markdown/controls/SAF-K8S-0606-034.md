# SAF-K8S-0606-034 - Emergency direct-access audit logging and justification governance

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production clusters managed by GitOps must record auditable break-glass logging and justification for any emergency direct access outside the approved Git deployment path.

## Objective

Ensure exceptional emergency direct access outside GitOps is tightly governed, justified, and fully auditable for review after use.

## Rationale

Even with Git-only deployment enforcement, emergency access paths can be abused or become normal practice if exceptional use is not logged, justified, and reviewed. This is a separate accountability and oversight obligation from prohibiting routine direct changes.

## Scope

Applies to production namespaces and clusters governed through GitOps. Covers break-glass access logging, justification capture, and reviewable records for emergency direct interventions.

## Enforcement

- Primary Enforcement Point: Audit logging and operational governance for emergency direct cluster access.
- Implementation Surface: Audit logs for break-glass direct access, recorded justifications or approvals, and retained review records for emergency kubectl or API operations performed outside the normal GitOps path.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; SRE / Operations
