# SAF-K8S-0402-023 - Long-lived workload token exception governance and retirement tracking

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.2 - Service Accounts and Workload Identity
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations must inventory any remaining legacy, manually created, or extended-lifetime workload service account tokens, drive that population toward zero, and require documented, re-reviewed exceptions for any non-expiring or extended-lifetime token use that remains.

## Objective

Prevent insecure long-lived workload token use from becoming a hidden persistent exception by forcing explicit tracking and governance until it is eliminated.

## Rationale

Time-bounded projected tokens do not remove risk if organizations still tolerate legacy or manually created long-lived workload tokens without active governance. Inventory, trend monitoring, and re-reviewed exception handling prevent those exceptional credentials from becoming permanent blind spots.

## Scope

Applies to any remaining legacy token secrets, manually created long-lived service account tokens, or extended-lifetime workload tokens retained for constrained compatibility scenarios.

## Enforcement

- Primary Enforcement Point: Legacy token discovery, exception review, and retirement governance for long-lived workload credentials.
- Implementation Surface: Legacy token inventory and trend reports, documented long-lived token exceptions, and re-review records showing retirement progress.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Application Development
