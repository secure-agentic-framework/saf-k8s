# SAF-K8S-0201-007 - Node eviction threshold tuning for workload pressure

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.1 - Kubelet and Node Configuration Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes nodes must set hard eviction thresholds that trigger before workload resource exhaustion destabilizes kubelet, the container runtime, or operating system processes, and those thresholds must be tuned for node instance type and workload behavior.

## Objective

Maintain node stability during memory, storage, or other pressure conditions by triggering eviction early enough to preserve kubelet responsiveness and graceful workload shedding.

## Rationale

Reserved resources alone do not protect node stability if eviction thresholds are absent or tuned too late. Correct eviction settings let the kubelet reclaim resources before pressure conditions become cascading node failures or denial-of-service events affecting all workloads on the node.

## Scope

Applies to all production cluster nodes, with particular attention to nodes hosting GPU workloads or AI training jobs that can exhibit sudden memory spikes. Managed Kubernetes services that expose allocatable resource configuration must have these values tuned for the node instance type. Exemptions require documented risk acceptance.

## Enforcement

- Primary Enforcement Point: Node provisioning templates and kubelet configuration files specifying evictionHard and related reclaim thresholds.
- Implementation Surface: KubeletConfiguration evictionHard thresholds, enforceNodeAllocatable behavior, node pressure alerting, memory and ephemeral storage pressure dashboards, and tuning practices validated against workload burst and recovery behavior.
- Owner Primary: Platform Engineering
- Owner Supporting: Workload Teams; Capacity Planning
