# SAF-K8S-1001-016 - Audit policy coverage for AI-specific resource and workflow events

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Production Kubernetes clusters running AI workloads must configure kube-apiserver audit policies that explicitly capture AI-specific CRD operations, training and pipeline workflow changes, GPU-related resource changes, model serving object changes, and checkpoint or snapshot operations relevant to AI workload integrity and accountability.

## Objective

Ensure that API-server audit policies cover the AI-specific resource and workflow events needed to investigate model deployment changes, training job manipulation, GPU allocation abuse, and checkpoint or snapshot handling across Kubernetes AI environments.

## Rationale

Kubernetes audit logging only captures what the audit policy names. AI platforms introduce custom resources, GPU scheduling objects, and workflow operations that are not covered well by generic audit defaults. Without explicit rules for those AI-specific resources, investigators lose visibility into unauthorized model changes, resource hijacking, and workflow tampering.

## Scope

Applies to production Kubernetes clusters hosting AI workloads and to non-production AI environments where security investigation or compliance evidence is required. Managed Kubernetes services remain in scope where customers control workload-level audit coverage decisions.

## Enforcement

- Primary Enforcement Point: API server audit policy rule definitions for AI-specific resources and workflow objects.
- Implementation Surface: Kube-apiserver audit policy YAML covering model serving, training, pipeline, GPU allocation, and checkpoint or snapshot resource operations.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; AI/ML Engineering
