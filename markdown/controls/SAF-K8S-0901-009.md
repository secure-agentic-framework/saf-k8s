# SAF-K8S-0901-009 - Multi-node training worker mutual authentication

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.1 - Distributed Training Workload Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Multi-node distributed training jobs must enforce mutual authentication between worker nodes so only authorized participants can join inter-worker training communication.

## Objective

Prevent rogue worker participation in distributed training by requiring explicit worker identity verification before gradient exchange begins.

## Rationale

Distributed training jobs create ephemeral peer groups that exchange sensitive gradient data and influence model state. Without mutual authentication, a rogue or mis-scheduled worker can join the training fabric, observe data, or inject poisoned updates.

## Scope

Applies to multi-node distributed training workloads such as PyTorchJob, MPIJob, and TFJob that use multiple worker nodes or parameter exchange endpoints.

## Enforcement

- Primary Enforcement Point: Training job definitions and operator controls that provision and validate worker identities for inter-worker communication.
- Implementation Surface: Training job manifests, operator configuration, mTLS identity distribution, worker certificates, and framework settings that require authenticated worker participation.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Network Engineering
