# SAF-K8S-0601-021 - Container registry trusted-source network restriction

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Container registries must enforce network restrictions that limit registry access to trusted sources, approved CIDR ranges, private endpoints, or equivalent managed network paths.

## Objective

Prevent registry access from untrusted networks by enforcing trusted-source network boundaries around container registries.

## Rationale

Even when registry users authenticate correctly, broad network exposure increases the opportunity for credential abuse, scanning, and exfiltration. Network restrictions create a separate boundary that reduces the exposed attack surface for registry services.

## Scope

Applies to all container registries used for storing production, staging, and development images, including model-serving image registries and OCI artifact stores. Public mirror registries must be proxied through an authenticated pull-through cache.

## Enforcement

- Primary Enforcement Point: Registry network policy, private endpoint, firewall, and CIDR restriction configuration.
- Implementation Surface: Cloud registry network settings, private endpoints, firewall rules, trusted CIDR restrictions, VPC endpoint policies, and reachability validation for registry access paths.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
