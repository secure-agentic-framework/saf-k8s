# SAF-K8S-0102-007 - etcd backup storage encryption

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must ensure that etcd backups and snapshot files inherit equivalent encryption protections to the live etcd volumes, with backup storage using separate encryption keys from the primary etcd volume encryption.

## Objective

Prevent exposure of cluster state through etcd backup exfiltration by requiring that backup storage is encrypted with separately managed keys, providing protection even if the primary etcd volume key is compromised.

## Rationale

etcd backups contain the complete cluster state including all Secrets. A backup encrypted only with the source volume key provides no incremental protection if that key is compromised. Requiring separate key management for backup storage ensures a node-level key compromise does not simultaneously expose both live etcd data and backup archives. Backup encryption is a distinct audit obligation from live-volume encryption (SAF-K8S-0102-001).

## Scope

Applies to all etcd snapshot files and backup archives from production clusters, regardless of storage destination. Backup integrity requirements are governed by SAF-K8S-0102-003.

## Enforcement

- Primary Enforcement Point: Backup Storage Configuration
- Implementation Surface: Backup storage encryption configuration and backup encryption key management separate from primary etcd volume keys.
