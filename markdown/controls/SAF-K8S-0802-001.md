# SAF-K8S-0802-001 - GPU driver lifecycle and vulnerability management

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.2 - GPU Driver, Library, and Toolkit Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

GPU drivers must be maintained under a formal vulnerability management program that includes version tracking, compatibility validation against CUDA toolkits and workloads, timely patching of known CVEs, and controlled rollout using driver containers or GPU Operator lifecycle management.

## Objective

Ensure GPU drivers are current, free of known vulnerabilities, and compatible with deployed AI workloads, preventing exploitation of driver-level vulnerabilities that grant kernel-level access on GPU nodes.

## Rationale

GPU drivers operate at the kernel level and vulnerabilities in them can enable local privilege escalation, container escape, or denial of service to all GPU workloads on the node. Driver version incompatibilities with CUDA toolkits can cause silent computation errors in training jobs, potentially corrupting model weights. Without lifecycle management through driver containers or GPU Operator, driver updates require node reboots and create extended vulnerability windows.

## Scope

Applies to all Kubernetes nodes with GPU hardware, whether using host-installed drivers, driver containers, or GPU Operator-managed driver installations. Includes both self-managed and managed Kubernetes GPU node pools. Managed service provider driver management must be validated against organizational patching SLAs.

## Enforcement

- Primary Enforcement Point: GPU Operator driver manager configuration, driver container image registry policies, node vulnerability scanning, and change management workflows for driver upgrades.
- Implementation Surface: GPU driver kernel modules (nvidia.ko, nvidia-uvm.ko), NVIDIA GPU Operator driver container images and lifecycle management configuration, host-installed driver packages and node provisioning templates, NVIDIA GPU driver security bulletin subscriptions, driver-CUDA compatibility matrix tracking, driver container rollout and rollback procedures per node pool, node vulnerability scanning for GPU driver CVEs, and change management workflows for driver upgrades.
- Owner Primary: GPU Infrastructure Engineering
- Owner Supporting: ML Platform Engineering; Vulnerability Management; Security Engineering
