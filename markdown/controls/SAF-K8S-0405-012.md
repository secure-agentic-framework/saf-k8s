# SAF-K8S-0405-012 - Privileged MFA enforcement for cluster administration

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.5 - Identity Abuse Detection and Mitigation
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must enforce multi-factor authentication for all privileged human access to Kubernetes clusters and AI infrastructure, with explicitly governed non-interactive workload exceptions and emergency access dependencies handled through approved break-glass procedures.

## Objective

Ensure privileged administrative access requires a second factor so compromise of a single password or primary credential does not grant cluster-administrative control.

## Rationale

Privileged access without MFA leaves the cluster vulnerable to straightforward credential theft and phishing. MFA for human administrators materially reduces the chance that a single compromised credential yields full control over cluster or AI infrastructure administration.

## Scope

Applies to privileged human access paths for Kubernetes clusters, OIDC provider-backed administration, and cloud-provider administrative integrations. Covers MFA enforcement for administrators and clarified exceptions for non-interactive workload identities.

## Enforcement

- Primary Enforcement Point: MFA enforcement policy in identity providers and privileged access paths for cluster administration.
- Implementation Surface: OIDC provider MFA policies, cloud IAM privileged access requirements, API server authentication integrations, and documented exception handling for non-interactive service identities and break-glass dependencies.
- Owner Primary: Identity and Access Management
- Owner Supporting: Platform Engineering; Security Engineering
