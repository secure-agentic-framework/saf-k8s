# SAF-K8S-0404-007 - Automated certificate rotation before expiry

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.4 - Certificate Management
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must automate certificate rotation before expiry across cluster and workload PKI assets.

## Objective

Prevent certificate-expiry outages by ensuring certificate renewal occurs before certificates expire.

## Rationale

Executing renewal before expiry is independently auditable from the telemetry used to observe upcoming expirations. This control isolates renewal automation so teams can verify that PKI assets are configured to rotate before their notAfter date without manual intervention.

## Scope

Applies to API server, etcd peer, kubelet, webhook, service mesh, and application certificates managed in the cluster. Covers rotation configuration and renew-before settings. Expiry monitoring and alerting remain governed by SAF-K8S-0404-008.

## Enforcement

- Primary Enforcement Point: Certificate renewal configuration for cluster PKI assets.
- Implementation Surface: cert-manager Certificate resources with renewBefore settings, issuer integrations, and automated renewal workflows for cluster and workload certificates.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Security Operations
