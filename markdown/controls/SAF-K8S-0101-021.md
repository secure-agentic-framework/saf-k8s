# SAF-K8S-0101-021 - API server approved admission controller chain configuration

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must restrict the API server admission controller chain to approved components and must document and govern any deviations from the approved admission baseline.

## Objective

Reduce API server policy bypass risk by ensuring only approved admission controllers are enabled and that deviations are explicitly governed.

## Rationale

Unapproved or weakly governed admission controllers can disable, weaken, or bypass policy enforcement at the API server entry point. Constraining the admission chain to approved components preserves centralized enforcement of validation and policy decisions for cluster access and workload admission.

## Scope

Applies to all production Kubernetes clusters and to non-production clusters that host sensitive data, shared workloads, or provide a path to production. For managed Kubernetes services, this control applies to all customer-exposed API server admission controller settings and to compensating controls where provider-managed settings are not directly configurable.

## Enforcement

- Primary Enforcement Point: Admission Control Baseline Enforcement
- Implementation Surface: Cluster provisioning templates, API server configuration management, managed Kubernetes security settings, and periodic configuration assessment.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Cloud Platform Engineering
