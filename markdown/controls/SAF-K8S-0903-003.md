# SAF-K8S-0903-003 - Inference-time resource controls

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.3 - Inference Resilience, Adversarial Defense, and Resource Controls
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Inference workloads must enforce computational complexity bounds per request, input size restrictions, multi-level timeouts (request, session, batch), GPU memory ceilings per inference pod, and overload handling mechanisms including graceful degradation and circuit breakers to prevent Economic Denial of Sustainability (EDoS) attacks.

## Objective

Bound the resource consumption of individual inference requests and overall inference workloads to prevent resource exhaustion, cost overruns, and denial-of-service through computational abuse.

## Rationale

Without per-request computational bounds, an attacker can submit inputs designed to maximize inference time and GPU utilization, exhausting cluster resources. Without GPU memory ceilings, a single pod can consume all available GPU memory and crash co-located workloads. Without circuit breakers and graceful degradation, overload conditions cascade into full service outages. EDoS attacks exploit cloud elasticity to generate unsustainable costs.

## Scope

Applies to all inference pods running on Kubernetes with GPU or accelerator resources. Covers per-request limits, per-pod resource ceilings, and infrastructure-level overload protection. Includes cloud-hosted inference services where EDoS risk applies. Excludes training workloads, which have separate resource controls under KA 9.1.

## Enforcement

- Primary Enforcement Point: Inference server configuration (timeout and input size settings), Kubernetes resource limits and GPU resource quotas, and service mesh circuit breaker configuration.
- Implementation Surface: Inference server configuration (per-request timeouts, max input size, max batch size, max output tokens, max beam width), Kubernetes resource limits and GPU resource quotas (MIG/vGPU partition limits), service mesh circuit breaker configuration (Istio, Envoy), graceful degradation policies (reduced output quality under load), per-request cost estimation middleware, and session-level timeout and request count limits.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Platform Engineering; Security Engineering; FinOps
