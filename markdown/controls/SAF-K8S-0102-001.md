# SAF-K8S-0102-001 - etcd storage-layer disk encryption with externally managed keys

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must protect etcd data at the storage layer using provider-managed or infrastructure-level disk encryption with externally managed keys (KMS, CMK, or equivalent), ensuring that key material is not stored on the same node as the data it protects.

## Objective

Provide a defense-in-depth encryption layer beneath API-resource encryption (SAF-K8S-0101-003) by ensuring that etcd storage volumes are encrypted at the infrastructure level with externally managed, access-controlled keys, protecting against physical media access and volume snapshot theft.

## Rationale

API-resource encryption via EncryptionConfiguration (SAF-K8S-0101-003) protects targeted resource values within etcd, but does not encrypt etcd metadata, non-targeted resources, or WAL and snapshot files at the storage level. Storage-layer encryption, whether provided through cloud disk encryption, dm-crypt or LUKS on self-managed nodes, or KMS envelope encryption, ensures that all etcd data is protected against physical media access, volume snapshot theft, and backup exfiltration. Externally managed keys ensure that key material is not stored alongside the data it protects, reducing the blast radius of a node-level compromise. KMS v2 integration (preferred for new deployments) reduces KMS call volume by using a single DEK per API server instance.

## Scope

Applies to all production Kubernetes clusters and to non-production clusters that store sensitive data or provide a path to production. Covers etcd data directory disk encryption and KMS provider deployment. For managed Kubernetes services, applies to enabling and verifying provider-managed envelope encryption and CMK integration where available. Key rotation is governed by SAF-K8S-0102-006; backup encryption by SAF-K8S-0102-007.

## Enforcement

- Primary Enforcement Point: Node / OS
- Implementation Surface: Infrastructure disk encryption configuration (cloud provider KMS, LUKS, or equivalent), KMS plugin deployment for envelope encryption, backup storage encryption policies, and periodic key rotation verification.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Key Management/Cryptography; Cloud Platform Engineering
