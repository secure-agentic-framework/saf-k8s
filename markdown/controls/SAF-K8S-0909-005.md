# SAF-K8S-0909-005 - RAG prompt injection defense

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.9 - RAG Infrastructure Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

RAG systems must implement defenses against indirect prompt injection, where adversarial instructions embedded in retrieved documents manipulate the LLM's behavior, including input sanitization of retrieved chunks, instruction-data separation in prompt templates, and monitoring for prompt injection patterns in retrieval results.

## Objective

Prevent adversarial instructions embedded in retrieved documents from manipulating LLM behavior through indirect prompt injection attacks.

## Rationale

Indirect prompt injection is uniquely dangerous in RAG systems because the attack payload lives in the knowledge base, not in the user's query. An attacker plants a document containing adversarial instructions in a location that the RAG ingestion pipeline will index. When a user's query retrieves that document, the adversarial instructions are injected into the LLM's context. This attack bypasses user-facing input validation because the malicious content enters through the retrieval path rather than the query path.

## Scope

Applies to all RAG systems that assemble LLM prompts from retrieved content on Kubernetes. Covers retrieved chunk sanitization, prompt template design with instruction-data separation, and monitoring for prompt injection indicators. Excludes direct prompt injection via user queries (see inference request validation in KA 9.2).

## Enforcement

- Primary Enforcement Point: Retrieval-to-prompt assembly middleware, prompt template configuration, and output validation in the RAG serving pipeline.
- Implementation Surface: Retrieval-to-prompt assembly middleware with chunk sanitization rules (escaping instruction overrides, role-playing triggers, delimiter manipulation), prompt template design with structural instruction-data separation and system/user/assistant message roles, model-specific instruction precedence testing and documentation, output validation rules detecting leaked system prompts and policy violations, and monitoring for prompt injection indicators in retrieved chunks (imperative language, system prompt references).
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Application Security
