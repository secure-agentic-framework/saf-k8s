# SAF-K8S-1007-015 - GPU node onboarding security baseline validation gates

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.7 - Cluster Lifecycle and Asset Inventory
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must enforce security validation gates during GPU cluster scaling and node onboarding so new GPU nodes meet required kubelet, container runtime, operating system, and immutable baseline requirements before accepting workloads.

## Objective

Ensure newly added GPU nodes satisfy required security baselines before they are admitted to the trusted compute environment.

## Rationale

GPU scaling can rapidly introduce nodes that are missing foundational hardening controls. Baseline validation gates prevent insecure nodes from joining production AI clusters and reduce the chance that scaling operations bypass standard node-security requirements.

## Scope

Applies to all GPU-capable node onboarding and scale-out operations in Kubernetes clusters hosting AI workloads.

## Enforcement

- Primary Enforcement Point: Node onboarding automation pipeline with baseline validation gates.
- Implementation Surface: Node onboarding security validation for kubelet hardening, container runtime protections, node OS baseline checks, immutable root enforcement where supported, and workload admission readiness.
- Owner Primary: Infrastructure Engineering
- Owner Supporting: Platform Engineering; Security Engineering
