# SAF-K8S-0702-023 - Virtual cluster tenant isolation validation and host-cluster access review

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Advanced
- Classification: baseline

## Statement

Organizations using virtual cluster technologies for tenant isolation must periodically validate tenant isolation through testing and must review tenant access to the host cluster API server to confirm the intended tenant boundary remains enforced.

## Objective

Ensure virtual-cluster-based tenant isolation is periodically verified through testing and host-cluster access review.

## Rationale

Documented guarantees are insufficient unless the tenant boundary is periodically validated in the live environment. Isolation testing and host-cluster access review confirm that virtual cluster separation, tenant privilege boundaries, and host API restrictions remain effective over time.

## Scope

Applies to multi-tenant Kubernetes environments that rely on virtual cluster or equivalent tenant control-plane separation. Validation must cover tenant boundary testing and review of tenant access to host-cluster APIs.

## Enforcement

- Primary Enforcement Point: Periodic isolation validation exercises and RBAC or host-cluster access review for tenant identities.
- Implementation Surface: Penetration tests or tenant-isolation validation exercises, RBAC audits, host-cluster API access reviews, and recurring verification records confirming the tenant boundary still holds.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Cloud Platform Engineering
