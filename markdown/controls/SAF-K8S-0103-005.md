# SAF-K8S-0103-005 - Pod garbage collection threshold configuration

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Foundational
- Classification: ai-specific

## Statement

Production Kubernetes clusters must configure the controller-manager terminated pod garbage collection threshold (--terminated-pod-gc-threshold) to limit the accumulation of terminated pods. This reduces the window during which terminated pod metadata, including environment variable references, volume mounts, and container status, remains queryable via the API server.

## Objective

Minimize the exposure window for sensitive metadata retained in terminated pod objects by ensuring timely garbage collection of completed and failed pods.

## Rationale

Terminated pods persist in the API server until the pod GC controller removes them. These objects retain environment variable references (which may point to Secrets), volume mount specifications, container image references, and exit status details. In AI clusters, large distributed training jobs can create thousands of worker pods per run, and the default threshold of 12500 allows a large backlog of terminated pods to accumulate. A lower threshold reduces the queryable metadata surface and limits the forensic value available to an attacker with read access to the namespace.

## Scope

Applies to all production Kubernetes clusters. Particularly relevant for clusters running batch AI workloads (distributed training, pipeline jobs) that create large numbers of short-lived pods. For managed Kubernetes services, verify whether the provider exposes this setting or applies a default threshold.

## Enforcement

- Primary Enforcement Point: Controller-Manager Configuration
- Implementation Surface: Controller-manager static pod manifest or flag configuration (--terminated-pod-gc-threshold), monitoring for terminated pod accumulation per namespace.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; AI/ML Engineering
