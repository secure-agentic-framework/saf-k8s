# SAF-K8S-0605-001 - OPA/Gatekeeper policies for Kubernetes and AI workloads

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.5 - Admission Control
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Kubernetes clusters must deploy OPA/Gatekeeper with constraint templates and constraints that enforce organizational security policies, including AI-workload-specific admission policies for GPU resource request validation, image provenance verification, and runtime class enforcement for GPU-enabled workloads. Constraint templates must be version-controlled and tested before deployment.

## Objective

Enforce Kubernetes security policies at admission time using OPA/Gatekeeper, including AI-specific constraints that govern GPU allocation, image trust, and runtime class selection for ML workloads.

## Rationale

Without admission-time policy enforcement, workloads can be deployed with excessive GPU resource requests (causing resource exhaustion or cost overruns), from untrusted image sources (enabling supply chain attacks), or with incorrect runtime classes (bypassing GPU isolation or security boundaries). OPA/Gatekeeper provides declarative, auditable policy enforcement that prevents non-compliant workloads from entering the cluster.

## Scope

Applies to all Kubernetes clusters running AI/ML workloads. Constraint templates must cover both general Kubernetes security policies and AI-specific policies. Clusters not running AI workloads may omit GPU-specific constraints. Policy exceptions must be namespaced and documented.

## Enforcement

- Primary Enforcement Point: Kubernetes OPA/Gatekeeper validating admission webhook.
- Implementation Surface: OPA/Gatekeeper deployment on Kubernetes clusters, Rego-based constraint templates and constraint resources for AI-specific policies, GPU resource request validation constraints (nvidia.com/gpu resource limits), image provenance verification constraints requiring cosign signatures for GPU pods, RuntimeClass enforcement constraints for nvidia RuntimeClass on GPU workloads, AI CRD resource limit constraints for TrainingJob GPU quotas, gator test CLI for constraint template testing, and version-controlled constraint template Git repositories deployed via GitOps.
- Owner Primary: Platform Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
