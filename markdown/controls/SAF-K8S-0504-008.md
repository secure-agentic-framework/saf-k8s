# SAF-K8S-0504-008 - L7 service authorization policy enforcement

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.4 - Zero Trust Architecture and Service Mesh
- Maturity: Practitioner
- Classification: baseline

## Statement

Microsegmentation must extend beyond L3/L4 network policies to include L7 authorization that restricts service access based on HTTP method, path, and headers at the application protocol layer.

## Objective

Achieve fine-grained authorization on allowed service connections so authorized network paths cannot be used for unauthorized operations.

## Rationale

L3/L4 network policies only control who can connect, not what operations are performed once connected. Separating L7 authorization from API contract validation makes per-request service authorization independently auditable.

## Scope

Applies to production clusters running a service mesh or CNI with L7 policy support. Priority should be given to services handling sensitive data, admin operations, or cross-trust-boundary communication. L7 policies require application protocol awareness and may not apply to non-HTTP workloads without protocol-specific support.

## Enforcement

- Primary Enforcement Point: Application-layer authorization policy enforcement.
- Implementation Surface: Service mesh L7 authorization policies, envoy or gateway authorization rules, and policy tests verifying blocked HTTP methods, paths, and headers on otherwise allowed network connections.
- Owner Primary: Security Engineering
- Owner Supporting: Network Engineering; Application Development
