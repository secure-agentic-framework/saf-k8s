# SAF-K8S-0702-022 - Virtual cluster isolation guarantee and residual risk documentation

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Advanced
- Classification: baseline

## Statement

Organizations using virtual cluster technologies for tenant isolation must document the isolation guarantees, architectural assumptions, and residual risks of the chosen virtual cluster approach so the tenant boundary is explicitly understood and governed.

## Objective

Ensure the guarantees and limitations of virtual-cluster-based tenant isolation are documented rather than assumed.

## Rationale

Virtual cluster technologies improve tenant isolation, but the strength of that isolation depends on the architecture chosen and the assumptions it makes about host-cluster boundaries, API mediation, and privileged access. Documenting guarantees and residual risks is necessary so operators, auditors, and tenant owners understand what protections exist and what risks remain.

## Scope

Applies to multi-tenant Kubernetes environments that rely on virtual cluster or equivalent tenant control-plane separation. Documentation must cover isolation guarantees, boundary assumptions, and residual risks for the selected approach.

## Enforcement

- Primary Enforcement Point: Residual risk documentation and architecture review for the selected virtual cluster isolation model.
- Implementation Surface: Isolation model documentation, residual risk assessments, architecture decision records, and governance records describing the chosen virtual cluster guarantees and limitations.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Cloud Platform Engineering
