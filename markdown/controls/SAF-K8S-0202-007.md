# SAF-K8S-0202-007 - Container runtime user namespace isolation

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.2 - Container Runtime Security
- Maturity: Foundational
- Classification: baseline

## Statement

Clusters must enable container runtime user namespace isolation on supported nodes and runtimes, and must validate that privileged user IDs inside containers do not map directly to host root unless a documented exception with compensating controls is approved.

## Objective

Reduce the blast radius of container escape paths by isolating container user identities from host user identities at the runtime and kernel namespace boundary.

## Rationale

User namespace isolation prevents UID 0 inside a container from being equivalent to root on the host. This weakens a common post-exploitation path for container escapes and limits the impact of workloads that require elevated in-container privileges.

## Scope

Applies to production node pools and runtime configurations that support user namespace isolation. Unsupported workloads or runtimes must have documented exceptions and compensating controls.

## Enforcement

- Primary Enforcement Point: Container runtime and node configuration governing user namespace remapping or isolation for supported workloads.
- Implementation Surface: Container runtime user namespace configuration, node pool capability validation, workload compatibility testing, and operational exception handling for runtimes or workloads that cannot yet run with user namespaces enabled.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; OS/Infrastructure
