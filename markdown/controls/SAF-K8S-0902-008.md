# SAF-K8S-0902-008 - Inference endpoint authentication and authorization

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.2 - Inference Server and Model Serving Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Inference model endpoints must enforce authentication and authorization via API gateway integration, token validation, and per-model access controls to ensure that only authorized clients can invoke specific models.

## Objective

Prevent unauthorized access to model inference endpoints by enforcing per-model authentication and authorization at the API gateway or ingress layer.

## Rationale

Without endpoint authentication and per-model authorization, any network-reachable client can invoke any model, enabling data exfiltration through model abuse or unauthorized use of high-cost GPU resources.

## Scope

Applies to all inference endpoints exposed on Kubernetes. Covers API gateway and ingress configurations for model endpoints, OAuth/mTLS token validation, and per-model RBAC or token-based access controls. Excludes model loading integrity covered by SAF-K8S-0902-002.

## Enforcement

- Primary Enforcement Point: API gateway policies and ingress controller authentication settings for inference endpoints.
- Implementation Surface: API gateway policies (Kong, Envoy, Istio ingress) for per-model authentication and authorization, OAuth/mTLS token validation configuration, per-model RBAC access control rules, and service mesh AuthorizationPolicy for inference services.
- Owner Primary: API Platform Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
