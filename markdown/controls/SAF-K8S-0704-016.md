# SAF-K8S-0704-016 - Cloud IAM to Kubernetes RBAC mapping review and drift remediation

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.4 - Cloud Provider Security Integration
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must review cloud IAM to Kubernetes RBAC entitlement mappings whenever cloud IAM roles, access entries, or Kubernetes RBAC bindings change, and must remediate drift so intended Kubernetes permissions remain accurate over time.

## Objective

Ensure documented IAM-to-RBAC entitlement mappings remain current as cloud IAM roles, access entries, and Kubernetes RBAC bindings evolve.

## Rationale

A documented mapping becomes stale unless it is reviewed and corrected when either side of the managed Kubernetes authorization model changes. Without ongoing review and remediation, access drift accumulates and cloud identities may retain unintended Kubernetes permissions despite nominal documentation.

## Scope

Applies to managed Kubernetes services and any self-managed clusters using cloud IAM or external cloud identity for Kubernetes authentication. Review must cover changes to cloud IAM roles, access entries, identity integrations, and Kubernetes RBAC bindings that affect granted access.

## Enforcement

- Primary Enforcement Point: Recurring entitlement review workflows and drift-remediation processes.
- Implementation Surface: Review workflows comparing cloud IAM identities, access entries, and Kubernetes RBAC bindings; drift reports; and remediation actions correcting mismatched or stale IAM-to-RBAC mappings.
- Owner Primary: Cloud Platform Engineering
- Owner Supporting: Security Engineering; Platform Engineering
