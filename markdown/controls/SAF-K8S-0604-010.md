# SAF-K8S-0604-010 - Automated AI workload rebuild and redeployment patch pipelines

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.4 - SBOMs and Vulnerability Intelligence
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must maintain automated rebuild and redeployment pipelines that can patch AI workload images and supporting components without requiring model retraining.

## Objective

Enable timely patching of AI workloads without the operational cost and delay of retraining models for routine image and dependency fixes.

## Rationale

If patching an AI workload requires retraining or manual repackaging of model artifacts, teams will defer remediation and accept unnecessary exposure. Automated rebuild and redeployment pipelines break that dependency by separating model weights from patchable runtime images and by validating that patched images can be promoted without retraining the underlying model.

## Scope

Applies to AI and ML workload container images, supporting runtime components, and deployment pipelines in Kubernetes. Covers base image and dependency patch rebuilds, redeployment automation, and validation that patched runtime images do not require model retraining. Does not cover vulnerability classification or SLA prioritization, which are addressed by SAF-K8S-0604-008 and SAF-K8S-0604-009.

## Enforcement

- Primary Enforcement Point: CI/CD rebuild and redeployment pipeline triggers for patched AI workload images.
- Implementation Surface: Automated CI/CD rebuild and redeployment pipelines triggered by base image or dependency patches, model weight loading at runtime from model registries rather than baking models into container images, regression test automation for patched AI workload images, promotion workflows for patched runtime images, and deployment records demonstrating patch rollout without model retraining.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; DevSecOps / Release Engineering
