# SAF-K8S-0403-023 - Credential scope drift monitoring for AI workloads

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations running AI workloads at scale must monitor credential scope drift by comparing granted permissions with actual usage and identifying over-broad or exception-expanded credentials.

## Objective

Provide governance over credential breadth by detecting when issued AI workload credentials exceed their actually used privilege scope.

## Rationale

Scope drift is independently auditable from aggregate lifecycle metrics. This control isolates the requirement to compare granted versus used permissions so teams can prove that broadening credential scope and exceptions are being detected and governed.

## Scope

Applies to all credentials issued to AI and ML workloads including Vault dynamic secrets, cloud IAM temporary credentials, Kubernetes service account tokens, API keys for external services, and model registry authentication tokens. Covers scope drift analysis, exception identification, and governance review of over-broad credentials. Lifecycle metric publication remains governed by SAF-K8S-0403-024.

## Enforcement

- Primary Enforcement Point: Credential scope drift analysis and over-broad permission review workflow.
- Implementation Surface: Granted-versus-used permission analysis, exception review queues, IAM policy usage data, and governance review records for over-scoped workload credentials.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; ML Engineering; Cloud Platform Engineering
