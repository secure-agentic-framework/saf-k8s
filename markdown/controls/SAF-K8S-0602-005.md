# SAF-K8S-0602-005 - Notary v2 OCI signature artifact registry integration

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.2 - Image Signing and Admission Enforcement
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations using Notary v2 (notation) must maintain registry integration that stores notation signatures as OCI-compliant artifacts alongside their corresponding image manifests.

## Objective

Ensure notation signatures remain discoverable and synchronized with their corresponding registry artifacts through OCI-compliant storage integration.

## Rationale

Registry integration for notation signatures is distinct from trust policy governance and admission verification. If signatures are stored outside OCI artifact flows or drift from image manifests, verification becomes brittle and signature discovery can fail across registries and toolchains.

## Scope

Applies to all container registries and Kubernetes clusters where Notary v2 is the designated or supplementary signing mechanism. Includes environments using Azure Container Registry, AWS ECR, or other registries with native notation support. Organizations exclusively using Sigstore/cosign may document this control as not applicable.

## Enforcement

- Primary Enforcement Point: Registry-level signature artifact storage configuration and notation registry integration.
- Implementation Surface: OCI artifact signature storage, registry integration for notation signatures in Azure Container Registry, AWS ECR, Harbor, or equivalent registries, and notation push or discovery workflows tied to image manifests.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; DevSecOps / Release Engineering
