# SAF-K8S-0702-030 - Promotion-time automatic classification uplift and reclassification execution

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must implement automated reclassification during model promotion so classification metadata is adjusted when the target environment or promoted artifact requires a higher or different protection level than the source environment.

## Objective

Ensure model promotion events trigger the correct automated classification uplift or reclassification actions before deployment.

## Rationale

Promotion into a more sensitive environment can change the required classification level of an artifact. Automated reclassification is a distinct operational obligation from simply checking whether metadata exists, because the system must actively compute and apply the correct higher classification when promotion context changes.

## Scope

Applies to CI/CD and model promotion pipelines where artifacts move into environments with different sensitivity or governance requirements. Covers automatic classification uplift and reclassification event recording during promotion.

## Enforcement

- Primary Enforcement Point: Promotion pipeline reclassification logic and event recording.
- Implementation Surface: CI/CD promotion pipelines with automated reclassification hooks, environment-aware classification uplift logic, and reclassification event logs showing original classification, applied classification, environment, and promotion timestamp.
- Owner Primary: ML Engineering
- Owner Supporting: Data Governance; Platform Engineering; Security Engineering
