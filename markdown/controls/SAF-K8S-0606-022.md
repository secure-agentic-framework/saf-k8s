# SAF-K8S-0606-022 - Admission-time verification of Kubernetes manifest signatures

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Advanced
- Classification: baseline

## Statement

Admission control must verify Kubernetes manifest signatures before cluster acceptance and reject manifests that do not present a valid signature from an authorized signing identity.

## Objective

Ensure only manifests with valid approved signatures can be admitted to the cluster.

## Rationale

Signing alone does not stop deployment of unsigned or improperly signed manifests unless the cluster enforces verification at admission. Admission-time verification is independently auditable because it governs cluster acceptance behavior, rejection logic, and authorized signing trust roots.

## Scope

Applies to production and staging Kubernetes clusters that admit manifests through direct apply, GitOps reconciliation, or deployment automation. Covers signature trust roots, admission policies, and rejection behavior.

## Enforcement

- Primary Enforcement Point: Admission controller verification policy and cluster rejection logic.
- Implementation Surface: Admission controller policy verifying manifest signatures, configured trust roots or authorized signing identities, and validation records showing unsigned or invalidly signed manifests are rejected.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Platform Engineering; Security Engineering
