# SAF-K8S-0702-019 - Supplementary namespace isolation control enforcement for multi-tenant AI clusters

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must implement and periodically validate supplementary controls, including network policies, namespace-scoped RBAC, admission controls, and isolation testing, to achieve the required isolation level between AI and non-AI workloads in namespace-based multi-tenant clusters.

## Objective

Enforce and validate the supplementary technical controls required to make namespace-based tenant isolation effective in practice.

## Rationale

A documented namespace model is insufficient unless supporting controls actually enforce the intended boundary. Without network isolation, RBAC scoping, admission restrictions, and periodic isolation validation, namespaces become a weak logical grouping rather than an effective tenant boundary. This enforcement obligation is distinct from selecting and documenting the boundary model itself.

## Scope

Applies to all multi-tenant Kubernetes clusters where namespaces are used as tenant boundaries. Covers the supporting network, RBAC, admission, and validation controls required to enforce the intended isolation level between AI and non-AI workloads.

## Enforcement

- Primary Enforcement Point: NetworkPolicy, namespace-scoped RBAC, admission policies, and isolation validation procedures.
- Implementation Surface: Default-deny inter-namespace traffic rules, namespace-scoped RBAC bindings, admission controls blocking disallowed cross-namespace references, periodic isolation tests, and other supplementary controls that make namespace-based multi-tenancy effective.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; ML Engineering
