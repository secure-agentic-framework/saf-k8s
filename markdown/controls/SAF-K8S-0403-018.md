# SAF-K8S-0403-018 - AI platform cryptographic key access domain separation

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations operating AI platforms must enforce key access policies that separate training, serving, signing, and governance identities so that workload or operator access in one cryptographic domain does not grant cross-domain use of other keys.

## Objective

Prevent cryptographic privilege crossover between training, serving, signing, and governance environments by enforcing domain-separated key access policies.

## Rationale

A sound key hierarchy is insufficient if the same workloads or operators can still use every key. Domain-separated access policies ensure that training systems cannot sign production models, serving systems cannot decrypt training datasets, and governance identities cannot be bypassed through shared cryptographic permissions.

## Scope

Applies to KMS and HSM key policies for AI platform cryptographic operations across training, serving, model signing, checkpoint handling, and governance environments.

## Enforcement

- Primary Enforcement Point: KMS key policies, workload identity bindings, and audit enforcement for domain-separated key use.
- Implementation Surface: KMS key access policies, IAM bindings, workload identity permissions, and audit logs showing key usage by workload identity without cross-domain access.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; ML Engineering; Key Management/Cryptography
