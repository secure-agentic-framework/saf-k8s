# SAF-K8S-0304-013 - Tmpfs-backed handling for sensitive training intermediates

- Domain: D03 - Workload and Pod Security
- Knowledge Area: 3.4 - Secure Defaults and Resource Constraints
- Maturity: Foundational
- Classification: baseline

## Statement

Sensitive intermediate training data that does not require durable persistence must use tmpfs-backed emptyDir or equivalent memory-backed ephemeral storage so those artifacts are not written to node-local disk and are removed with pod lifecycle termination.

## Objective

Reduce exposure of sensitive temporary training data by keeping qualifying intermediate artifacts in memory-backed ephemeral storage rather than on node-local disks.

## Rationale

Sensitive training intermediates can contain fragments of training data, embeddings, or model state that should not persist on node-local SSD or host filesystems after job completion. tmpfs-backed storage limits that exposure window by keeping qualifying data in memory and removing it with pod termination.

## Scope

Applies to AI and ML training workloads that stage sensitive intermediate data, tokenized samples, gradients, or temporary outputs that do not require durable persistence.

## Enforcement

- Primary Enforcement Point: Pod specification validation for emptyDir.medium=Memory and workload design reviews for sensitive intermediate-data handling.
- Implementation Surface: Pod spec emptyDir medium selections and workload manifests showing which sensitive intermediate datasets use memory-backed scratch space.
- Owner Primary: AI/ML Platform team
- Owner Supporting: Platform Engineering; Security Engineering
