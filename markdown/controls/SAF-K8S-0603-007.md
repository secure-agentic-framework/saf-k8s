# SAF-K8S-0603-007 - Build environment and process attestations per NIST SP 800-204D

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.3 - Attestation, Provenance, and Cryptographic Assurance
- Maturity: Advanced
- Classification: baseline

## Statement

Build pipelines must generate independently signed and verifiable environment attestations documenting build platform inventory and process attestations recording build tools and transformations applied, in alignment with NIST SP 800-204D.

## Objective

Ensure the build environment and build-step execution path are independently attested so compromised runners or unauthorized build-step changes are detectable.

## Rationale

Environment and process attestations expose compromise in build infrastructure and unauthorized pipeline modifications that a single monolithic attestation would obscure. Separating these dimensions makes the build platform and transformation chain independently verifiable.

## Scope

Applies to all CI/CD pipelines that produce container images or deployable artifacts for production clusters, including managed build services that must provide equivalent environment and process provenance.

## Enforcement

- Primary Enforcement Point: CI/CD attestation generation for build platform inventory and transformation steps.
- Implementation Surface: Environment attestation for runner image, node, and platform inventory; process attestation for build tools, Dockerfile transformations, and pipeline stages; per-category signatures; and verification of environment and process attestation presence.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Platform Engineering
