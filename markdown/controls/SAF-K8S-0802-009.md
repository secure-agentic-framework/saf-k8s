# SAF-K8S-0802-009 - GPU driver binary path file integrity monitoring

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.2 - GPU Driver, Library, and Toolkit Security
- Maturity: Advanced
- Classification: ai-specific

## Statement

GPU driver binary and shared library paths must be monitored with file integrity tools to detect unauthorized replacement, tampering, or drift from approved baselines.

## Objective

Detect unauthorized replacement or tampering of GPU driver binaries and shared libraries on the filesystem before compromised driver code is used by workloads.

## Rationale

Driver binary replacement on the filesystem can subvert GPU workloads even when kernel module trust controls are in place. File integrity monitoring is a distinct detection obligation because it validates the integrity of driver binary paths and shared libraries linked by workloads after installation.

## Scope

Applies to all Kubernetes GPU nodes running NVIDIA driver shared libraries and binaries. Covers approved GPU driver binary paths, baseline establishment, and monitoring for unauthorized modification or replacement.

## Enforcement

- Primary Enforcement Point: Driver Binary Path Integrity Monitoring
- Implementation Surface: File integrity monitoring tools on GPU driver binary paths and shared libraries, approved baseline capture for libnvidia paths, and alerting on unauthorized replacement or tampering.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; GPU Infrastructure Engineering
