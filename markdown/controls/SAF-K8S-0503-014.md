# SAF-K8S-0503-014 - Ingress rate limiting and abuse throttling

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.3 - Ingress, Egress, and DNS Hardening
- Maturity: Practitioner
- Classification: baseline

## Statement

Ingress controllers protecting internet-facing services must apply rate limiting controls appropriate to the exposed service so backend systems are protected from abuse, credential stuffing, and application-layer denial-of-service attacks.

## Objective

Throttle abusive ingress traffic before it can overwhelm backend services or enable high-volume online attacks.

## Rationale

Request filtering and rate limiting address different attack modes. Separating abuse throttling from WAF integration makes capacity protection and request-shaping controls independently auditable.

## Scope

Applies to internet-facing ingress controllers and to internal ingress paths where risk assessment requires request filtering or abuse throttling. Covers WAF integrations, request filtering rules, and per-route or per-client rate limiting.

## Enforcement

- Primary Enforcement Point: Ingress abuse throttling controls.
- Implementation Surface: Ingress annotations, CRDs, or gateway policies implementing per-route or per-client rate limiting, plus telemetry showing abuse throttling triggers and enforcement.
- Owner Primary: Network Engineering
- Owner Supporting: Application Security; Platform Engineering
