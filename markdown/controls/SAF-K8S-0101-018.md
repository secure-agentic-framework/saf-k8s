# SAF-K8S-0101-018 - API server anonymous authentication disablement

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must disable anonymous authentication on the API server.

## Objective

Prevent unauthenticated access attempts from reaching the Kubernetes API server as anonymous requests.

## Rationale

Anonymous API access bypasses the identity controls that protect the control plane and can expose discovery paths or weakly guarded endpoints to unauthenticated callers. Disabling anonymous authentication forces all API access through authenticated identities.

## Scope

Applies to all production Kubernetes clusters and to non-production clusters that host sensitive data, shared workloads, or provide a path to production. For managed services, applies to all customer-exposed API server authentication settings and compensating controls when provider-managed settings are fixed.

## Enforcement

- Primary Enforcement Point: API server authentication configuration.
- Implementation Surface: API server manifest flags, managed Kubernetes security settings, cluster provisioning templates, and periodic configuration assessment for anonymous authentication status.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Cloud Platform Engineering
