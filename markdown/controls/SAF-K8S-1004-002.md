# SAF-K8S-1004-002 - Software supply chain threat model per NIST SP 800-204D

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.4 - AI and Supply Chain Threat Taxonomy
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must apply the NIST SP 800-204D software supply chain threat model to their Kubernetes CI/CD and AI pipeline infrastructure, analyzing the three-stage SSC attack lifecycle (Compromise, Propagation, Exploitation), distinguishing SSC defects from SSC attacks, identifying attack targets (source code, credentials, build systems, internal operations) and attack vectors (dependency confusion, typosquatting, malicious PRs, compromised build tools), and must implement controls at each stage to disrupt the attack lifecycle.

## Objective

Systematically identify and mitigate software supply chain threats to Kubernetes and AI pipeline infrastructure using the NIST SP 800-204D framework, preventing adversaries from compromising the build and deployment chain of AI workloads.

## Rationale

AI pipelines depend on complex software supply chains, including ML frameworks, model repositories, training data sources, and container images, that are vulnerable to supply chain attacks at each stage. Without a structured threat model distinguishing defects from deliberate attacks and mapping specific vectors to Kubernetes CI/CD components, organizations cannot prioritize controls for the highest-risk supply chain entry points such as compromised model registries or poisoned base images.

## Scope

Applies to all CI/CD pipelines, container image build systems, model registries, and artifact repositories supporting Kubernetes AI workloads. Covers source code management, dependency resolution, build environments, and artifact distribution channels. Must be reviewed when pipeline architecture or dependency sources change.

## Enforcement

- Primary Enforcement Point: CI/CD pipeline security configuration, supply chain security review process, and artifact provenance verification gates.
- Implementation Surface: NIST SP 800-204D software supply chain threat model documentation, three-stage SSC attack lifecycle analysis (Compromise, Propagation, Exploitation), CI/CD pipeline infrastructure security controls (source code repositories, build systems, artifact registries), SSC attack vector identification (dependency confusion, typosquatting, malicious PRs, compromised build tools), SSC defect vs. SSC attack classification, Kubernetes CI/CD and AI pipeline infrastructure analysis (model training pipelines, container image builds, model registry operations), and disruption controls at each lifecycle stage.
- Owner Primary: Security Architecture
- Owner Supporting: DevOps/Platform Engineering; AI/ML Engineering; Supply Chain Security
