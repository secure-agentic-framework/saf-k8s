# SAF-K8S-0405-010 - API impersonation audit logging and alerting

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.5 - Identity Abuse Detection and Mitigation
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must audit Kubernetes API impersonation events through API server audit logging, capture sufficient event detail to identify both the impersonating subject and impersonated identity, and alert on unexpected or unapproved impersonation usage.

## Objective

Ensure legitimate impersonation activity is fully traceable and that suspicious impersonation attempts are rapidly detected.

## Rationale

Even approved impersonation is high risk because it enables actions under another identity. Without audit detail and alerting, misuse can blend into normal API traffic and forensic attribution becomes difficult. Monitoring and alerting are distinct from RBAC restriction and must stand on their own as an auditable obligation.

## Scope

Applies to all clusters where impersonation could be attempted or legitimately used. Covers API server audit policy, log retention for impersonation events, and SIEM or detection rules for unexpected impersonation activity.

## Enforcement

- Primary Enforcement Point: Audit / Telemetry
- Implementation Surface: API server audit policy, audit log pipeline, and SIEM detections keyed to impersonation event fields.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Security Operations
