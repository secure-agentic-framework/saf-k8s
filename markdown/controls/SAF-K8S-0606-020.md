# SAF-K8S-0606-020 - Helm values override restriction and dependency integrity governance

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Helm chart deployments must restrict override of security-critical values, pin and verify chart dependencies with integrity metadata, and enforce trusted repository or mirror governance for dependency retrieval and chart consumption.

## Objective

Prevent security regression through unsafe values overrides and unpinned or silently substituted Helm dependencies.

## Rationale

Even when a chart is authentic, unsafe values injection can disable security settings or redirect image sources, and unpinned dependencies allow malicious or unexpected sub-chart substitution. Dependency locking, integrity verification, and override governance are independently auditable obligations separate from chart provenance validation.

## Scope

Applies to all Helm-based deployment workflows in staging and production Kubernetes clusters, including internal charts, mirrored third-party charts, and AI platform dependency trees.

## Enforcement

- Primary Enforcement Point: Chart dependency resolution, values schema enforcement, and repository / mirror governance in CI/CD and GitOps.
- Implementation Surface: Chart.lock files with pinned dependencies and digests, dependency verification steps, values schema or policy controls blocking security-critical overrides, and trusted repository or mirror configuration for approved dependency retrieval.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Platform Engineering; Security Engineering
