# SAF-K8S-0704-017 - Least-privilege cloud-to-cluster privileged access boundary enforcement

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.4 - Cloud Provider Security Integration
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must enforce least-privilege cloud IAM roles for Kubernetes access and must prevent cloud IAM identities from resolving to broader Kubernetes permissions than intended by the RBAC design.

## Objective

Prevent privilege escalation through cloud IAM identities that resolve to excessive Kubernetes permissions in managed Kubernetes environments.

## Rationale

Even with a documented IAM-to-RBAC map, security fails if cloud identities still resolve to broader Kubernetes privileges than intended. Restricting privileged cloud-to-cluster access paths closes the gap between documented intent and actual effective privilege.

## Scope

Applies to managed Kubernetes services and any cloud-authenticated administrative or automation identities that can obtain privileged Kubernetes access. Includes identities mapped to system:masters, cluster-admin, or equivalent broad authorization paths outside governed emergency procedures.

## Enforcement

- Primary Enforcement Point: Cloud IAM role design and privileged access boundary controls for Kubernetes administration.
- Implementation Surface: Cloud IAM least-privilege role definitions, audits of system:masters or cluster-admin-equivalent mappings, and remediation evidence showing privileged cloud identities do not exceed intended Kubernetes RBAC boundaries.
- Owner Primary: Cloud Platform Engineering
- Owner Supporting: Security Engineering; Platform Engineering
