# SAF-K8S-0403-011 - Secrets KMS key rotation and re-encryption verification

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes Secrets encryption keys must rotate on a defined schedule, and periodic re-encryption verification must confirm that no Secrets remain encrypted under retired keys.

## Objective

Ensure long-lived KMS keys protecting Kubernetes Secrets are rotated and that existing Secrets are re-encrypted so retired key material does not remain in use.

## Rationale

Using an external KMS without key lifecycle enforcement leaves old keys in service indefinitely. This control isolates rotation and re-encryption verification so teams can independently prove that key retirement actually propagates to stored Secrets.

## Scope

Applies to all production Kubernetes clusters using KMS-backed Secrets encryption. Covers automated key rotation schedules, retired-key re-encryption checks, and verification that existing Secrets are protected under current keys.

## Enforcement

- Primary Enforcement Point: KMS rotation automation and re-encryption verification.
- Implementation Surface: Automated key rotation schedules, KMS metadata, and re-encryption verification tooling or procedures confirming current-key coverage for stored Secrets.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Key Management/Cryptography; Cloud Platform Engineering
