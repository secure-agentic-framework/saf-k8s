# SAF-K8S-0905-005 - Automated model promotion gates

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Models must pass automated security scan, bias evaluation, performance threshold, and adversarial robustness gates before production deployment. Gate results must be recorded as attestation evidence linked to the model artifact.

## Objective

Prevent deployment of models that fail security, fairness, or robustness requirements through mandatory automated gate checks.

## Rationale

Without mandatory promotion gates, models with known vulnerabilities, biases, or adversarial weaknesses can reach production. Automated gates provide consistent, repeatable enforcement that does not depend on manual review completeness.

## Scope

Applies to all model promotion workflows on Kubernetes. Covers automated pre-deployment gate checks including security scanning, bias evaluation, performance threshold validation, and adversarial robustness testing. Excludes manual approval workflows (SAF-K8S-0905-010) and canary/A-B deployment controls (SAF-K8S-0905-011).

## Enforcement

- Primary Enforcement Point: Model promotion pipeline gate configurations.
- Implementation Surface: Model promotion pipeline gate configurations (security scan, bias evaluation, performance threshold, adversarial robustness via ART/Foolbox), signature verification gate, and gate result attestation recording.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; ML Engineering
