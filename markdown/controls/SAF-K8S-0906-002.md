# SAF-K8S-0906-002 - Large-scale data integrity verification

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.6 - Training Data Integrity and Poisoning Defense
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Large training datasets must be protected using hierarchical hashing, manifest-based integrity verification, and segmented integrity checking across distributed storage, with resilience strategies (erasure coding or mirroring) selected based on ML data availability and integrity requirements.

## Objective

Detect tampering or corruption in large training datasets efficiently through scalable integrity verification mechanisms that work across distributed storage systems.

## Rationale

Training datasets can be terabytes to petabytes in size, stored across distributed file systems and object stores. Flat hashing of entire datasets is computationally infeasible for verification. Without hierarchical and segmented integrity checking, small targeted modifications (e.g., poisoning a subset of training images) go undetected. Without appropriate resilience strategies, storage failures that corrupt training data are not recoverable, and the wrong resilience choice can trade integrity for availability or vice versa.

## Scope

Applies to all training datasets stored on persistent volumes, distributed file systems, or object storage on Kubernetes that exceed single-node verification capacity. Covers hierarchical hashing, manifest generation, segmented verification, and storage resilience configuration. Excludes small datasets that can be verified with flat hashing.

## Enforcement

- Primary Enforcement Point: Data storage configuration, dataset manifest generation tooling, and integrity verification job configurations in data pipeline orchestrators.
- Implementation Surface: Hierarchical hashing with Merkle tree construction over dataset segments, signed dataset manifests listing segment hashes, sizes, and locations, segmented integrity verification jobs in data pipeline orchestrators, storage resilience configuration (erasure coding, mirroring, object versioning) on distributed file systems and object storage, and incremental verification for changed segments.
- Owner Primary: Data Engineering
- Owner Supporting: Storage Engineering; ML Platform Engineering
