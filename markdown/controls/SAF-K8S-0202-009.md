# SAF-K8S-0202-009 - Container runtime secure baseline settings and debug endpoint disablement

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.2 - Container Runtime Security
- Maturity: Foundational
- Classification: baseline

## Statement

Container runtimes must be configured according to an approved security baseline that enforces secure default settings and disables unnecessary features such as deprecated API versions or debug endpoints.

## Objective

Minimize container runtime attack surface through approved baseline configuration settings.

## Rationale

Even when socket access is restricted, insecure runtime defaults and exposed debug or deprecated interfaces expand the runtime attack surface. An approved runtime baseline with unnecessary features disabled is a distinct hardening obligation from filesystem protection of the socket itself.

## Scope

Applies to all nodes running containerd or CRI-O in production clusters. Managed Kubernetes services where the runtime is provider-managed satisfy this control through provider security documentation and compensating network controls. Custom or alternative CRI implementations must demonstrate equivalent hardening.

## Enforcement

- Primary Enforcement Point: Node image build pipeline and container runtime configuration files.
- Implementation Surface: containerd config.toml, CRI-O crio.conf, debug endpoint settings, deprecated API toggles, and approved secure default runtime options.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; OS/Infrastructure
