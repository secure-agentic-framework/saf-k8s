# SAF-K8S-0503-015 - Approved DNS resolution path enforcement

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.3 - Ingress, Egress, and DNS Hardening
- Maturity: Practitioner
- Classification: baseline

## Statement

DNS policy enforcement must prevent workloads from using unauthorized DNS resolution paths so name resolution cannot bypass approved cluster DNS controls or external DNS policy boundaries.

## Objective

Constrain how workloads resolve names so DNS cannot be used to bypass network controls or approved resolver paths.

## Rationale

Even a hardened DNS infrastructure becomes a policy-bypass channel if workloads can resolve names through unauthorized paths. Separating resolution-path enforcement from service-discovery scoping makes each control independently auditable.

## Scope

Applies to DNS egress restrictions, CNI DNS-aware policies, namespace-scoped service discovery controls, and any external DNS proxy or enforcement layer used by production workloads.

## Enforcement

- Primary Enforcement Point: DNS resolution path enforcement controls.
- Implementation Surface: DNS-aware network policies, egress restrictions to CoreDNS or approved DNS proxies, and verification that workloads cannot resolve names through unauthorized DNS paths.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering
