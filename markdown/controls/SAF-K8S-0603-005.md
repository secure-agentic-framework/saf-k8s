# SAF-K8S-0603-005 - FIPS 140 cryptographic module validation

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.3 - Attestation, Provenance, and Cryptographic Assurance
- Maturity: Practitioner
- Classification: baseline

## Statement

All cryptographic modules used for artifact signing and signature verification operations must be validated to FIPS 140 requirements where mandated by organizational policy or regulatory obligation. Signing infrastructure must use FIPS-validated cryptographic libraries, and key storage must meet the applicable FIPS 140 security level for the sensitivity of the artifacts being signed.

## Objective

Ensure that cryptographic operations underpinning supply chain integrity use validated modules that meet recognized assurance standards, preventing reliance on unvetted or weak cryptographic implementations.

## Rationale

Supply chain signing and verification depend on the correctness and strength of the underlying cryptographic implementation. An unvalidated module may contain implementation flaws (side-channel leaks, weak random number generation) that compromise key material or signature integrity. FIPS 140 validation provides third-party assurance that the module meets minimum security requirements, which is also a prerequisite for many government and regulated-industry deployment environments.

## Scope

Applies to all signing and verification operations for container images, attestations, SBOMs, model artifacts, and Helm charts. Includes HSM-backed signing, software-based signing libraries, and verification libraries in admission controllers. Organizations not subject to FIPS requirements must document their cryptographic module assurance strategy.

## Enforcement

- Primary Enforcement Point: Signing infrastructure provisioning and cryptographic library selection during build system configuration.
- Implementation Surface: FIPS 140-validated cryptographic libraries (go-fips compiled cosign and notation), HSM and cloud KMS services validated to FIPS 140-2 Level 2 or higher (AWS CloudHSM, Azure Managed HSM, Google Cloud HSM), FIPS-approved algorithms for key generation, signing, and verification operations, signing infrastructure provisioning configuration, and admission controller verification libraries compiled against FIPS-validated modules.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Compliance / GRC
