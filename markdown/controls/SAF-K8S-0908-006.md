# SAF-K8S-0908-006 - Secure aggregation for privacy-preserving model outputs

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.8 - Model Abuse and Extraction Prevention
- Maturity: Advanced
- Classification: ai-specific

## Statement

Inference endpoints must implement secure aggregation protocols for privacy-preserving model outputs when serving aggregate queries.

## Objective

Prevent decomposition of aggregate query results into individual-level information through privacy-preserving aggregation protocols at the inference serving boundary.

## Rationale

Without secure aggregation, aggregate queries can be decomposed to reveal individual-level information, enabling reconstruction of individual data points from batch inference responses. This control complements differential privacy and output perturbation (SAF-K8S-0908-004) by specifically protecting the aggregation pathway, ensuring that adversaries cannot reconstruct individual data points by analyzing aggregate query responses. See SAF-K8S-0908-001 for rate limiting controls that reduce the number of queries an adversary can issue.

## Scope

Applies to inference endpoints on Kubernetes serving models trained on sensitive data where aggregate or batch query privacy guarantees are required. Covers secure aggregation configuration for federated and batch inference workflows. Excludes individual (non-aggregate) inference serving, which is addressed in SAF-K8S-0908-004.

## Enforcement

- Primary Enforcement Point: Secure aggregation service configuration and aggregate query routing middleware.
- Implementation Surface: Secure aggregation service configuration for aggregate or federated query workflows, per-query aggregation parameter validation, batch size enforcement for privacy guarantees, and monitoring of aggregation pathway integrity.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Privacy Engineering
