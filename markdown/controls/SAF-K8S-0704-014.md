# SAF-K8S-0704-014 - Managed Kubernetes audit log retention enforcement

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.4 - Cloud Provider Security Integration
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must retain cloud provider audit logs and Kubernetes audit events according to policy and verify that retention settings on the common monitoring platform meet organizational and regulatory requirements.

## Objective

Ensure centralized managed Kubernetes audit telemetry remains available for required operational, investigative, and compliance time horizons.

## Rationale

Audit telemetry loses operational and forensic value when centralized ingestion is not paired with enforced retention. Without retention governance, key cloud and Kubernetes audit records may age out before investigations, incident response, or compliance reviews require them.

## Scope

Applies to all managed Kubernetes clusters on cloud providers including EKS, GKE, and AKS. Retention requirements apply to centralized cloud and Kubernetes audit records stored in the common security monitoring platform or any approved long-term archive used by that platform.

## Enforcement

- Primary Enforcement Point: Retention configuration and verification for centralized cloud and Kubernetes audit telemetry.
- Implementation Surface: Retention controls on SIEM or log analytics platforms, approved archival tiers for audit telemetry, and verification workflows confirming cloud and Kubernetes audit records remain available for required policy periods.
- Owner Primary: Security Engineering
- Owner Supporting: Cloud Platform Engineering; Platform Engineering; SOC
