# SAF-K8S-0503-007 - Ingress TLS termination and boundary configuration hardening

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.3 - Ingress, Egress, and DNS Hardening
- Maturity: Practitioner
- Classification: baseline

## Statement

Ingress controllers must enforce TLS termination with organization-approved cipher suites and minimum TLS version for externally facing endpoints, using approved certificate management processes and ingress boundary configuration that prevents weak or unintended exposure.

## Objective

Ensure encrypted and intentionally configured ingress paths protect backend services from downgrade, interception, and accidental insecure exposure.

## Rationale

Ingress controllers are the main external boundary into cluster-hosted services. Weak TLS settings or poorly governed ingress boundary configuration expose traffic to interception, downgrade, and misconfiguration risk before any application-layer filtering takes effect. This is a distinct enforcement point from WAF and rate-limiting policy.

## Scope

Applies to all ingress controllers in production clusters, including NGINX Ingress, Envoy, Traefik, and cloud-managed ALB/NLB ingress controllers. Covers TLS termination settings, approved certificate management, and secure ingress boundary configuration for externally exposed endpoints.

## Enforcement

- Primary Enforcement Point: Ingress Boundary / TLS
- Implementation Surface: Ingress controller TLS configuration, boundary exposure settings, and cert-manager or equivalent certificate management integrations.
- Owner Primary: Network Engineering
- Owner Supporting: Application Security; Platform Engineering
