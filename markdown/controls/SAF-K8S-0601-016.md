# SAF-K8S-0601-016 - Approved minimal base image catalog enforcement

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

All container images deployed to production clusters must be built from approved minimal base images selected from an authorized base image catalog or allowlist.

## Objective

Reduce container attack surface by enforcing use of approved minimal base images for production workloads.

## Rationale

Controlling which base images may be used is an independent supply-chain control claim: approved minimal bases reduce inherited packages, libraries, and utilities before build customization occurs and create a governed boundary for image provenance and review.

## Scope

Applies to all container images built or deployed in production and pre-production Kubernetes clusters, regardless of workload type.

## Enforcement

- Primary Enforcement Point: Approved base image catalogs, allowlists, and CI/CD image selection policy enforcement.
- Implementation Surface: Dockerfiles, OCI build configurations, base image allowlists, admission policies, and CI/CD checks that validate approved minimal base image usage.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering
