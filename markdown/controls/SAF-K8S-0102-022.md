# SAF-K8S-0102-022 - etcd backup repository least-privilege access restriction

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must restrict read and write access to etcd backup repositories and restore media to dedicated backup automation identities and explicitly approved operational roles only.

## Objective

Ensure etcd backups do not become a broadly accessible copy of full cluster state by enforcing least-privilege access to backup repositories and restore media.

## Rationale

Even when backups are encrypted, excessive repository access expands the attack surface and weakens separation of duties. Restricting routine access to dedicated backup automation identities and approved operational roles reduces the risk that backup repositories become a secondary path to full cluster-state exposure.

## Scope

Applies to all production clusters. Covers object storage policies, IAM roles, and restore media handling controls for etcd backup repositories.

## Enforcement

- Primary Enforcement Point: Backup repository authorization
- Implementation Surface: Backup storage IAM policies, bucket or repository access policies, dedicated backup automation identities, and restore media handling restrictions.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Backup and Recovery Operations
