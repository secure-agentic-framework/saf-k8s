# SAF-K8S-0905-043 - Durable ML-BOM attachment to model artifacts and versions

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must attach ML-BOM records to production-bound model artifacts and model versions in a durable, reviewable form so lineage metadata remains linked to the corresponding model throughout storage, distribution, and promotion workflows.

## Objective

Preserve ML-BOM usability by ensuring lineage metadata remains durably linked to the exact model artifact or version it describes.

## Rationale

Generating ML-BOM metadata is insufficient if it is stored separately or lost during promotion and distribution. Durable attachment ensures downstream operators can retrieve the correct lineage record for the exact model version under review.

## Scope

Applies to production-bound model artifacts generated or promoted through Kubernetes-connected AI delivery workflows. Covers durable attachment of ML-BOM records to model artifacts, registries, or version records. Excludes the upstream generation of ML-BOM content itself.

## Enforcement

- Primary Enforcement Point: Artifact publication and registry linkage of ML-BOM records to model versions.
- Implementation Surface: Registry attachment records, OCI or artifact metadata linkage, version records, and preservation of ML-BOM references through promotion workflows.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; DevOps Engineering
