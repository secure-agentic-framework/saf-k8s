# SAF-K8S-0102-006 - etcd encryption key rotation scheduling and verification

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must enforce automated rotation of etcd encryption keys within organization-defined intervals and verify successful rotation through key version audit logs.

## Objective

Ensure that etcd encryption keys are rotated on a scheduled basis and that rotation events are logged and verifiable, limiting the exposure window of any single compromised key.

## Rationale

Long-lived encryption keys increase the blast radius of a key compromise. Automated rotation within defined intervals limits the time window during which a compromised key can decrypt etcd data. Most cloud KMS implementations support transparent rotation with no operational downtime. Audit logs of rotation events provide evidence for compliance and incident response.

## Scope

Applies to all production Kubernetes clusters using KMS-managed etcd encryption keys (SAF-K8S-0102-001). Covers external KMS key rotation policies and verification that rotation occurred within the policy interval.

## Enforcement

- Implementation Surface: KMS key rotation policy configuration and key version audit log review process.
