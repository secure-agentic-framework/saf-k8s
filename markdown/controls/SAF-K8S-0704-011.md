# SAF-K8S-0704-011 - Network-level blocking of cloud metadata endpoint access for pods

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.4 - Cloud Provider Security Integration
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations must implement network-level restrictions blocking pod egress to the cloud metadata service endpoint by default, with documented and justified exceptions granted only to infrastructure pods that require metadata access.

## Objective

Prevent pods from reaching the cloud metadata service at the network layer so compromised workloads cannot retrieve node-scoped cloud credentials through metadata access paths.

## Rationale

Even with IMDSv2 or equivalent session enforcement on nodes, a pod that can still reach the metadata endpoint may be able to obtain short-lived session tokens or otherwise abuse metadata paths. Network-level blocking eliminates this residual exposure for workloads that do not require metadata access and provides defense in depth independent of node launch configuration.

## Scope

Applies to all pods and namespaces on cloud providers that expose a metadata endpoint. Covers default-deny metadata egress controls, justified infrastructure exceptions, and validation that unauthorized workloads cannot reach the metadata service.

## Enforcement

- Primary Enforcement Point: Pod Egress Controls for Cloud Metadata Access
- Implementation Surface: Kubernetes NetworkPolicy, node-level packet filtering where needed, namespace policy rollout, and metadata endpoint connectivity validation for pods.
- Owner Primary: Cloud Platform Engineering
- Owner Supporting: Platform Engineering; Security Engineering
