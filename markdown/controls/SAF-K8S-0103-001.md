# SAF-K8S-0103-001 - Controller-manager service account token hardening

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must configure the controller-manager to issue bound, time-limited service account tokens and disable legacy token auto-provisioning (--controllers=-serviceaccount-token or equivalent). Feature gate governance is addressed in SAF-K8S-0104-004.

## Objective

Reduce the privilege and persistence of service account credentials and limit controller-manager behavior to well-understood, security-reviewed feature sets by enforcing bound tokens, disabling legacy provisioning, and governing feature gates.

## Rationale

Legacy service account tokens are long-lived, non-audience-bound, and automatically mounted into every pod, providing persistent credentials that an attacker can exfiltrate and reuse from any compromised workload. Bound service account tokens are time-limited, audience-scoped, and tied to the pod lifecycle, significantly reducing credential theft impact.

## Scope

Applies to all production clusters. Covers controller-manager flags for service account token issuance and legacy token controller configuration. For managed Kubernetes services, applies to customer-configurable controller-manager settings and to verification that provider defaults align with bound token requirements.

## Enforcement

- Primary Enforcement Point: Controller-Manager Configuration
- Implementation Surface: Controller-manager manifest or managed-service configuration, feature gate policy documentation, and periodic configuration assessment.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Application Platform teams
