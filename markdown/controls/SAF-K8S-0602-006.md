# SAF-K8S-0602-006 - Fail-closed admission enforcement of image signature verification

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.2 - Image Signing and Admission Enforcement
- Maturity: Practitioner
- Classification: baseline

## Statement

Kubernetes clusters must deploy an admission controller configured to reject any container image that lacks a valid cryptographic signature from an authorized signing identity, and the admission policy must fail closed when verification is unavailable.

## Objective

Enforce image signature verification at the Kubernetes admission layer so unsigned or tampered images cannot run even during admission component failure conditions.

## Rationale

Mandatory admission-time verification is a distinct control claim from exception governance. Without fail-closed enforcement, a webhook outage or policy-controller error silently degrades image integrity protection and allows unsigned images to deploy.

## Scope

Applies to all production and staging Kubernetes clusters. All namespaces must be covered unless explicitly exempted in a documented policy (e.g., kube-system for provider-managed components). Emergency break-glass procedures must be audited and time-limited.

## Enforcement

- Primary Enforcement Point: Kubernetes validating admission webhook and image verification policy configuration.
- Implementation Surface: ValidatingWebhookConfiguration with failurePolicy: Fail, admission controller deployments, signature verification rules, verifyImages or equivalent policy configuration, and audit logs for rejected unsigned images.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
