# SAF-K8S-0606-025 - Artifact freshness limit enforcement for CI/CD promotion

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

CI pipelines must enforce artifact freshness limits so artifacts exceeding the organization-defined maximum age from committed code cannot be promoted or deployed.

## Objective

Reduce reliance on stale build outputs by enforcing build-horizon limits as a promotion prerequisite.

## Rationale

An artifact that passed security checks days or weeks ago may no longer reflect current vulnerability, dependency, or configuration risk. Build-horizon enforcement is independently auditable because it measures artifact age against policy and blocks stale deployments even when the repository itself remains acceptable.

## Scope

Applies to production deployment pipelines and promoted artifacts derived from source repositories. Covers build timestamp validation, commit age comparison, and deployment-time freshness gates.

## Enforcement

- Primary Enforcement Point: Artifact promotion gate and deployment approval logic.
- Implementation Surface: Build timestamp validation, artifact age calculation relative to committed code, and promotion or deployment controls that reject stale artifacts exceeding policy thresholds.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Development Teams
