# SAF-K8S-0203-011 - Immutable node root filesystem enforcement

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.3 - Host OS and Kernel Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes node operating systems must enforce immutable root filesystems where supported by the platform so persistent host-level modifications are prevented or made directly detectable through image drift.

## Objective

Prevent persistent host compromise by enforcing immutable node root filesystems where the platform supports it.

## Rationale

Immutable root filesystems prevent persistent rootkits and unauthorized host modifications from surviving reboot or image replacement. This is a distinct obligation from minimizing installed software because it governs host mutability and drift resistance after deployment.

## Scope

Applies to all production cluster node OS images. Cloud-managed node pools must use the provider's hardened or minimal node image options. Custom node images must document their base image, removed packages, and hardening steps. Development clusters may use standard OS images with documented exceptions.

## Enforcement

- Primary Enforcement Point: Node OS image selection, platform immutability settings, and validation of immutable root filesystem behavior on supported node pools.
- Implementation Surface: Immutable OS distributions, root filesystem immutability settings, dm-verity or equivalent platform enforcement, and validation of host drift resistance.
- Owner Primary: Platform Engineering
- Owner Supporting: OS/Infrastructure; Security Engineering
