# SAF-K8S-0502-001 - CNI plugin security selection criteria

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.2 - CNI Plugins and Pod Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

CNI plugin selection must be based on documented evaluation of security features including network policy support completeness, pod-to-pod encryption capabilities, FQDN-based filtering, L7 policy enforcement, and performance characteristics, with the selected plugin meeting all organization-defined minimum security requirements.

## Objective

Ensure the cluster networking foundation supports the full range of required security controls by selecting a CNI plugin with adequate security feature coverage.

## Rationale

The CNI plugin is the foundational network layer for all pod communication. Selecting a plugin without adequate network policy support, encryption capability, or performance under security-enabled configurations undermines all higher-layer network security controls. A CNI plugin that cannot enforce egress FQDN filtering or L7 policies forces the organization to rely on weaker compensating controls.

## Scope

Applies to all Kubernetes cluster deployments, including new cluster provisioning and CNI migration decisions. Managed Kubernetes services with limited CNI choice must document the available security features and any gaps requiring compensating controls.

## Enforcement

- Primary Enforcement Point: Cluster provisioning standards and architecture review process governing CNI plugin selection and configuration.
- Implementation Surface: CNI plugin evaluation matrix comparing Cilium, Calico, Flannel, and Canal across security features (NetworkPolicy support, WireGuard/IPsec encryption, FQDN filtering, L7 policy, eBPF dataplane, flow observability), architecture decision records, cluster provisioning standards, managed Kubernetes CNI documentation (AWS VPC CNI, GKE Dataplane V2, Azure NPM), and gap analysis with compensating controls for managed environments with limited CNI choice.
- Owner Primary: Platform Engineering
- Owner Supporting: Network Engineering; Security Architecture
