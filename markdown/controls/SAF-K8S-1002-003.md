# SAF-K8S-1002-003 - AI workload telemetry integration into cluster monitoring

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.2 - Monitoring, Metrics, and Tracing
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must integrate AI workload telemetry, including GPU metrics via DCGM exporter, training job progress, inference latency, and model performance indicators, into the cluster monitoring stack.

## Objective

Ensure that AI-specific telemetry is available in the cluster monitoring stack for performance monitoring and anomaly detection.

## Rationale

Without integrated AI workload telemetry, operators lack visibility into GPU health, training progress, and inference degradation, delaying detection of cryptomining, model theft via side-channel analysis, or adversarial input attacks.

## Scope

Applies to all Kubernetes clusters hosting AI workloads. Covers GPU metrics collection, training job telemetry, inference serving metrics, and model performance indicators. Includes DCGM exporter deployment and AI workload metric endpoint integration.

## Enforcement

- Primary Enforcement Point: DCGM exporter deployment manifests and AI workload metric endpoint configuration.
- Implementation Surface: DCGM exporter DaemonSet deployment on GPU nodes with taint tolerations and minimal RBAC, AI workload telemetry endpoints from inference serving frameworks (Triton, vLLM, TorchServe native metrics), training job progress metrics (epoch completion, loss convergence), and Prometheus ServiceMonitor or PodMonitor configuration for AI workload metric scraping.
- Owner Primary: AI/ML Engineering
- Owner Supporting: Platform Engineering; Security Operations
