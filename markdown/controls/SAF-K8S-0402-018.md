# SAF-K8S-0402-018 - Default service account disablement and token automount hardening

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.2 - Service Accounts and Workload Identity
- Maturity: Foundational
- Classification: baseline

## Statement

The default service account in each namespace must not be used by application workloads, and automatic mounting of tokens on the default service account must be disabled unless a documented system exception is approved.

## Objective

Remove the shared default service account as a broadly reusable workload credential and eliminate automatic token exposure through default service account configuration.

## Rationale

The default service account creates a shared identity surface across pods in a namespace. Disabling its routine use and turning off token auto-mounting reduces opportunistic API credential exposure and limits common lateral-movement paths after pod compromise.

## Scope

Applies to all application namespaces in production and pre-production clusters. System namespaces may retain documented exceptions only for platform-managed components.

## Enforcement

- Primary Enforcement Point: Namespace service account configuration and admission controls that reject application workloads using the default service account or inheriting default token mounts.
- Implementation Surface: Default ServiceAccount automount configuration, Pod admission policies, namespace bootstrap configuration, and workload validation rejecting implicit default service account usage.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Application Development
