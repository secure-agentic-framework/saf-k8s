# SAF-K8S-0103-012 - Cloud controller-manager cloud IAM least-privilege scoping

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters using a cloud controller-manager must grant only the minimum cloud IAM permissions required for cloud provider operations performed by the cloud controller-manager.

## Objective

Reduce cloud-environment blast radius by ensuring the cloud controller-manager operates with least-privilege cloud permissions scoped only to required cloud APIs and infrastructure actions.

## Rationale

The cloud controller-manager can manipulate high-impact infrastructure such as load balancers, routes, and volumes. Excessive IAM permissions allow a Kubernetes control-plane compromise to expand into the broader cloud account, so permission scope must be independently auditable from credential lifecycle controls.

## Scope

Applies to production clusters on cloud providers with cloud controller-manager integrations. Covers cloud IAM policy scope, allowed cloud APIs, resource scoping, and periodic review of granted permissions.

## Enforcement

- Primary Enforcement Point: Cloud IAM Authorization Scope
- Implementation Surface: Cloud IAM policy definitions, provider-specific role bindings, scoped service permissions for load balancer, route, and volume operations, and IAM access review evidence.
- Owner Primary: Platform Engineering
- Owner Supporting: Cloud Platform Engineering; Security Engineering
