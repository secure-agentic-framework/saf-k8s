# SAF-K8S-0101-020 - API server profiling and debug exposure disablement

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must disable unnecessary API server profiling and debug exposure on the API server or restrict those interfaces through approved compensating controls where direct disablement is not available.

## Objective

Reduce API server reconnaissance and denial-of-service risk by removing exposed profiling and debug interfaces from the control plane attack surface.

## Rationale

Profiling and debug interfaces can leak CPU profiles, heap data, goroutine traces, and other runtime details that aid reconnaissance or can be abused to trigger expensive diagnostic operations. Disabling or tightly restricting these interfaces reduces information disclosure and denial-of-service risk on the API server.

## Scope

Applies to all production Kubernetes clusters and to non-production clusters that host sensitive data, shared workloads, or provide a path to production. For managed Kubernetes services, this control applies to all customer-exposed API server profiling or debug settings and to compensating controls where provider-managed settings are not directly configurable.

## Enforcement

- Primary Enforcement Point: API Server Surface Reduction
- Implementation Surface: Cluster provisioning templates, API server configuration management, managed Kubernetes security settings, and periodic configuration assessment.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Cloud Platform Engineering
