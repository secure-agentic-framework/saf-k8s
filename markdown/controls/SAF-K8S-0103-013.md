# SAF-K8S-0103-013 - Cloud controller-manager workload identity and credential rotation

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters using a cloud controller-manager must enforce workload identity or equivalent federated credential use, or automated rotation and short-lived issuance, for all cloud credentials used by the cloud controller-manager.

## Objective

Reduce cloud-environment blast radius by ensuring the cloud controller-manager does not rely on long-lived static cloud credentials and instead uses federated identity, short-lived credentials, or automated rotation.

## Rationale

Long-lived cloud credentials create a persistent pivot point from cluster compromise into the cloud account. Separating workload identity and credential rotation from IAM scope makes it possible to audit whether the controller uses non-persistent credentials even when the granted permissions are already minimal.

## Scope

Applies to production clusters on cloud providers with cloud controller-manager integrations. Covers workload identity or equivalent federation, short-lived credential issuance, automated credential rotation, and supporting review records.

## Enforcement

- Primary Enforcement Point: Cloud Credential Lifecycle
- Implementation Surface: Workload identity or equivalent credential federation configuration, cloud controller-manager credential rotation automation, short-lived credential issuance settings, and credential lifecycle review evidence.
- Owner Primary: Platform Engineering
- Owner Supporting: Cloud Platform Engineering; Security Engineering
