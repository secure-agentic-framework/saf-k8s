# SAF-K8S-0502-002 - Pod-to-pod traffic encryption

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.2 - CNI Plugins and Pod Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Pod-to-pod network traffic must be encrypted in transit using WireGuard, IPsec, or equivalent transparent encryption provided by the CNI plugin, ensuring that all inter-pod communication is protected against eavesdropping and tampering without requiring application-level changes.

## Objective

Protect the confidentiality and integrity of data in transit between pods by encrypting all pod-to-pod communication at the network layer.

## Rationale

Pod-to-pod traffic within a Kubernetes cluster traverses shared node networks and potentially physical or virtual switches accessible to other tenants or compromised nodes. Without encryption, an attacker with network access (via a compromised node, misconfigured network tap, or cloud provider vulnerability) can eavesdrop on inter-pod communication including API tokens, database queries, and sensitive application data.

## Scope

Applies to all production Kubernetes clusters. Transparent encryption must cover all pod-to-pod traffic across nodes. Intra-node pod traffic encryption is recommended where the CNI supports it. Performance impact must be evaluated and documented, particularly for high-throughput workloads.

## Enforcement

- Primary Enforcement Point: CNI plugin encryption configuration (Cilium WireGuard, Calico WireGuard or IPsec), validated via network traffic capture and encryption verification.
- Implementation Surface: CNI plugin encryption configuration (Cilium Helm values encryption.enabled/encryption.type: wireguard, Calico felixconfiguration wireguardEnabled patch via calicoctl), WireGuard kernel module, IPsec configuration, cilium encrypt status verification, tcpdump packet capture for encryption validation, and performance benchmarking tooling comparing encrypted vs. unencrypted throughput.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Security Engineering
