# SAF-K8S-0909-013 - Approved source repository access control and allowlist enforcement for RAG ingestion

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.9 - RAG Infrastructure Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Document ingestion pipelines for RAG must restrict ingestion to approved source repositories and enforce access controls and allowlists that prevent unapproved repositories, publishers, or paths from being read into the ingestion workflow.

## Objective

Prevent unauthorized or out-of-scope content sources from entering the RAG ingestion pipeline by enforcing approved-source access boundaries.

## Rationale

RAG knowledge-base poisoning often begins before document validation, when ingestion workers can read from repositories or paths that were never approved for inclusion. Approved-source access boundaries and allowlists create the admission perimeter for ingestion before any document-level validation occurs.

## Scope

Applies to document ingestion pipelines for RAG systems on Kubernetes. Covers source repository authentication, read-scope restriction, approved-source allowlists, and ingestion service-account permissions. Excludes document-level provenance and integrity validation after a source has been admitted.

## Enforcement

- Primary Enforcement Point: Ingestion pipeline repository access control and approved-source allowlist enforcement.
- Implementation Surface: Ingestion pipeline service-account permissions, source repository authentication, repository or path allowlists, and controls restricting ingestion workers to approved document sources only.
- Owner Primary: Data Engineering
- Owner Supporting: Security Engineering; ML Platform Engineering
