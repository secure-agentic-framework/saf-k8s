# SAF-K8S-0602-001 - Sigstore/cosign keyless signing and Rekor transparency logging

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.2 - Image Signing and Admission Enforcement
- Maturity: Practitioner
- Classification: baseline

## Statement

All container images promoted to production must be signed using Sigstore/cosign keyless signing with identity tokens issued by Fulcio, and all signatures must be recorded in the Rekor transparency log. Verification workflows must validate both the signature and the transparency log entry before image deployment.

## Objective

Establish cryptographic proof of image provenance and integrity using keyless signing infrastructure, enabling auditable verification without long-lived signing key management.

## Rationale

Without cryptographic signatures, there is no mechanism to verify that a container image was produced by an authorized build pipeline and has not been tampered with in transit or at rest. Keyless signing via Fulcio eliminates the operational burden and risk of long-lived signing keys, while Rekor transparency log entries provide an immutable audit trail that enables detection of unauthorized signing events.

## Scope

Applies to all container images destined for production and staging clusters. Includes model-serving images, operator images, and sidecar images. Development-only images in isolated namespaces may be exempted with documented approval.

## Enforcement

- Primary Enforcement Point: CI/CD pipeline post-build signing step and admission controller signature verification at deployment time.
- Implementation Surface: Sigstore/cosign CLI and CI/CD pipeline integration (cosign sign, cosign verify), Fulcio short-lived signing certificate issuance via OIDC identity tokens (GitHub Actions, GitLab CI, organizational IdP), Rekor transparency log entries and inclusion proofs, Sigstore policy-controller for Kubernetes admission, cosign-installer GitHub Action, and OIDC subject/issuer verification configuration in admission policies.
- Owner Primary: Security Engineering
- Owner Supporting: DevSecOps / Release Engineering; Platform Engineering
