# SAF-K8S-0601-014 - Container image runtime hardening with non-root and read-only filesystem

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Container images must run as non-root users, mount filesystems as read-only where operationally feasible, and exclude shell binaries and unnecessary utilities that expand post-compromise capabilities.

## Objective

Reduce the blast radius of container compromise by enforcing hardened runtime defaults and minimizing readily usable attack tooling inside deployed images.

## Rationale

Running containers as root or shipping writable filesystems and shell tooling gives attackers immediate execution and persistence advantages after compromise. Hardened runtime defaults constrain what an attacker can do even when a workload is successfully exploited.

## Scope

Applies to production and pre-production container images deployed in Kubernetes clusters, with documented and time-limited exceptions only for approved debugging workflows outside production.

## Enforcement

- Primary Enforcement Point: CI/CD image validation and admission controls enforcing non-root, read-only filesystem, and minimal-runtime constraints.
- Implementation Surface: Dockerfile USER directives, pod security contexts, admission policies, image inspection, and build validation for removal of shells and unnecessary utilities.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: ML Platform Engineering; Platform Engineering
