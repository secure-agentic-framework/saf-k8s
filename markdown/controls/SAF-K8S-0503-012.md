# SAF-K8S-0503-012 - Client source IP preservation for external services

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.3 - Ingress, Egress, and DNS Hardening
- Maturity: Practitioner
- Classification: baseline

## Statement

Services exposed to external traffic must preserve client source IP addresses where client IP-based access controls, geographic restrictions, audit logging, or forensic analysis require origin visibility.

## Objective

Preserve client IP visibility for external services where security controls or investigations depend on accurate request origin identity.

## Rationale

When source IP is hidden behind SNAT or load-balancer rewriting, teams lose the origin information needed for IP-based access control, geo restrictions, and forensic attribution. Separating preservation from routing mode governance makes source identity handling independently auditable.

## Scope

Applies to all Services of type LoadBalancer or NodePort in production clusters that receive external traffic. The choice between Local and Cluster externalTrafficPolicy must be documented per service with security and availability justification.

## Enforcement

- Primary Enforcement Point: External client source identity preservation controls.
- Implementation Surface: Cloud load balancer proxy protocol configuration, X-Forwarded-For or equivalent header preservation settings, service exposure configuration, and validation that client source IP remains visible to applications or policy engines where required.
- Owner Primary: Network Engineering
- Owner Supporting: Application Development; Security Engineering
