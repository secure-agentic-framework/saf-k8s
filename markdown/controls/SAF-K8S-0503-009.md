# SAF-K8S-0503-009 - CoreDNS and upstream resolver hardening

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.3 - Ingress, Egress, and DNS Hardening
- Maturity: Practitioner
- Classification: baseline

## Statement

CoreDNS must be configured with hardened settings that restrict upstream resolver use to approved destinations, apply secure caching and forwarding behavior, and limit DNS infrastructure changes to authorized configuration paths.

## Objective

Secure the cluster DNS infrastructure itself so CoreDNS and related resolver components cannot be abused as a weak or untrusted control plane dependency.

## Rationale

CoreDNS is a critical shared service in every Kubernetes cluster. Weak forwarding, caching, or upstream resolver configuration can expose workloads to malicious resolution behavior, redirection, or unauthorized infrastructure changes. Hardening the DNS infrastructure is a distinct obligation from controlling which DNS paths and service names workloads may use.

## Scope

Applies to CoreDNS deployments, NodeLocal DNSCache, and any approved upstream resolver integrations in production clusters.

## Enforcement

- Primary Enforcement Point: DNS Infrastructure
- Implementation Surface: CoreDNS Corefile configuration, NodeLocal DNSCache configuration, and approved upstream resolver settings.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering
