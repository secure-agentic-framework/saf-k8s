# SAF-K8S-1001-026 - Durable audit backend delivery for Kubernetes AI workloads

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Production Kubernetes clusters running AI workloads must deliver audit records to a durable backend so captured audit events remain available for investigation, compliance, and recovery even when local components fail or restart.

## Objective

Ensure that captured audit records are durably delivered to backend storage or downstream logging infrastructure after collection.

## Rationale

Audit policy coverage provides little value if collected records never reach durable storage or downstream analysis systems. Durable backend delivery ensures that audit data survives local restarts, buffer loss, and node or control-plane disruption long enough to support detection and investigation.

## Scope

Applies to audit record delivery paths and backend targets for Kubernetes clusters hosting AI workloads, including webhook targets, log file destinations, centralized logging platforms, and archival pipelines. Covers reliable delivery of audit events to durable backends. Excludes retention immutability and deletion resistance on the backend after arrival.

## Enforcement

- Primary Enforcement Point: Audit backend configuration and audit record delivery path health.
- Implementation Surface: Kube-apiserver audit webhook configuration, audit log path configuration, durable external storage or SIEM delivery targets, fallback delivery design, and health validation that audit records reach configured backends.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; GRC/Compliance
