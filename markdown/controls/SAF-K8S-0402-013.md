# SAF-K8S-0402-013 - Ephemeral training job credential rotation

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.2 - Service Accounts and Workload Identity
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Credentials issued to ephemeral training jobs with expected durations exceeding the credential validity period must be automatically rotated before expiration.

## Objective

Ensure long-running training jobs maintain valid credentials without requiring excessively long-lived tokens, reducing the exposure window while sustaining job continuity.

## Rationale

Long-running training jobs may outlast short-lived credential TTLs. Without automatic rotation, jobs either fail on credential expiry or require long-lived tokens that create unacceptable exposure windows. Automatic rotation maintains access while constraining each individual token lifetime.

## Scope

Applies to training jobs with expected durations exceeding the initial credential validity period, covering Kubernetes projected token refresh, Vault lease renewal, and cloud credential refresh mechanisms.

## Enforcement

- Primary Enforcement Point: Job controller credential refresh configuration, Vault lease renewal policies, and cloud IAM token refresh mechanisms for long-running workloads.
- Implementation Surface: Kubernetes projected ServiceAccountToken automatic refresh, Vault agent credential renewal, Kubeflow TrainingJob and Argo Workflow credential rotation hooks, and cloud SDK automatic token refresh configuration.
- Owner Primary: ML Engineering
- Owner Supporting: Platform Engineering; Security Engineering
