# SAF-K8S-0102-025 - etcd backup repository access review and alerting

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must review etcd backup repository access records or generate alerts for unauthorized or anomalous backup access activity.

## Objective

Ensure etcd backup access activity is not only logged but actively reviewed or alerted so suspicious repository access can be detected and investigated.

## Rationale

Audit records alone do not reduce risk if no one reviews them or if anomalous backup access fails to trigger operator attention. Review workflows and alerting make backup repository misuse operationally detectable before it becomes an undetected path to full cluster-state exposure.

## Scope

Applies to all production clusters. Covers periodic review workflows, alert thresholds, and response routing for etcd backup repository and restore-media access activity.

## Enforcement

- Primary Enforcement Point: Backup repository access monitoring review
- Implementation Surface: Periodic access review workflows, anomaly detection or alert rules, and response routing for monitored repository access.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Backup and Recovery Operations
