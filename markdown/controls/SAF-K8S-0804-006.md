# SAF-K8S-0804-006 - Sealed storage binding of encrypted AI artifacts to enclave measurements

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.4 - Confidential Computing for AI Workloads
- Maturity: Advanced
- Classification: ai-specific

## Statement

Sealed storage must be used to bind encrypted model artifacts and data to specific attested enclave configurations so protected artifacts cannot be decrypted after unauthorized enclave configuration changes.

## Objective

Ensure encrypted models and data remain bound to their authorized enclave measurements and cannot be reused outside the approved enclave configuration.

## Rationale

Sealed storage extends confidential computing beyond memory encryption by binding encrypted artifacts to a specific enclave identity or measurement. This is distinct from key release because it governs whether previously protected artifacts remain usable when the enclave configuration changes after initial provisioning.

## Scope

Applies to confidential AI workloads using sealed storage mechanisms for encrypted models or data at rest. Covers enclave sealing identity configuration, binding of encrypted artifacts to enclave measurements, and verification of decryption success or failure when enclave state changes.

## Enforcement

- Primary Enforcement Point: Enclave Sealed Storage Configuration
- Implementation Surface: TEE sealed storage mechanisms binding encrypted model artifacts to enclave measurements, sealed storage policy configuration in enclave runtimes, and model decryption success or failure evidence for expected and changed enclave configurations.
- Owner Primary: Security Engineering
- Owner Supporting: Key Management; ML Platform Engineering; GPU Infrastructure Engineering
