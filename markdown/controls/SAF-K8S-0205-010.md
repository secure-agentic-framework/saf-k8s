# SAF-K8S-0205-010 - Node-level firewall rule restriction for cluster communication

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.5 - kube-proxy and Node Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Kubernetes nodes must implement host-level firewall rules that restrict inbound and outbound traffic to required cluster communication ports and protocols.

## Objective

Provide defense-in-depth network segmentation at the node level by constraining node communication to approved cluster paths.

## Rationale

Node-level firewall rules provide an independent protection layer even if pod-level policy enforcement is bypassed or misconfigured. Restricting allowed ports and protocols is independently auditable from periodic review or change governance because it governs the active network exposure of each node.

## Scope

Applies to all production cluster nodes. Cloud environments may use security groups or VPC firewall rules as the node-level firewall layer. On-premises environments may use iptables, nftables, or firewalld.

## Enforcement

- Primary Enforcement Point: Cloud security group or host firewall rules applied through infrastructure provisioning and node bootstrap.
- Implementation Surface: Cloud security groups, VPC firewall rules, NSGs, iptables, nftables, firewalld, and firewall verification scripts for allowed cluster communication paths.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Cloud Infrastructure; Security Engineering
