# SAF-K8S-0103-014 - Scheduler API and decision endpoint access restriction

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters must restrict access to the scheduler API and scheduling decision endpoints to authorized control-plane components only.

## Objective

Prevent unauthorized influence over workload placement decisions by limiting scheduler-access paths to authorized control-plane components only.

## Rationale

The scheduler determines where workloads run, making it a high-value target for attackers seeking to place pods on compromised nodes, co-locate with sensitive workloads, or bypass security-aware placement controls. Restricting scheduler API and decision endpoints to authorized control-plane components reduces the opportunity for direct placement manipulation.

## Scope

Applies to all production clusters. Covers default scheduler configuration, scheduler API exposure, bind-address constraints, and RBAC policies governing scheduler-related resources. For managed Kubernetes services, applies to customer-visible scheduler-access settings and compensating controls where scheduler internals are provider-managed.

## Enforcement

- Primary Enforcement Point: Scheduler API exposure and RBAC restriction
- Implementation Surface: Scheduler configuration, bind address settings, scheduler RBAC bindings, and periodic validation that scheduler endpoints are not exposed beyond authorized control-plane paths.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Application Platform teams
