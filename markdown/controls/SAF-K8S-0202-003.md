# SAF-K8S-0202-003 - Container runtime patching and version management

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.2 - Container Runtime Security
- Maturity: Foundational
- Classification: baseline

## Statement

Container runtime versions must be tracked against a supported version matrix, patched within defined SLA timelines for security vulnerabilities, and validated for compatibility with the Kubernetes version and workload requirements before deployment to production nodes.

## Objective

Ensure that known container runtime vulnerabilities are remediated promptly and that runtime updates do not introduce compatibility regressions that could affect workload availability or security posture.

## Rationale

Container runtime vulnerabilities (e.g., CVE-2024-21626 in runc) can enable container escape, host filesystem access, or denial of service. Delayed patching leaves nodes exposed to publicly known exploits. However, untested runtime updates can break container networking, storage drivers, or GPU passthrough, causing workload failures. A structured patching cadence with compatibility validation balances timely remediation against operational stability.

## Scope

Applies to containerd, CRI-O, runc, and any supplementary runtime components (gVisor runsc, Kata Containers runtime) on all cluster nodes. Managed Kubernetes services satisfy this control when the provider documents their runtime patching SLA and the customer validates provider compliance. Version pinning documentation must accompany all production runtime deployments.

## Enforcement

- Primary Enforcement Point: Node image build pipeline, vulnerability scanning integration, and change management process for runtime version updates.
- Implementation Surface: Runtime version tracking matrix (containerd, runc, NVIDIA Container Runtime), CVE monitoring feeds (GitHub advisories, NVIDIA security bulletins), node pool rolling update procedures, and runtime compatibility validation tooling.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Vulnerability Management
