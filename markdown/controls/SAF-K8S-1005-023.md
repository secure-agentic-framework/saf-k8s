# SAF-K8S-1005-023 - GPU-aware node draining and accelerator workload containment

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.5 - Incident Response for Kubernetes
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must define and document AI-specific containment procedures for compromised GPU nodes and accelerator-attached workloads, including GPU-aware node draining and quarantine actions that preserve evidence while minimizing unnecessary disruption to unaffected AI workloads.

## Objective

Enable safe containment of incidents involving GPU nodes and shared accelerator capacity through targeted, GPU-aware isolation procedures that preserve evidence and avoid avoidable collateral damage to co-scheduled AI workloads.

## Rationale

Compromised GPU nodes and shared accelerator environments require containment decisions that differ from standard Kubernetes node isolation. Draining or quarantining these nodes without accelerator-aware procedures can destroy evidence, corrupt training state, or unnecessarily disrupt unaffected workloads sharing the same scarce hardware.

## Scope

Applies to Kubernetes clusters running GPU or accelerator-backed AI workloads. Covers GPU-aware node draining, accelerator node quarantine, and related containment actions for compromised AI workloads attached to specialized hardware.

## Enforcement

- Primary Enforcement Point: AI-specific incident response playbooks and accelerator-node containment procedures.
- Implementation Surface: GPU-aware drain and quarantine procedures, accelerator node isolation playbooks, evidence-preservation guidance for GPU workloads, and service-specific runbooks for compromised accelerator capacity.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; AI/ML Engineering; SRE
