# SAF-K8S-0909-012 - Context poisoning monitoring and incident response for RAG retrieval

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.9 - RAG Infrastructure Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

RAG platforms must monitor vector index changes for unauthorized or anomalous insertions and modifications, generate alerts for suspected context poisoning, and investigate or contain poisoning events before manipulated retrieval results can influence LLM responses.

## Objective

Detect, investigate, and contain context poisoning attacks that manipulate RAG retrieval outputs through compromised or tampered vector index state.

## Rationale

Context poisoning is an operational detection and response problem as much as a retrieval-quality problem. Even with retrieval validation, poisoned indexes can continue to influence responses unless index changes are monitored, alerts are triaged, and suspicious insertions or modifications are investigated and contained.

## Scope

Applies to RAG retrieval platforms on Kubernetes. Covers vector index change monitoring, poisoning alert generation, alert triage, and incident investigation or containment records for suspected poisoned retrieval state. Excludes generic retrieval relevance threshold tuning and chunk-level prompt assembly validation.

## Enforcement

- Primary Enforcement Point: Vector index monitoring, poisoning alerting, and incident investigation workflow.
- Implementation Surface: Vector index snapshot or diff monitoring, alert rules for unauthorized insertions or modifications, retrieval integrity monitoring dashboards, and incident handling records for suspected context poisoning events.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Security Operations
