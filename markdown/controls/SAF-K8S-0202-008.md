# SAF-K8S-0202-008 - Container runtime socket root-only protection

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.2 - Container Runtime Security
- Maturity: Foundational
- Classification: baseline

## Statement

Container runtime control sockets must be owned by root and protected with permissions that prevent non-root processes and workloads from accessing the runtime control plane.

## Objective

Prevent unauthorized local access to the container runtime control socket on cluster nodes.

## Rationale

The container runtime socket provides full control over container lifecycle operations. If non-root users, compromised workloads, or local processes can reach the socket, they can create privileged containers, escape isolation boundaries, or interfere with other workloads. Socket ownership and permission controls are independently auditable from broader runtime configuration settings.

## Scope

Applies to all nodes running containerd or CRI-O in production clusters. Managed Kubernetes services where the runtime is provider-managed satisfy this control through provider security documentation and compensating network controls. Custom or alternative CRI implementations must demonstrate equivalent hardening.

## Enforcement

- Primary Enforcement Point: Node image build pipeline and node-level runtime socket ownership and permission settings.
- Implementation Surface: Runtime socket filesystem paths such as /run/containerd/containerd.sock or CRI-O equivalents, file ownership, file mode, and validation of non-root access denial.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; OS/Infrastructure
