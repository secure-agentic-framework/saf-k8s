# SAF-K8S-0605-009 - AI custom resource validation and policy constraint enforcement

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.5 - Admission Control
- Maturity: Practitioner
- Classification: ai-specific

## Statement

AI-specific Custom Resource Definitions must have validating admission webhooks that enforce structural schema correctness, security constraints, and organizational policy requirements before AI custom resources are admitted to the cluster.

## Objective

Prevent malformed or policy-violating AI custom resources from creating unsafe workloads, excessive GPU consumption, or unauthorized model and data access patterns.

## Rationale

AI CRDs such as InferenceService, TrainingJob, RayCluster, and PodGroup describe high-impact workloads and resource requests. Without CRD-specific validation and constraint enforcement, malformed or malicious objects can bypass operator assumptions and create unsafe runtime state before downstream controls can intervene.

## Scope

Applies to all Kubernetes clusters running AI/ML operator frameworks such as KServe, Kubeflow, Ray, and Volcano that install AI-specific CRDs. Clusters without AI operator CRDs may document this control as not applicable.

## Enforcement

- Primary Enforcement Point: Kubernetes validating admission webhooks for AI CRDs.
- Implementation Surface: ValidatingWebhookConfiguration resources for AI CRDs, operator-specific validation rules, structural schema validation, security and policy constraint logic, and test results demonstrating rejection of non-compliant AI custom resources.
- Owner Primary: Platform Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
