# SAF-K8S-0202-011 - Node mandatory access control activation for container workloads

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.2 - Container Runtime Security
- Maturity: Foundational
- Classification: baseline

## Statement

Supported nodes must activate AppArmor or SELinux mandatory access control for container workloads and maintain the policy or labeling prerequisites required for those protections to remain effective by default.

## Objective

Reduce container escape and cross-workload impact by ensuring node-level mandatory access control is active for container execution.

## Rationale

AppArmor and SELinux constrain file access, process behavior, and labeling boundaries even when a workload reaches a vulnerable code path. Activation and maintenance of mandatory access control on supported nodes is a distinct hardening obligation from default seccomp enforcement.

## Scope

Applies to production cluster nodes and the workloads scheduled onto them. Seccomp defaults and AppArmor or SELinux activation must be validated for supported operating systems, node pools, and runtime configurations, with exceptions documented and reviewed.

## Enforcement

- Primary Enforcement Point: Node operating-system MAC activation and workload policy or labeling prerequisites for AppArmor or SELinux enforcement.
- Implementation Surface: AppArmor profile loading, SELinux enablement and labeling prerequisites, and validation that workloads run with active MAC protections on supported nodes.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; OS/Infrastructure
