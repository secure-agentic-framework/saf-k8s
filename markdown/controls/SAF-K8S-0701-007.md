# SAF-K8S-0701-007 - High-performance AI storage backend hardening

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.1 - Persistent Storage Security
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations using high-performance storage backends for AI workloads (Lustre, GPFS/Spectrum Scale, parallel NFS, object storage) must document and mitigate the elevated privilege requirements of these backends, restrict host-level mounts and privileged containers to dedicated storage nodes, and enforce compensating controls for expanded attack surface.

## Objective

Ensure that high-performance storage backends required by AI workloads do not introduce unmitigated privilege escalation paths despite their inherent need for elevated host access.

## Rationale

High-performance AI storage backends such as Lustre and GPFS often require host-level mounts and privileged containers, which expand the attack surface beyond standard CSI drivers. Without dedicated node isolation, AppArmor/SELinux confinement, and documented risk assessments, these backends create persistent privilege escalation vectors on GPU nodes.

## Scope

Applies to Kubernetes clusters using high-performance storage backends for AI workloads including Lustre, GPFS/Spectrum Scale, parallel NFS, and object storage gateways. Covers storage node dedication, privilege confinement, and risk assessment.

## Enforcement

- Primary Enforcement Point: Node taints dedicating storage nodes, admission control restricting privileged storage pods, and documented risk registers for elevated privilege requirements.
- Implementation Surface: High-performance storage backend configurations (Lustre, GPFS/Spectrum Scale, parallel NFS), node taints dedicating storage nodes, AppArmor/SELinux profiles for privileged storage pods, runtime security monitoring for storage backend behavior, and risk register documentation for elevated privilege requirements with compensating controls.
- Owner Primary: Storage Engineering
- Owner Supporting: Platform Engineering; Security Engineering
