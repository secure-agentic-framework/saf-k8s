# SAF-K8S-0606-033 - Git-only production deployment path enforcement

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production clusters managed by GitOps must prohibit direct runtime changes through cluster RBAC and enforce Git as the sole approved deployment path for production configuration changes.

## Objective

Prevent human or ad hoc tooling from bypassing GitOps by enforcing Git as the only approved production change path and tightly restricting direct cluster modification rights.

## Rationale

If operators can directly modify production resources outside Git, review and deployment discipline can be bypassed even when GitOps controllers are healthy. Enforcing Git as the sole production deployment path is distinct from governing and logging exceptional emergency access.

## Scope

Applies to production namespaces and clusters governed through GitOps. Covers RBAC restrictions on direct create, update, and delete operations and Git-only deployment path enforcement for production changes.

## Enforcement

- Primary Enforcement Point: Cluster RBAC and operational controls governing approved production change paths.
- Implementation Surface: Cluster RBAC denying create, update, and delete permissions on deployed resources for human users in production namespaces, GitOps-only deployment operating procedures, and controls preventing routine direct kubectl changes in production.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; SRE / Operations
