# SAF-K8S-0203-015 - Cloud metadata endpoint restriction settings

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.3 - Host OS and Kernel Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes nodes on cloud infrastructure must apply restrictive metadata endpoint or hop-limit settings appropriate to the provider so metadata access cannot traverse unintended network paths or permissive platform defaults.

## Objective

Constrain provider metadata endpoint behavior through restrictive instance or node-pool settings beyond authenticated access mode alone.

## Rationale

Even when authenticated metadata modes are enabled, permissive hop limits or endpoint behaviors can still expose metadata access through unintended local routing paths, host-network reachability, or provider-default settings. Restrictive endpoint or hop-limit settings are independently auditable from metadata protocol mode enforcement.

## Scope

Applies to Kubernetes nodes on AWS, GCP, Azure, and other cloud providers exposing instance metadata services. Covers node group, VM, or managed service metadata configuration that customers can set or must verify through provider documentation and platform APIs.

## Enforcement

- Primary Enforcement Point: Cloud infrastructure provisioning and managed service configuration governing metadata endpoint behavior on node instances or managed node pools.
- Implementation Surface: Provider-specific endpoint enablement flags, hop-limit settings, local-routing constraints, and managed service verification of restrictive metadata endpoint defaults.
- Owner Primary: Cloud Infrastructure
- Owner Supporting: Platform Engineering; Security Engineering
