# SAF-K8S-0801-013 - MPS and time-slicing memory remnant prevention verification

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Advanced
- Classification: ai-specific

## Statement

Clusters using NVIDIA MPS or GPU time-slicing must verify that GPU memory remnants are not recoverable between workload transitions and must validate the effectiveness of memory clearing or reset controls for shared GPU contexts.

## Objective

Provide technical assurance that weaker GPU-sharing modes do not expose recoverable memory remnants between co-located or successive workloads on the same GPU context.

## Rationale

Approving weaker GPU-sharing modes is insufficient without independent evidence that prior workload data cannot be recovered from shared GPU memory. This control focuses on testable remnant-prevention outcomes and validation of the memory clearing mechanisms used to support those outcomes.

## Scope

Applies to any Kubernetes cluster using NVIDIA MPS or GPU time-slicing. Covers GPU memory remnant testing and validation of memory clearing or reset behavior between workload transitions in shared GPU deployments.

## Enforcement

- Primary Enforcement Point: Shared-GPU Memory Remnant Prevention Verification
- Implementation Surface: GPU memory isolation verification tests, GPU reset or clearing validation, and technical evidence showing no recoverable data remnants across workload transitions in shared GPU modes.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Data Protection Office
