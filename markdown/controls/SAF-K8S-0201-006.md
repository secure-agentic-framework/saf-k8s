# SAF-K8S-0201-006 - Node system and kube reserved resource allocations

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.1 - Kubelet and Node Configuration Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes nodes must configure system-reserved and kube-reserved resource allocations so that kubelet, the container runtime, and operating system processes retain sufficient CPU, memory, and storage to remain responsive under workload pressure.

## Objective

Prevent user workloads from starving node-level control processes by reserving sufficient capacity for kubelet, runtime, and operating system functions on every production node.

## Rationale

Node stability depends on protecting core system processes from workload contention. Explicit system and kube reservations prevent AI training or inference workloads from consuming the resources needed for health checks, logging, networking, and pod lifecycle operations.

## Scope

Applies to all production cluster nodes, with particular attention to nodes hosting GPU workloads or AI training jobs that can exhibit sudden memory spikes. Managed Kubernetes services that expose allocatable resource configuration must have these values tuned for the node instance type. Exemptions require documented risk acceptance.

## Enforcement

- Primary Enforcement Point: Node provisioning templates and kubelet configuration files specifying systemReserved and kubeReserved settings.
- Implementation Surface: KubeletConfiguration resource reservation settings for systemReserved and kubeReserved, GPU driver and node agent resource accounting, node capacity planning tooling, and node allocatable validation in provisioning pipelines.
- Owner Primary: Platform Engineering
- Owner Supporting: Workload Teams; Capacity Planning
