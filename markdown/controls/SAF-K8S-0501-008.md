# SAF-K8S-0501-008 - AI workload type network microsegmentation

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.1 - Network Policies
- Maturity: Foundational
- Classification: ai-specific

## Statement

Training, inference, pipeline orchestration, and notebook environments must reside in separate network segments with explicit policy-controlled communication paths between workload types.

## Objective

Prevent unauthorized lateral movement and cross-contamination between distinct AI workload classes by enforcing network boundaries around each workload type.

## Rationale

AI environments combine training systems, notebooks, orchestration services, and production inference paths that have different trust assumptions and data sensitivity. Network microsegmentation confines compromise within a workload class and reduces direct access between development, training, and serving planes.

## Scope

Applies to all Kubernetes clusters hosting AI and ML workloads, including training jobs, inference services, pipeline controllers, and interactive notebook environments.

## Enforcement

- Primary Enforcement Point: Kubernetes network policy enforcement separating AI workload namespaces and limiting allowed communication paths between workload types.
- Implementation Surface: Namespace-scoped and cluster-scoped network policies, workload-type selectors, and connectivity validation between training, inference, pipeline, and notebook segments.
- Owner Primary: Network Engineering
- Owner Supporting: ML Platform Engineering
