# SAF-K8S-0405-007 - Authentication endpoint availability and DoS protection

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.5 - Identity Abuse Detection and Mitigation
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must protect Kubernetes authentication endpoints against denial-of-service attacks through API server rate limiting, API Priority and Fairness configuration, connection throttling, and availability monitoring to ensure authentication infrastructure remains accessible during attacks or high-load conditions.

## Objective

Ensure that authentication endpoints remain available and resilient against brute force and denial-of-service attacks that could lock out legitimate users or degrade cluster operations during an incident.

## Rationale

Authentication endpoints (API server, OIDC providers, webhook token reviewers) are vulnerable to denial-of-service attacks that can prevent legitimate administrators from accessing the cluster during an incident. Rate limiting and connection throttling protect against brute force credential attacks and ensure authentication infrastructure availability under load.

## Scope

Applies to all Kubernetes API server authentication endpoints, OIDC provider endpoints, webhook token review services, and cloud provider authentication integrations. Covers rate limiting, API Priority and Fairness configuration, and availability monitoring.

## Enforcement

- Primary Enforcement Point: API server rate limiting configurations (--max-requests-inflight, APF priority levels) and monitoring/alerting for authentication endpoint availability and error rates.
- Implementation Surface: API server rate limiting flags (--max-requests-inflight, --max-mutating-requests-inflight), API Priority and Fairness (APF) FlowSchema and PriorityLevelConfiguration resources, authentication endpoint availability monitoring and SLA dashboards, and authentication error rate alerting (401/403 spike detection).
- Owner Primary: Platform Engineering
- Owner Supporting: Identity and Access Management; Security Engineering
