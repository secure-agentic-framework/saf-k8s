# SAF-K8S-0904-012 - Cross-classification pipeline data transfer authorization gates

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.4 - AI Pipeline Orchestration and Experimentation Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Data transfers between pipeline stages with different classification levels must require explicit authorization through pipeline-level authorization gates.

## Objective

Enforce approved, policy-governed movement of data across classification boundaries inside ML pipelines.

## Rationale

Even when stages are separately isolated, data-classification boundaries can still be bypassed if outputs are automatically transferred across sensitivity levels without explicit policy approval. Authorization gates are required to prevent uncontrolled classification downgrades or unauthorized access to restricted data.

## Scope

Applies to ML pipelines on Kubernetes that move data, artifacts, or derived outputs between stages with different classification levels. Covers policy evaluation for cross-classification transfers, approval gates, and classification-aware transfer controls. Excludes artifact encryption and storage access controls covered by SAF-K8S-0904-002.

## Enforcement

- Primary Enforcement Point: Data classification gateway policy and transfer authorization control point.
- Implementation Surface: Data classification gateway policies, admission or workflow authorization gates for cross-classification transfers, and logs or policy decisions showing approved and denied movement of data between differently classified pipeline stages.
- Owner Primary: Data Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
