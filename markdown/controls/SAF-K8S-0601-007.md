# SAF-K8S-0601-007 - AI GPU and ML framework base image validation

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

AI workload images must use validated CUDA base images and ML framework base images that have been assessed for GPU compatibility, version alignment, and removal of non-essential components, and any non-standard AI base image must be documented in an approved exception process.

## Objective

Ensure AI-specific base images are selected from validated GPU and ML framework variants rather than generic or unvetted images that introduce compatibility risk and excess attack surface.

## Rationale

AI images carry enlarged native dependency chains and compatibility constraints that do not apply to generic application images. CUDA, cuDNN, PyTorch, TensorFlow, Triton, and similar bases must be explicitly validated and curated so organizations do not trade operational compatibility for unnecessary exposure.

## Scope

Applies to training, inference, and model-serving images that depend on CUDA, GPU drivers, or heavyweight ML framework runtimes. Exceptions require documented justification and approved compensating controls.

## Enforcement

- Primary Enforcement Point: Image Approval / ML Platform Governance
- Implementation Surface: Approved AI base image catalogs, GPU compatibility validation records, and exception approvals for non-standard AI images.
- Owner Primary: ML Platform Engineering
- Owner Supporting: DevSecOps / Release Engineering; Security Engineering
