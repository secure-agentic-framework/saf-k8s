# SAF-K8S-0802-002 - CUDA library and container toolkit security

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.2 - GPU Driver, Library, and Toolkit Security
- Maturity: Advanced
- Classification: ai-specific

## Statement

CUDA libraries and the NVIDIA Container Toolkit must be configured with security-hardened settings, including disabling unnecessary compat libraries, restricting driver persistence mode to authorized use cases, and validating container toolkit configurations to prevent unauthorized host GPU access paths.

## Objective

Harden the CUDA runtime and container toolkit stack to minimize the attack surface through which containerized workloads interact with GPU hardware, preventing privilege escalation via misconfigured toolkit components.

## Rationale

The NVIDIA Container Toolkit (nvidia-container-runtime) modifies container configurations to inject GPU device access, libraries, and driver bindings. Misconfigured toolkit settings can expose host filesystem paths, enable unauthorized device access beyond requested GPUs, or allow containers to load arbitrary compat libraries that bypass version controls. Driver persistence mode, while improving GPU initialization latency, maintains a persistent kernel-level process that increases the attack surface window.

## Scope

Applies to all Kubernetes nodes where the NVIDIA Container Toolkit is installed and GPU workloads are scheduled. Covers runtime configuration files (nvidia-container-runtime config), CDI specifications, and CUDA library injection settings. Managed Kubernetes GPU services must verify provider toolkit configurations meet hardening requirements.

## Enforcement

- Primary Enforcement Point: NVIDIA Container Toolkit runtime configuration, container runtime (containerd or CRI-O) GPU integration settings, and node-level configuration management for CUDA library paths and persistence mode.
- Implementation Surface: NVIDIA Container Toolkit runtime configuration (/etc/nvidia-container-runtime/config.toml including no-cgroups, visible-devices settings), Container Device Interface (CDI) specifications, CUDA base images (nvidia/cuda:*) in approved image registries with digest pinning, CUDA compatibility library (libcuda.so compat mode) settings, nvidia-persistenced daemon configuration, containerd/CRI-O GPU runtime hook configuration controlling device file mounts (/dev/nvidia*), and container image vulnerability scanning for CUDA components.
- Owner Primary: ML Platform Engineering
- Owner Supporting: GPU Infrastructure Engineering; Container Platform Engineering; Security Engineering
