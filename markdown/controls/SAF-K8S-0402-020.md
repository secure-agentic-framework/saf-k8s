# SAF-K8S-0402-020 - Projected service account token issuance path enforcement

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.2 - Service Accounts and Workload Identity
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes clusters must issue workload service account tokens only through projected service account token volumes or the TokenRequest API, and must enforce those issuance paths through admission and policy controls instead of allowing legacy secret-mounted token delivery.

## Objective

Ensure workload identity tokens are issued only through explicit projected-token mechanisms with verifiable request flows.

## Rationale

Projected token volumes and TokenRequest-based issuance provide a controlled, inspectable token delivery path that avoids implicit secret mounting and enables workload identity policies to verify how a token was issued. Without issuance-path enforcement, workloads can fall back to legacy or unmanaged token delivery paths that are harder to govern and rotate.

## Scope

Applies to all workload service account token issuance paths used for in-cluster and federated workload authentication.

## Enforcement

- Primary Enforcement Point: Admission and policy enforcement for workload token issuance method and legacy token delivery prohibition.
- Implementation Surface: Projected ServiceAccountToken volume configuration, TokenRequest API usage, and admission controls enforcing projected-token issuance paths.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Application Development
