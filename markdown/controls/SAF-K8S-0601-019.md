# SAF-K8S-0601-019 - Runtime container vulnerability exposure monitoring and exception governance

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Deployed workloads must be continuously monitored for runtime vulnerability exposure, including AI-specific dependency findings, and unresolved critical or high-severity exposures must have documented risk acceptance and review records.

## Objective

Maintain current visibility into vulnerability exposure in running workloads and govern unresolved high-risk findings until remediation is complete.

## Rationale

Runtime exposure monitoring is a distinct operational control claim from registry rescanning: it identifies which vulnerabilities are present in deployed workloads, tracks AI-specific dependency exposure in use, and ensures unresolved critical or high findings are explicitly governed rather than silently tolerated.

## Scope

Applies to container registries and deployed workloads across all Kubernetes clusters, including runtime scanning of AI/ML images and continuous monitoring of native library exposure.

## Enforcement

- Primary Enforcement Point: Runtime scanning agents, exposure dashboards, and vulnerability exception governance workflows.
- Implementation Surface: Runtime scanning operators, workload coverage telemetry, exposure dashboards for running containers and AI dependencies, and documented risk acceptance workflows for unresolved findings.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
