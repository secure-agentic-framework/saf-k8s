# SAF-K8S-1002-002 - Distributed tracing for ML pipelines

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.2 - Monitoring, Metrics, and Tracing
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must implement end-to-end distributed tracing across ML pipeline stages, including data ingestion, training, and inference serving, with trace context propagated across service boundaries to enable rapid root-cause analysis of pipeline failures and security incidents.

## Objective

Provide full observability into ML pipeline execution to enable rapid root-cause analysis of failures and detection of malicious interference across pipeline stages.

## Rationale

ML pipelines span multiple Kubernetes services and batch jobs, making it difficult to trace failures or malicious interference across pipeline stages without distributed tracing. Trace context propagation enables correlation of events across data ingestion, training, model registry, and inference serving stages.

## Scope

Applies to all ML pipeline orchestration systems running on Kubernetes, including data ingestion services, training job controllers, model registries, and inference serving platforms. Trace propagation must cover both synchronous and asynchronous pipeline transitions. Alerting is covered by SAF-K8S-1002-004.

## Enforcement

- Primary Enforcement Point: Tracing instrumentation libraries and trace collector deployment configuration.
- Implementation Surface: OpenTelemetry instrumentation libraries and trace collector deployment (OTel Collector), W3C Trace Context propagation (traceparent header) across ML pipeline stages (data ingestion, training, model registry, inference serving), and trace context injection via environment variables and Kubernetes annotations on training jobs and InferenceService CRDs.
- Owner Primary: AI/ML Engineering
- Owner Supporting: Platform Engineering; SRE
