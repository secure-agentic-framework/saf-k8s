# SAF-K8S-0909-010 - Vector database encryption in transit and at rest

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.9 - RAG Infrastructure Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Vector databases used for RAG retrieval must encrypt client connections in transit and encrypt vector storage at rest so embeddings and metadata remain protected on the network and on persistent storage volumes.

## Objective

Protect vector embeddings and related metadata from interception and storage compromise through transport and storage encryption.

## Rationale

Even when access is restricted, embeddings and metadata remain exposed if vector databases do not use TLS for client connections or storage-layer encryption for persistent volumes. Encryption reduces disclosure risk from network interception, stolen disks, snapshots, and underlying storage exposure.

## Scope

Applies to vector database instances used for RAG retrieval on Kubernetes, including self-managed and managed vector database services. Covers TLS for client connections and encryption at rest for vector storage volumes or equivalent managed storage layers. Excludes authentication and collection-level authorization.

## Enforcement

- Primary Enforcement Point: TLS configuration for vector database client connections and storage-layer encryption for vector persistence.
- Implementation Surface: TLS certificate configuration for vector database client connections, encryption-at-rest settings for persistent volumes or managed vector storage, and verification that vector storage volumes inherit approved encryption controls.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Infrastructure Engineering
