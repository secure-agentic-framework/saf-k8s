# SAF-K8S-1005-020 - Vulnerability response ownership and multi-party coordination governance for Kubernetes AI infrastructure

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.5 - Incident Response for Kubernetes
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must define ownership, escalation responsibilities, and multi-party coordination procedures for vulnerabilities affecting shared Kubernetes and AI infrastructure components, including upstream open-source projects and cloud providers when remediation spans organizational boundaries.

## Objective

Ensure vulnerability reports affecting shared Kubernetes and AI infrastructure components are coordinated through named owners, escalation paths, and external partner engagement processes until remediation is complete.

## Rationale

Shared platform vulnerabilities often require coordinated action across security engineering, platform owners, ML platform teams, cloud providers, and upstream open-source maintainers. Without defined ownership and coordination governance, critical issues stall between teams and boundary-spanning vulnerabilities remain unresolved.

## Scope

Kubernetes control plane, node components, cluster add-ons, AI/ML operators, and GPU infrastructure vulnerabilities that require coordination across internal teams or external providers.

## Enforcement

- Primary Enforcement Point: governance
- Implementation Surface: Roles and responsibilities matrix, escalation runbooks, cloud provider and upstream coordination playbooks, and vulnerability response governance records for shared infrastructure issues.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; ML Platform Engineering; Site Reliability Engineering
