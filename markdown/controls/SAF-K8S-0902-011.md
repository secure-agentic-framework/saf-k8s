# SAF-K8S-0902-011 - Streaming token-level output leakage and policy-violation filtering

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.2 - Inference Server and Model Serving Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

LLM serving infrastructure must inspect streaming model outputs in real time for PII, secrets, and policy-violating content and block, redact, or terminate responses before disallowed tokens reach the client through controls that operate independently of the model's own safety training.

## Objective

Prevent sensitive data leakage and policy-violating content release in streaming LLM responses by filtering output tokens before they reach the client.

## Rationale

For streaming inference, post-generation review is too late because tokens may already be delivered to the caller. Token-window scanners operating in the serving path can detect secrets, PII, and policy violations in real time and stop or redact the response before disclosure occurs, complementing both model-level safety training and general response filtering.

## Scope

Applies to LLM inference endpoints deployed on Kubernetes that return streaming or chunked responses. Covers real-time token inspection, redaction, and stream termination controls. Excludes input-side prompt injection classification, which is addressed separately.

## Enforcement

- Primary Enforcement Point: Streaming response filter middleware, sidecar proxies, or gateway components in the output path.
- Implementation Surface: Streaming token scanners, regex and NER-based PII or secret detectors, policy classifiers operating on token windows, stream redaction or termination logic, and test harnesses demonstrating pre-delivery blocking in SSE or WebSocket output flows.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Application Security
