# SAF-K8S-0905-009 - Model provenance verification at deployment

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Model origin and integrity must be verified at deploy time using Kubernetes admission controllers or deployment pipeline hooks that validate cryptographic signatures, verify in-toto attestation chain completeness, and confirm OCI digest integrity before allowing model deployment to production inference endpoints.

## Objective

Ensure that only signed and verified models from known provenance reach production by enforcing verification at the deployment boundary.

## Rationale

Build-time signing (SAF-K8S-0905-002) creates the cryptographic evidence, but without deploy-time enforcement an unsigned or tampered model can still be deployed by bypassing the signing pipeline. Admission controller verification provides an independent enforcement point that catches supply chain bypasses.

## Scope

Applies to all model deployments to production inference endpoints on Kubernetes. Covers admission controller signature verification, attestation chain validation, and OCI digest confirmation. Excludes build-time signing and attestation generation covered by SAF-K8S-0905-002.

## Enforcement

- Primary Enforcement Point: Kubernetes admission controllers (Kyverno, Gatekeeper) or deployment pipeline verification hooks.
- Implementation Surface: Admission controllers (Kyverno, Gatekeeper) verifying model signatures and attestation chain completeness at deploy time, deployment pipeline verification hooks, OCI digest validation against registry metadata, and verification failure alerting and blocking.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; DevOps Engineering
