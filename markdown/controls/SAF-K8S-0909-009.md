# SAF-K8S-0909-009 - Vector database authentication and collection-level access control

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.9 - RAG Infrastructure Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Vector databases used for RAG retrieval must enforce authentication for all client connections and implement collection-level access controls that restrict which applications and principals can read from or write to each vector collection.

## Objective

Prevent unauthorized access to vector embeddings and unauthorized modification of vector collections by enforcing authenticated client identity and collection-scoped authorization.

## Rationale

RAG vector databases expose high-value embeddings that encode organizational knowledge. Without authenticated client identity and collection-level authorization, unauthorized principals can enumerate, read, or poison vector collections across applications or tenants.

## Scope

Applies to vector database instances used for RAG retrieval on Kubernetes, including Milvus, Weaviate, Qdrant, pgvector, Chroma, and equivalent managed services. Covers client authentication, RBAC or equivalent collection-level authorization, and tenant- or application-scoped access boundaries. Excludes encryption at rest and in transit.

## Enforcement

- Primary Enforcement Point: Vector database authentication and collection-level authorization configuration.
- Implementation Surface: Vector database authentication enablement, Kubernetes ServiceAccount tokens or mTLS for service identity, collection-level RBAC or equivalent authorization policies, and tenant- or application-scoped collection access restrictions.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Infrastructure Engineering
