# SAF-K8S-1001-012 - SIEM correlation rules for AI-specific attack patterns

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must develop and maintain SIEM correlation rules that detect AI-specific attack patterns, including unauthorized model access, anomalous GPU utilization, training job manipulation, and inference endpoint abuse.

## Objective

Detect sophisticated attacks targeting AI workloads through purpose-built SIEM correlation rules that recognize AI-specific threat patterns not captured by generic infrastructure detection logic.

## Rationale

Generic SIEM correlation rules designed for traditional workloads do not detect AI-specific attack patterns such as GPU cryptomining through sustained utilization without a matching training job, unauthorized model deployment via InferenceService creation by an unexpected ServiceAccount, or inference endpoint abuse via request rate spikes. Purpose-built correlation rules are required to surface these patterns from the AI workload event streams ingested per SAF-K8S-1001-006.

## Scope

Applies to all production Kubernetes clusters hosting AI workloads. Covers SIEM correlation rule sets for AI-specific attack detection scenarios. Depends on SIEM ingestion being operational per SAF-K8S-1001-006.

## Enforcement

- Primary Enforcement Point: Correlation rule engine management within the SIEM platform.
- Implementation Surface: Correlation rules covering: unauthorized model deployment via InferenceService CREATE by unexpected ServiceAccount, GPU cryptomining via sustained utilization without matching training job, training job manipulation via PATCH/UPDATE by non-owner ServiceAccount, inference endpoint abuse via request rate spikes, model exfiltration via large egress from model storage PV, and audit evasion via audit policy ConfigMap UPDATE.
- Owner Primary: Detection Engineering
- Owner Supporting: Security Operations; AI/ML Engineering; Platform Engineering
