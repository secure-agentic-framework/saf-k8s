# SAF-K8S-0703-021 - GPU admission enforcement against unauthorized access and quota bypass

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.3 - Resource Governance and Priority
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must enforce admission controls that block unauthorized GPU access methods, prevent known quota-circumvention patterns, and require sanctioned GPU resource request paths for workloads admitted to GPU-capable clusters.

## Objective

Prevent non-sanctioned GPU consumption at admission time by blocking unauthorized device-access methods and quota-bypass patterns before workloads reach execution.

## Rationale

Detection alone is insufficient when users or malware can request GPUs through unsanctioned mechanisms or evade quota policy before runtime. Admission enforcement reduces abuse by rejecting workloads that bypass approved GPU request paths, violate labeling or allowlisting requirements, or attempt known quota-circumvention patterns.

## Scope

Applies to all Kubernetes clusters with GPU-capable nodes or GPU device plugins. Enforcement must cover direct device access bypasses, hostPath access to GPU device files, and known quota-circumvention patterns for GPU-consuming workloads.

## Enforcement

- Primary Enforcement Point: Admission policies and validation controls on GPU workload submission paths.
- Implementation Surface: Admission control rules, policy engines, and validation webhooks enforcing approved GPU request mechanisms, label requirements, and quota-circumvention prevention before pod admission.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; ML Engineering
