# SAF-K8S-0605-002 - Kyverno admission policies

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.5 - Admission Control
- Maturity: Practitioner
- Classification: baseline

## Statement

Kubernetes clusters using Kyverno must implement validation policies that reject non-compliant workloads, mutation policies that enforce security defaults (labels, resource limits, security contexts), generation policies that create required companion resources, and image verification rules that validate image signatures and attestations before admission. Policies must be version-controlled and tested in audit mode before enforcement.

## Objective

Enforce Kubernetes security and operational policies at admission time using Kyverno's validation, mutation, generation, and image verification capabilities.

## Rationale

Without Kyverno policies (or equivalent), workloads can be deployed without required security contexts, labels, or resource limits, and images can enter the cluster without signature verification. Mutation policies prevent configuration drift by enforcing defaults, while generation policies ensure companion resources (NetworkPolicies, ResourceQuotas) are consistently created. Operating without audit-mode testing risks blocking legitimate deployments.

## Scope

Applies to all Kubernetes clusters where Kyverno is the selected admission controller. Covers all namespaces except those explicitly exempted in documented policy. Organizations using OPA/Gatekeeper exclusively may document this control as not applicable.

## Enforcement

- Primary Enforcement Point: Kubernetes Kyverno admission webhook (validating and mutating).
- Implementation Surface: Kyverno ClusterPolicy and Policy resources, validation policies (validationFailureAction: Audit/Enforce) rejecting non-compliant workloads, mutation policies enforcing security defaults (runAsNonRoot, allowPrivilegeEscalation: false, readOnlyRootFilesystem: true, labels, resource limits), generation policies creating companion resources (NetworkPolicies, ResourceQuotas), verifyImages rules for image signature and attestation verification, kyverno test CLI for policy testing, and Kyverno PolicyReport CRDs for violation and enforcement statistics.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
