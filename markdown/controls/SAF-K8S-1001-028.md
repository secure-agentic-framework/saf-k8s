# SAF-K8S-1001-028 - AI-specific SIEM source onboarding and forwarding

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must onboard Kubernetes audit logs and AI workload security event sources into the SIEM and forward those events through approved ingestion pipelines so AI-relevant telemetry is available for centralized security analysis.

## Objective

Ensure that Kubernetes and AI-specific security telemetry sources are onboarded and forwarded into the SIEM for centralized analysis.

## Rationale

Correlation rules and SOC workflows are ineffective if the required AI security telemetry never reaches the SIEM. Source onboarding and forwarding ensure that Kubernetes audit logs and AI workload security events are actually present in the centralized analysis platform.

## Scope

Applies to SIEM onboarding and event forwarding for Kubernetes audit logs and AI workload security telemetry, including inference, training, registry, policy, and runtime security event sources. Covers source onboarding and forwarding pipeline configuration. Excludes monitoring of delivery completeness and ingestion health after onboarding.

## Enforcement

- Primary Enforcement Point: SIEM source onboarding and forwarding pipeline configuration.
- Implementation Surface: SIEM ingestion configuration for Kubernetes audit logs and AI workload security event sources, source connector onboarding, event pipeline routing, and forwarding paths that deliver AI security telemetry into the centralized SIEM.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; GRC/Compliance
