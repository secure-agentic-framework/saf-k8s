# SAF-K8S-0505-006 - API server private endpoint and authorized network enforcement

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.5 - API Server and Service Exposure Protection
- Maturity: Practitioner
- Classification: baseline

## Statement

Kubernetes API server endpoints must be exposed only through private cluster endpoints or authorized network allowlists and firewall rules that limit reachability to approved management networks.

## Objective

Reduce control-plane attack surface by limiting API server network reachability to explicitly approved management networks.

## Rationale

Private endpoints, authorized network allowlists, and firewall restrictions prevent direct discovery and reachability of the API server from unapproved networks, eliminating a large portion of opportunistic and credential-driven attacks before authentication is even attempted.

## Scope

Applies to all production Kubernetes clusters. Managed Kubernetes services must use private endpoint or authorized network features. Self-managed clusters must place API servers on private networks with bastion or VPN access. Non-production clusters with access to production secrets or infrastructure must also restrict API server access.

## Enforcement

- Primary Enforcement Point: Cloud provider private cluster configuration, API server authorized network settings, and VPC firewall or security-group rules restricting API server endpoint reachability.
- Implementation Surface: Cloud provider private cluster settings (GKE private endpoint and master authorized networks, EKS endpointPublicAccess and endpointPrivateAccess, AKS private cluster and authorized IP ranges), self-managed API server bind and advertise addresses, VPC firewall rules, cloud security groups, and external reachability validation for port 6443.
- Owner Primary: Platform Engineering
- Owner Supporting: Network Engineering; Security Engineering; Cloud Platform Engineering
