# SAF-K8S-0403-005 - AI pipeline secret leakage prevention

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Foundational
- Classification: ai-specific

## Statement

Organizations must prevent secret leakage through training logs, model artifacts, checkpoint files, and pipeline metadata by implementing output filtering, artifact scanning, and dedicated secrets management for model registry credentials, training data access tokens, and API keys used by inference endpoints.

## Objective

Ensure that secrets used by AI/ML pipelines are not inadvertently persisted in training outputs, model artifacts, or pipeline logs where they could be extracted by anyone with access to those artifacts.

## Rationale

AI/ML pipelines present unique secret leakage vectors beyond traditional application workloads. Training jobs may log environment variables including credentials to experiment tracking systems. Model checkpoint files can embed configuration objects containing API keys. Pipeline metadata in orchestrators like Kubeflow or Argo may persist secret values in workflow status fields. Inference endpoints with hardcoded API keys bake credentials into serving containers that may be widely distributed.

## Scope

Applies to all AI/ML pipeline components including training jobs, experiment tracking systems, model registries, checkpoint storage, and inference serving infrastructure. Covers secrets for model registry authentication, training data access, feature store connections, GPU cluster access, and external API keys used by inference endpoints.

## Enforcement

- Primary Enforcement Point: Training log sanitization filters, model artifact scanning in CI/CD pipelines, pipeline metadata redaction in workflow orchestrators, and container image scanning for hardcoded credentials in inference containers.
- Implementation Surface: Training log sanitization filters and structured logging with secret redaction, model artifact and checkpoint file scanning in CI/CD pipelines for embedded credentials, pipeline orchestrator metadata redaction (Kubeflow, Argo Workflow status field sanitization), container image scanning for hardcoded API keys in inference serving containers, experiment tracking client configuration (MLflow, Weights & Biases) with parameter filtering, and dedicated secret injection for model registry credentials, training data access tokens, and inference endpoint API keys.
- Owner Primary: ML Engineering
- Owner Supporting: Security Engineering; Platform Engineering
