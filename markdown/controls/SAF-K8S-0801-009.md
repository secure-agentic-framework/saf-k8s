# SAF-K8S-0801-009 - GPU topology metadata protection and node label visibility restriction

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Kubernetes node labels, device plugin advertisements, and topology information exposing GPU hardware details must be restricted so unauthorized users and workloads cannot enumerate sensitive accelerator metadata.

## Objective

Prevent reconnaissance and tenant-placement inference by limiting exposure of GPU topology and hardware metadata.

## Rationale

GPU feature discovery and device plugin metadata can reveal hardware models, interconnect layouts, driver versions, MIG capabilities, and co-tenant placement clues. Restricting visibility reduces the value of topology information for targeted attacks and reconnaissance against shared GPU infrastructure.

## Scope

Applies to all Kubernetes clusters with GPU nodes where device plugin advertisements or GPU Feature Discovery labels expose accelerator metadata. Visibility exceptions require documented justification.

## Enforcement

- Primary Enforcement Point: GPU metadata publication controls and RBAC restrictions on node and label visibility.
- Implementation Surface: GPU Feature Discovery configuration, node label exposure controls, device plugin metadata filtering, and RBAC limiting node object and label reads.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Platform Engineering; Security Engineering
