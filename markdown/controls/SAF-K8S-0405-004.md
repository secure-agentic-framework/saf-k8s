# SAF-K8S-0405-004 - Security awareness for Kubernetes and GPU administrators

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.5 - Identity Abuse Detection and Mitigation
- Maturity: Practitioner
- Classification: baseline

## Statement

Personnel with cluster-admin, GPU resource management, or other elevated Kubernetes access must complete security awareness training covering social engineering, credential phishing (kubeconfig theft, cloud console credential harvesting, CI/CD token exfiltration), and operational security practices specific to Kubernetes administration.

## Objective

Reduce the risk of credential compromise through social engineering by ensuring that personnel with the highest-impact cluster access are trained to recognize and resist phishing, pretexting, and credential harvesting attacks targeting Kubernetes administrators.

## Rationale

Kubernetes administrators and GPU resource managers hold credentials that provide broad cluster access and control over expensive compute resources. These personnel are high-value targets for social engineering attacks including phishing emails that mimic cluster alerts, fake kubeconfig distribution, cloud console credential harvesting through OAuth consent phishing, and CI/CD token exfiltration through malicious pipeline configurations. Standard corporate security awareness training does not cover these Kubernetes-specific attack vectors.

## Scope

Applies to all personnel with cluster-admin ClusterRoleBinding, GPU ResourceQuota management permissions, cloud provider IAM admin roles for Kubernetes clusters, and CI/CD pipeline credentials that can deploy to production clusters. Training must be completed upon role assignment and refreshed annually.

## Enforcement

- Primary Enforcement Point: Security awareness training platform with Kubernetes-specific modules, training completion tracking tied to access provisioning workflows, and periodic phishing simulations targeting Kubernetes administrators.
- Implementation Surface: Security awareness training platforms with Kubernetes-specific modules covering kubeconfig theft, OAuth consent phishing for cloud console sessions, fake kubectl plugin distribution, CI/CD token exfiltration via malicious pipeline configurations, GPU quota approval social engineering, training completion tracking integrated with access provisioning workflows (cluster-admin and GPU ResourceQuota management access gating), and targeted phishing simulations for Kubernetes administrators.
- Owner Primary: Security Engineering
- Owner Supporting: Identity and Access Management; People Operations
