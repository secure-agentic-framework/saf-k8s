# SAF-K8S-0104-022 - Non-default feature gate inventory for production clusters

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.4 - CIS Benchmarks and Patch Management
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters must maintain an inventory of all non-default feature gates enabled or explicitly configured across production control plane and node components.

## Objective

Maintain authoritative visibility into all non-default Kubernetes feature gate usage in production so governance, upgrade, and risk review decisions start from a complete baseline.

## Rationale

Feature gate governance fails when organizations do not know which non-default gates are enabled or configured across production clusters. A current inventory is the prerequisite for approval review, upgrade transition tracking, and supportability decisions.

## Scope

Applies to all production clusters and all customer- or operator-configurable feature gates on API server, controller-manager, scheduler, kubelet, kube-proxy, and managed service surfaces where feature gate behavior can be influenced or must be tracked.

## Enforcement

- Primary Enforcement Point: Cluster provisioning, configuration inventory, and change tracking processes for production feature gate settings.
- Implementation Surface: Feature gate inventory records, cluster configuration reconciliation, provisioning templates, and scans of configured non-default feature gates across production control plane and node components.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Architecture Review
