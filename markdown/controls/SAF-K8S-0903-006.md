# SAF-K8S-0903-006 - Inference request queue priority, timeout, and depth controls

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.3 - Inference Resilience, Adversarial Defense, and Resource Controls
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Inference request queues must enforce priority controls, timeout settings, and queue depth limits to prevent backpressure attacks from blocking legitimate inference traffic.

## Objective

Protect inference availability through hardened request queue management that constrains backlog growth, prioritizes legitimate work, and expires abusive or stalled requests.

## Rationale

Without queue priority, depth ceilings, and enforced request timeouts, an attacker can fill the inference backlog with slow or malformed requests, creating backpressure that degrades service for legitimate users and can amplify resource exhaustion.

## Scope

Applies to inference-serving request queues, admission backlogs, and asynchronous work queues in Kubernetes-based AI serving platforms. Covers queue priority, timeout enforcement, and maximum depth controls. Excludes autoscaling bounds and cost-governed scale-out, which are covered by SAF-K8S-0903-005.

## Enforcement

- Primary Enforcement Point: Inference server queue configuration, API gateway admission policy, and backpressure control settings.
- Implementation Surface: Inference queue implementations with bounded depth, per-request timeout enforcement, priority queueing or class-of-service controls, and degraded-service handling that sheds excess load rather than allowing unbounded backlog accumulation.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Platform Engineering; FinOps
