# SAF-K8S-0405-018 - Secure credential storage and lifecycle governance

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.5 - Identity Abuse Detection and Mitigation
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must enforce secure credential storage and credential lifecycle handling requirements for Kubernetes cluster and AI infrastructure authentication pathways.

## Objective

Ensure authentication credentials are protected at rest and governed through secure lifecycle handling so theft, reuse, or unmanaged persistence do not become initial access paths into the cluster.

## Rationale

Secure storage and lifecycle handling are independently auditable from baseline credential strength policy. This control isolates the protection and lifecycle governance claim so teams can verify that credentials are stored securely and handled under controlled lifecycle standards.

## Scope

Applies to authentication pathways for Kubernetes clusters, OIDC integrations, and cloud provider IAM integrations that rely on managed credentials. Covers secure storage controls, password or secret storage protections, certificate-based authentication policy baselines, and administrative credential lifecycle standards. Baseline credential policy requirements remain governed by SAF-K8S-0405-017.

## Enforcement

- Primary Enforcement Point: Credential storage and lifecycle controls in identity providers, IAM systems, and cluster authentication systems.
- Implementation Surface: Password and secret storage protections, certificate-based authentication handling, and administrative credential lifecycle standards in OIDC providers, cloud IAM, and related authentication systems.
- Owner Primary: Identity and Access Management
- Owner Supporting: Platform Engineering; Security Engineering
