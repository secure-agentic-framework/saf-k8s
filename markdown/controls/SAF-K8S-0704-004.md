# SAF-K8S-0704-004 - IMDSv2 enforcement on Kubernetes nodes

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.4 - Cloud Provider Security Integration
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations must enforce session-based metadata access (IMDSv2 or cloud-equivalent) on all Kubernetes nodes by configuring node launch settings so metadata requests require session-oriented tokens rather than simple unauthenticated metadata GET access.

## Objective

Prevent SSRF-based credential theft from cloud metadata services by requiring session-based metadata access controls on every Kubernetes node.

## Rationale

Metadata services that allow simple unauthenticated requests are a common credential-theft path for compromised workloads and lateral movement. Requiring IMDSv2 or equivalent session-oriented access hardens the node-side metadata surface independently of separate network-level metadata blocking and per-workload cloud identity mechanisms.

## Scope

Applies to all Kubernetes nodes on cloud providers that expose an instance metadata service. Covers node launch template, node pool, or VM scale set settings that enforce session-based metadata access. Does not cover pod egress blocking or per-workload cloud identity binding, which are addressed separately.

## Enforcement

- Primary Enforcement Point: Node Launch Configuration for Metadata Service Session Enforcement
- Implementation Surface: AWS EC2 launch templates with HttpTokens required and appropriate hop-limit settings, GKE node pool metadata protections, Azure VMSS metadata security configuration, and validation scans across all node pools.
- Owner Primary: Cloud Platform Engineering
- Owner Supporting: Platform Engineering; Security Engineering
