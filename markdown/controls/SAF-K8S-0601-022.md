# SAF-K8S-0601-022 - Kubernetes image pull secret distribution and external secret integration

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Image pull credentials for authenticated registries must be distributed through Kubernetes Secrets or synchronized from an approved external secrets manager rather than embedded directly in workloads, Dockerfiles, or CI environment variables.

## Objective

Prevent unauthorized registry pulls and uncontrolled credential sprawl by managing image pull credential distribution through approved secret delivery paths.

## Rationale

How image pull credentials are stored and delivered is an independently auditable control claim from rotation. Centralized secret distribution through Kubernetes Secrets or external secret integrations prevents ad hoc credential embedding and supports controlled access to registry credentials.

## Scope

Applies to Kubernetes clusters and CI/CD workflows that use authenticated registries for pulling images. Covers image pull secret storage, synchronization from external secret managers, and automatic credential rotation.

## Enforcement

- Primary Enforcement Point: Kubernetes cluster image pull secret distribution and external secret synchronization configuration.
- Implementation Surface: Image pull secret resources, External Secrets Operator or equivalent integration, secret synchronization workflows, service account references, and controls preventing embedded registry credentials in build or deployment configuration.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
