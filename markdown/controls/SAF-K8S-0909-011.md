# SAF-K8S-0909-011 - Retrieved context integrity validation and relevance threshold enforcement

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.9 - RAG Infrastructure Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

RAG retrieval services must validate retrieved context before prompt assembly by enforcing chunk metadata integrity checks, source attribution or provenance verification, retrieval audit logging, and relevance thresholds that prevent low-confidence or malformed results from entering the LLM prompt.

## Objective

Ensure that only high-integrity, sufficiently relevant retrieved context reaches prompt assembly in RAG systems.

## Rationale

Retrieval integrity failures do not always require a poisoned index; weak chunk validation, absent provenance checks, or permissive similarity thresholds can allow malformed, stale, or low-confidence content into the prompt even when the vector index itself has not been maliciously altered.

## Scope

Applies to RAG retrieval services on Kubernetes. Covers chunk metadata validation, source attribution or provenance verification at retrieval time, similarity threshold enforcement, top-K result limiting, and retrieval audit logging. Excludes vector index monitoring and incident response for poisoning events.

## Enforcement

- Primary Enforcement Point: Retrieval service context validation and relevance-threshold configuration before prompt assembly.
- Implementation Surface: Retrieval service similarity-threshold configuration, chunk metadata validation logic, source attribution or provenance verification checks, top-K result limiting, and retrieval audit logging for prompt assembly.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Security Operations
