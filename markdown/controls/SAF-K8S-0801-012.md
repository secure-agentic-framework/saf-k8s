# SAF-K8S-0801-012 - MPS and time-slicing workload eligibility and same-trust co-location enforcement

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Advanced
- Classification: ai-specific

## Statement

Clusters using NVIDIA MPS or GPU time-slicing must restrict shared-GPU use to approved workload classes and enforce same-trust co-location and scheduling controls so incompatible or sensitive workloads cannot share weaker GPU contexts.

## Objective

Prevent incompatible workloads from sharing weaker GPU contexts by enforcing workload eligibility, same-trust co-location, and scheduling restrictions for MPS or time-sliced GPU pools.

## Rationale

Where MPS or time-slicing is permitted, the primary protection is not hardware isolation but limiting which workloads may share the GPU. Admission rules, node affinity, taints, and pool-specific scheduling constraints are required to keep different-trust or sensitive workloads off shared GPU contexts.

## Scope

Applies to Kubernetes clusters using NVIDIA MPS or GPU time-slicing for shared GPU allocation. Covers workload eligibility rules, same-trust co-location constraints, and pool-specific scheduling or admission enforcement for shared GPU modes.

## Enforcement

- Primary Enforcement Point: Shared-GPU Eligibility and Co-Location Enforcement
- Implementation Surface: Admission control policies, GPU Operator time-slicing configuration, NVIDIA MPS usage restrictions, node affinity, taints, and scheduling policies enforcing approved workload classes and same-trust placement for shared GPU contexts.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Data Protection Office
