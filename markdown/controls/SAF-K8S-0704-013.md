# SAF-K8S-0704-013 - Unified cloud and Kubernetes audit source onboarding

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.4 - Cloud Provider Security Integration
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must forward cloud provider audit logs and Kubernetes audit events into the same security monitoring platform so both telemetry sources are centrally available for managed Kubernetes security operations and investigations.

## Objective

Ensure cloud and Kubernetes audit telemetry is centrally available in one monitoring plane for managed cluster security operations.

## Rationale

Managed Kubernetes investigations fail when cloud-provider and Kubernetes audit records are collected into separate systems or are not jointly accessible to defenders. Centralizing both audit streams provides the common monitoring plane needed for downstream detection, triage, and investigation across the shared-responsibility boundary.

## Scope

Applies to all managed Kubernetes clusters on cloud providers including EKS, GKE, and AKS. Cloud audit logs and Kubernetes audit events must be forwarded to the same SIEM or log analytics platform used for managed cluster security operations.

## Enforcement

- Primary Enforcement Point: Cloud audit log forwarding configuration, Kubernetes audit log forwarding, and monitoring platform onboarding.
- Implementation Surface: Cloud provider audit log services, Kubernetes audit log pipelines, SIEM or log analytics platform ingestion connectors, and data-source health monitoring for audit ingestion.
- Owner Primary: Security Engineering
- Owner Supporting: Cloud Platform Engineering; Platform Engineering; SOC
