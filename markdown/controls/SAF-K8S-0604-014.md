# SAF-K8S-0604-014 - Source-to-artifact integrity linkage for built images

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.4 - SBOMs and Vulnerability Intelligence
- Maturity: Practitioner
- Classification: baseline

## Statement

Build pipelines must maintain source-to-artifact integrity links tying each deployed image to its exact source commit, dependency lock files, and build configuration.

## Objective

Preserve an auditable chain from source and dependency state to the produced artifact so investigators and reviewers can prove exactly what produced each deployed image.

## Rationale

Hermetic builds and pinned inputs reduce tampering opportunities, but organizations still need a durable linkage from source state and build configuration to the resulting artifact digest. Source-to-artifact integrity records provide that traceability and make downstream review reproducible.

## Scope

Applies to all CI/CD pipelines producing deployable Kubernetes artifacts, including application images, AI workload images, and infrastructure operator images.

## Enforcement

- Primary Enforcement Point: Build provenance capture and artifact metadata binding at image production time.
- Implementation Surface: Source commit capture, dependency lock file references, build configuration records, and integrity metadata linking those inputs to produced image digests or equivalent artifact identifiers.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Platform Engineering
