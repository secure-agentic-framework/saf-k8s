# SAF-K8S-0908-008 - Inference output perturbation and privacy-preserving response shaping

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.8 - Model Abuse and Extraction Prevention
- Maturity: Advanced
- Classification: ai-specific

## Statement

Inference endpoints must apply calibrated output perturbation or response shaping controls to reduce information leakage from model responses, including perturbation of confidence-bearing outputs and other response elements that would otherwise facilitate model extraction, inversion, or membership inference attacks.

## Objective

Reduce exploitable information leakage from inference responses by shaping or perturbing what the endpoint returns to untrusted callers.

## Rationale

Some inference privacy protections rely on response shaping rather than formal differential privacy accounting: limiting confidence disclosure, perturbing sensitive scores, and reducing high-fidelity output features that enable repeated extraction or inversion. These controls are distinct from privacy budget governance and must be implemented and validated at the response layer.

## Scope

Applies to inference endpoints on Kubernetes where model responses expose confidence scores, probabilities, rankings, or other detailed outputs that increase disclosure risk. Covers output perturbation configuration and utility-impact monitoring. Excludes formal differential privacy budget accounting and secure aggregation for aggregate queries.

## Enforcement

- Primary Enforcement Point: Inference server response transformation and output perturbation configuration.
- Implementation Surface: Calibrated noise addition to confidence scores or probability distributions, response shaping rules that suppress or coarsen sensitive output features, LLM output-level perturbation settings where applicable, and monitoring of utility degradation caused by privacy-preserving response transformations.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Privacy Engineering
