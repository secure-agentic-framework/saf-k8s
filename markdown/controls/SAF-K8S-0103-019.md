# SAF-K8S-0103-019 - API server load-balancer health-check failover

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters must expose the API server through a load balancer with automated health-check failover so control-plane endpoint reachability does not depend on a single node or static upstream target.

## Objective

Maintain API server availability during node or component failures by enforcing automated endpoint failover for the control-plane access path.

## Rationale

A multi-replica control plane still becomes unavailable if clients depend on a single API server endpoint or if upstream failover is manual. Requiring health-checked load-balancer failover ensures clients are redirected away from failed API server instances quickly enough to preserve control-plane access during outages.

## Scope

Applies to production clusters. Covers API server endpoint load-balancing, upstream health checks, and automatic failover behavior. For managed Kubernetes, verify provider-managed API endpoint failover architecture and SLA claims.

## Enforcement

- Primary Enforcement Point: API endpoint failover architecture
- Implementation Surface: Load balancer health checks, upstream pool configuration, API endpoint failover behavior, and availability SLO dashboards or failover validation evidence.
- Owner Primary: Platform Engineering
- Owner Supporting: Site Reliability Engineering; Security Engineering
