# SAF-K8S-0801-010 - Admission validation of authorized GPU resource requests

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Admission control policies must validate GPU resource requests against authorized workload scope, namespace quotas, and approved accelerator profiles to prevent unauthorized or excessive GPU allocation.

## Objective

Ensure only authorized workloads receive GPU resources and that requested accelerator quantities or profiles remain within approved limits.

## Rationale

Without admission validation, workloads can request accelerators beyond their authorization, manipulate placement, or bypass intended resource governance. Policy-driven request validation prevents unauthorized consumption and reduces opportunities for adversarial co-location or resource abuse.

## Scope

Applies to all Kubernetes clusters that schedule GPU workloads, including managed services where accelerator request validation is enforced through admission controls or equivalent policy gates.

## Enforcement

- Primary Enforcement Point: Admission policies validating GPU resource requests before scheduling.
- Implementation Surface: OPA Gatekeeper, Kyverno, or equivalent admission controls validating GPU counts, MIG profiles, workload authorization, and namespace-level accelerator quotas.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Platform Engineering; Security Engineering
