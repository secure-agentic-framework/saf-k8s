# SAF-K8S-0804-001 - TEE-based model and data protection for AI workloads

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.4 - Confidential Computing for AI Workloads
- Maturity: Advanced
- Classification: ai-specific

## Statement

AI workloads processing sensitive models or data must use hardware Trusted Execution Environment (TEE) capabilities (NVIDIA H100 Confidential Computing, Intel TDX, or AMD SEV-SNP) where available and operationally feasible, with TEE configurations validated for both inference and training use cases and memory encryption verified as active.

## Objective

Protect AI model intellectual property and sensitive training or inference data from unauthorized access by infrastructure operators, co-tenants, or compromised hypervisors through hardware-enforced memory encryption and isolation provided by TEEs.

## Rationale

Standard GPU workloads expose model weights and data in plaintext GPU memory, accessible to anyone with privileged host access including cloud operators, co-tenant attackers, and compromised hypervisors. TEE-enabled GPU computing encrypts GPU memory contents in hardware, ensuring that even with physical access or hypervisor compromise, model weights and training data remain confidential. Without TEE protection, regulated data processed by AI workloads may fail to meet data sovereignty and confidentiality compliance requirements.

## Scope

Applies to Kubernetes clusters with TEE-capable hardware (NVIDIA H100/H200 with Confidential Computing mode, Intel TDX-enabled CPUs, AMD SEV-SNP-enabled CPUs) processing sensitive AI workloads. Workloads on non-TEE-capable hardware must document compensating controls. This is an emerging control; organizations must evaluate readiness and plan adoption timelines.

## Enforcement

- Primary Enforcement Point: GPU Confidential Computing mode enablement in BIOS/firmware, TEE-aware Kubernetes device plugin configuration, confidential VM or container runtime settings, and admission control policies requiring TEE for classified workloads.
- Implementation Surface: NVIDIA H100/H200 Confidential Computing mode BIOS/firmware enablement, nvidia-smi conf-compute verification tooling, GPU Operator Confidential Computing configuration, Intel TDX and AMD SEV-SNP VM configuration for Kubernetes nodes, TEE-aware device plugin settings, confidential VM or container runtime (Kata Containers with CC) settings, node taints and labels for CC-enabled node identification, and admission control policies requiring TEE for classified workload scheduling.
- Owner Primary: ML Platform Engineering
- Owner Supporting: GPU Infrastructure Engineering; Security Engineering; Compliance
