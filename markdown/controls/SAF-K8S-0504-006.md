# SAF-K8S-0504-006 - SPIFFE/SPIRE workload identity issuance and lifecycle management

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.4 - Zero Trust Architecture and Service Mesh
- Maturity: Advanced
- Classification: baseline

## Statement

Workload identity must be established using SPIFFE-compliant identities issued by SPIRE or an equivalent workload identity provider, with workload registration, attestation, issuance, and automatic identity rotation configured to maintain verifiable service identity over time.

## Objective

Provide cryptographically verifiable workload identities that enable identity-based access control independent of network location.

## Rationale

Without cryptographic workload identity issuance and rotation, services fall back to network-based trust that can be spoofed or bypassed. Separating identity lifecycle management from federation governance makes local workload identity controls independently auditable.

## Scope

Applies to all production Kubernetes clusters implementing zero trust networking. SPIRE server and agent deployments must follow high-availability patterns. Trust domain naming must align with organizational boundaries. Cross-cluster federation applies where workloads in separate clusters need to authenticate to each other.

## Enforcement

- Primary Enforcement Point: SPIFFE/SPIRE workload identity issuance and attestation controls.
- Implementation Surface: SPIRE server and agent deployments, workload registration entries, workload attestation plugins, SVID issuance, automatic SVID renewal, and integration with service mesh or application-level mTLS for identity-based access control.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Network Engineering
