# SAF-K8S-0202-002 - RuntimeClass configuration for workload-appropriate isolation

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.2 - Container Runtime Security
- Maturity: Foundational
- Classification: ai-specific

## Statement

Clusters must define RuntimeClass resources that map workloads to appropriate container runtime handlers, ensuring that security-sensitive or AI/ML workloads use sandboxed runtimes (gVisor, Kata Containers) where required, and that GPU-accelerated workloads use validated GPU-compatible runtime configurations.

## Objective

Enable workload-specific runtime isolation by assigning appropriate container runtime sandboxing levels based on workload risk profile, with particular attention to AI/ML workloads requiring GPU access.

## Rationale

A single default runtime provides uniform isolation regardless of workload risk. Untrusted or multi-tenant AI workloads sharing a standard runc runtime can exploit kernel vulnerabilities to escape container boundaries. Without RuntimeClass configuration, GPU-accelerated workloads may require overly permissive security contexts to function, or sandboxed runtimes may block legitimate GPU device access. Explicit RuntimeClass mapping ensures each workload receives the appropriate isolation level.

## Scope

Applies to all production clusters running mixed workload types or AI/ML workloads requiring GPU access. Clusters running exclusively trusted first-party workloads with homogeneous isolation requirements may document a single-runtime exception. Managed services must configure available RuntimeClass options or document provider-enforced isolation equivalents.

## Enforcement

- Primary Enforcement Point: Kubernetes RuntimeClass resource definitions and admission policies enforcing RuntimeClass assignment for designated namespaces or workload labels.
- Implementation Surface: Kubernetes RuntimeClass API resource definitions, runtime handler configuration (nvidia, runsc, kata), node selectors for GPU scheduling, GPU compatibility validation matrix (runtime/driver/CUDA versions), and admission policies enforcing RuntimeClass selection.
- Owner Primary: Platform Engineering
- Owner Supporting: AI/ML Platform; Security Engineering
