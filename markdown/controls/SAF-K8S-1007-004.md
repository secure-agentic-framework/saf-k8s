# SAF-K8S-1007-004 - Continuous security posture management for AI clusters

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.7 - Cluster Lifecycle and Asset Inventory
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must implement continuous security posture assessment for Kubernetes AI clusters, monitoring configuration drift against security benchmarks, tracking vulnerability exposure, and evaluating compliance status, to detect security regression before the next periodic audit.

## Objective

Maintain a continuously accurate view of Kubernetes AI cluster security posture by detecting configuration drift, vulnerability exposure, and compliance deviations in near real-time.

## Rationale

Kubernetes cluster configurations drift over time through manual changes, operator updates, and scaling events. Without continuous posture assessment, drift from security benchmarks goes undetected until the next periodic audit, leaving windows of vulnerability.

## Scope

Applies to all Kubernetes clusters hosting AI workloads. Configuration drift monitoring must cover CIS Kubernetes Benchmark controls and AI-specific security configurations. Managed Kubernetes services must monitor customer-configurable security settings. GPU scaling governance is covered by SAF-K8S-1007-008.

## Enforcement

- Primary Enforcement Point: Kubernetes security posture management tooling with daily benchmark scanning and change detection alerting.
- Implementation Surface: CIS Kubernetes Benchmark scanning (kube-bench) on daily schedule with regression alerting, admission controller and policy engine configuration change monitoring (validating/mutating webhook, OPA constraint, Kyverno policy changes), NetworkPolicy change detection and alerting on access-widening modifications, RBAC change monitoring (ClusterRoleBindings, RoleBindings, ServiceAccount token creation), container image vulnerability scanning integration with posture dashboard, GPU driver and CUDA toolkit CVE exposure tracking against NVIDIA security bulletins, and vulnerability-to-asset-criticality correlation.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; AI/ML Engineering
