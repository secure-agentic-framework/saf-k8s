# SAF-K8S-0201-008 - Kubelet configuration and credential file ownership and permissions

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.1 - Kubelet and Node Configuration Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubelet configuration files, kubeconfig files, and certificate key material on each node must be owned by root with permissions no more permissive than 0644 for configuration files and 0600 for private keys so that unauthorized local users or escaped workloads cannot read or modify kubelet configuration or credential material.

## Objective

Protect kubelet configuration, kubeconfig material, and node credential files from unauthorized local access or tampering through restrictive ownership and permission settings.

## Rationale

Kubelet configuration and credential files contain authentication material, certificate references, and runtime parameters that control node behavior. This control isolates filesystem protection so teams can independently verify that kubelet-related files are not exposed to local attackers or compromised workloads.

## Scope

Applies to all files in /etc/kubernetes/, /var/lib/kubelet/, and kubelet certificate directories on every cluster node. Managed Kubernetes services where the node filesystem is provider-controlled satisfy this control through provider hardening attestation.

## Enforcement

- Primary Enforcement Point: Node image build pipeline, configuration management tooling, and periodic file permission audit scans.
- Implementation Surface: Node image build pipeline, configuration management tooling, file permission auditing, and CIS Benchmark scanning for kubelet file ownership and mode checks.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; OS/Infrastructure
