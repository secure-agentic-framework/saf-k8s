# SAF-K8S-0303-004 - MAC profile generation from runtime behavior

- Domain: D03 - Workload and Pod Security
- Knowledge Area: 3.3 - Mandatory Access Controls
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must generate seccomp, AppArmor, or SELinux profiles for workloads using runtime recording tools that observe actual syscalls, file accesses, and network operations, rather than writing profiles by hand, to produce accurate, workload-specific profiles.

## Objective

Produce workload-specific MAC profiles derived from actual runtime behavior rather than manual specification, ensuring profiles accurately reflect the full set of legitimate operations the workload performs.

## Rationale

Hand-written MAC profiles are frequently either too permissive (providing little security value) or too restrictive (breaking workload functionality). Runtime profiling tools such as Security Profiles Operator ProfileRecording, Inspektor Gadget, aa-logprof, and audit2allow observe actual syscalls, file accesses, and network operations during workload execution to generate accurate profiles. For GPU workloads, profiling must capture model load, training epochs, checkpoint saves, and inference paths, since GPU ioctl patterns differ between CUDA versions and driver versions.

## Scope

Applies to all workloads for which custom MAC profiles are being created or updated. Runtime profiling must be performed in an environment that exercises the full range of workload behavior including normal operation, error handling, startup, and shutdown.

## Enforcement

- Primary Enforcement Point: Profile generation tooling (Security Profiles Operator ProfileRecording CRD, Inspektor Gadget, aa-logprof, audit2allow) used during the profiling phase before profile deployment.
- Implementation Surface: Security Profiles Operator (SPO) ProfileRecording CRD for automated seccomp profile capture, Inspektor Gadget for runtime syscall tracing, aa-logprof for AppArmor profile generation from audit logs, audit2allow for SELinux policy generation, GPU driver version-matched profiling environments for AI workloads, and version-controlled profile files stored alongside application manifests.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Application Development teams
