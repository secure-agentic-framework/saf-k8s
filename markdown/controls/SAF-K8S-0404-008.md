# SAF-K8S-0404-008 - Certificate expiry monitoring and alerting

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.4 - Certificate Management
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must implement monitoring and alerting for certificates approaching defined expiration thresholds across cluster and workload PKI assets.

## Objective

Provide actionable visibility into certificates nearing expiry so renewal failures can be corrected before service impact.

## Rationale

Expiry telemetry is independently auditable from the renewal mechanism itself. This control isolates dashboards, age telemetry, and alerting so teams can verify that impending expirations are visible and routed before outage windows are reached.

## Scope

Applies to API server, etcd peer, kubelet, webhook, service mesh, and application certificates managed in the cluster. Covers expiry thresholds, dashboards, telemetry collection, and alert routing. Renewal execution remains governed by SAF-K8S-0404-007.

## Enforcement

- Primary Enforcement Point: Certificate age telemetry and alerting for cluster PKI assets.
- Implementation Surface: Expiry dashboards, Prometheus or scanner-based certificate age telemetry, and alerting integrations for certificates nearing expiration thresholds.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Security Operations
