# SAF-K8S-0505-004 - API server audit log analysis for network-based attack detection

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.5 - API Server and Service Exposure Protection
- Maturity: Practitioner
- Classification: baseline

## Statement

API server audit logs must be continuously analyzed for network-based attack indicators including unauthorized access attempts from unexpected source IPs, anomalous API request patterns indicating reconnaissance or exploitation, and authentication failures correlated with network origin to detect credential-based attacks.

## Objective

Detect network-based attacks against the Kubernetes API server through continuous audit log analysis, enabling rapid identification of unauthorized access attempts and attack patterns.

## Rationale

API server audit logs capture every request to the control plane including source IP, authenticated identity, requested resource, and response code. Without continuous analysis, brute-force authentication attempts, reconnaissance scans of API resources, and exploitation attempts go undetected until damage occurs. Correlating authentication failures with source network origin enables detection of distributed credential attacks and identification of compromised network segments.

## Scope

Applies to all production Kubernetes clusters with audit logging enabled. Audit logs must be forwarded to a centralized SIEM or log analysis platform in near real-time. Detection rules must cover authentication failures, access from unauthorized networks, and anomalous API request patterns. Managed Kubernetes audit log delivery mechanisms must be configured and validated.

## Enforcement

- Primary Enforcement Point: API server audit policy configuration, log forwarding pipeline to SIEM, and detection rule library covering network-based attack patterns.
- Implementation Surface: Kubernetes API server audit policy configuration (RequestResponse and Metadata logging levels), audit log forwarding pipelines to SIEM platforms (GKE Cloud Audit Logs, EKS CloudWatch log groups, Fluentd/Fluent Bit forwarders), SIEM detection rules for unauthorized source IPs, brute-force authentication (401 response spikes), API reconnaissance (multi-resource list operations), service account token usage from external IPs, and AI-specific detection rules for bulk Secret reads, GPU resource enumeration (nvidia.com/gpu), training job manipulation, and inference Deployment modification from unknown sources.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; Network Engineering
