# SAF-K8S-0502-010 - Kubernetes CNI IP pool exhaustion monitoring and alerting

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.2 - CNI Plugins and Pod Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

IP address management for Kubernetes networking must be monitored to detect impending pod, service, or node IP pool exhaustion before scheduling failures or service outages occur.

## Objective

Detect impending IP pool exhaustion early enough to prevent availability impact from depleted pod, service, or node address ranges.

## Rationale

Even correctly sized IP pools can be depleted over time by growth or abnormal consumption. Separate monitoring and alerting from sizing so exhaustion detection and response can be independently audited.

## Scope

Applies to all Kubernetes clusters. Covers pod CIDR, service CIDR, and node CIDR range sizing, utilization monitoring, exhaustion thresholds, and alerting for IP address pool depletion.

## Enforcement

- Primary Enforcement Point: CNI plugin IP pool utilization monitoring and alerting.
- Implementation Surface: IP pool utilization monitoring, exhaustion thresholds, alerting for pod, service, and node CIDR depletion, and operational review of capacity trend signals.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Security Operations
