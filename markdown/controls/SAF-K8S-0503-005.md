# SAF-K8S-0503-005 - DNS exfiltration detection

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.3 - Ingress, Egress, and DNS Hardening
- Maturity: Practitioner
- Classification: baseline

## Statement

DNS query logging and anomaly detection must be enabled to detect and prevent DNS-based data exfiltration, including detection of DNS tunneling indicators such as high-entropy subdomain labels, high-frequency queries, and anomalous TXT record queries.

## Objective

Detect and prevent covert data exfiltration channels that abuse the DNS protocol, which is rarely blocked and often overlooked in monitoring.

## Rationale

DNS exfiltration is particularly dangerous because DNS traffic is rarely blocked and often overlooked in monitoring. Attackers encode stolen data in DNS queries (long subdomain labels, TXT record queries) to bypass network egress controls. Without DNS query logging and anomaly detection, these covert channels operate undetected.

## Scope

Applies to all production clusters with CoreDNS. DNS query logs must be forwarded to a centralized analysis platform. Detection rules must cover DNS tunneling indicators including high-entropy subdomain labels, anomalous query volume, and unusual record types.

## Enforcement

- Primary Enforcement Point: DNS query log analysis pipelines and SIEM detection rules for DNS tunneling indicators.
- Implementation Surface: CoreDNS log plugin for query logging, DNS query log analysis pipelines, SIEM detection rules for DNS tunneling indicators (long subdomain labels, high-frequency queries, anomalous TXT record queries), Cilium DNS proxy logging, and DNS exfiltration anomaly detection tooling.
- Owner Primary: Security Operations
- Owner Supporting: Network Engineering; Platform Engineering
