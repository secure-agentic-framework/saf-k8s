# SAF-K8S-0205-007 - Node firewall compatibility validation with CNI and network policy

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.5 - kube-proxy and Node Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Node-level firewall rules must be validated for compatibility with the cluster CNI plugin's overlay and network policy enforcement requirements so that node firewalls do not create conflicts, blind spots, or broken cluster communication paths.

## Objective

Ensure node firewalls complement rather than disrupt CNI overlays and Kubernetes network policy enforcement.

## Rationale

Improperly configured node firewalls can break CNI overlays, bypass intended policy paths, or create gaps between node and pod-layer enforcement. This control isolates compatibility validation so teams can independently prove that firewall rules and CNI requirements are aligned.

## Scope

Applies to all production cluster nodes using CNI plugins or cloud networking models with specific overlay, encapsulation, or enforcement path requirements such as VXLAN, Geneve, or WireGuard.

## Enforcement

- Primary Enforcement Point: Compatibility validation between node firewall layer and pod-network enforcement layer.
- Implementation Surface: CNI port allowance configuration, documented compatibility validation, and network connectivity tests confirming legitimate cluster communication functions correctly.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Cloud Infrastructure; Security Engineering
