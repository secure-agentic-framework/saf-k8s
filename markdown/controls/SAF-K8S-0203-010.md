# SAF-K8S-0203-010 - Minimal purpose-built node OS baseline

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.3 - Host OS and Kernel Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes node host operating systems must use minimal, purpose-built OS distributions or hardened minimal installations with unnecessary packages, services, and user accounts removed.

## Objective

Minimize host OS attack surface by eliminating unnecessary software components and post-exploitation utilities from Kubernetes nodes.

## Rationale

General-purpose operating systems include package managers, shell utilities, compilers, and services that are unnecessary on Kubernetes nodes and provide tooling for post-exploitation activity. A minimal purpose-built node OS is independently auditable from immutability because it governs what software and services are present on the host image at baseline.

## Scope

Applies to all production cluster node OS images. Cloud-managed node pools must use the provider's hardened or minimal node image options. Custom node images must document their base image, removed packages, and hardening steps. Development clusters may use standard OS images with documented exceptions.

## Enforcement

- Primary Enforcement Point: Node OS image build pipeline, base image selection, package and service removal standards, and OS image compliance scanning.
- Implementation Surface: Minimal OS distributions such as Bottlerocket, Flatcar, Talos, Ubuntu Minimal, or RHEL CoreOS, plus node image manifests, package inventories, and service enumeration controls.
- Owner Primary: Platform Engineering
- Owner Supporting: OS/Infrastructure; Security Engineering
