# SAF-K8S-0703-023 - Investigation and termination of confirmed unauthorized GPU workloads

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.3 - Resource Governance and Priority
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must investigate confirmed abusive GPU consumption and terminate, quarantine, or otherwise stop non-sanctioned GPU workloads through approved incident response actions that document disposition and preserve evidence needed for follow-up analysis.

## Objective

Respond to confirmed unauthorized GPU consumption quickly enough to halt abusive usage, preserve evidence, and determine whether the event reflects simple misuse or a broader security incident.

## Rationale

Detection alone does not reclaim accelerator capacity or resolve the security risk created by confirmed abuse. A separate response control is required so organizations have governed workflows to investigate, terminate, or quarantine abusive GPU workloads while preserving the evidence needed to understand impact and recurrence.

## Scope

Applies to all confirmed abusive GPU consumption events in Kubernetes clusters, including cryptomining, personal workloads, resource squatting that persists after validation, and other unauthorized GPU use requiring response action.

## Enforcement

- Primary Enforcement Point: Incident triage, containment, and workload termination or quarantine workflows for confirmed GPU abuse.
- Implementation Surface: Abuse investigation tickets, termination or quarantine workflows, evidence capture steps, and approved incident response procedures for unauthorized GPU workloads.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; ML Engineering
