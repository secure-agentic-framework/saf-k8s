# SAF-K8S-0606-009 - SSDF v1.1 alignment for secure development practices

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Development practices must align with NIST SSDF v1.1 (Secure Software Development Framework) task groups (Prepare the Organization, Protect the Software, Produce Well-Secured Software, Respond to Vulnerabilities) for secure development lifecycle requirements. Organizations must maintain documented SSDF task mappings and audit evidence demonstrating alignment.

## Objective

Ensure secure development practices are systematic and auditable by aligning with the NIST SSDF v1.1 framework rather than relying on ad hoc practices.

## Rationale

Without SSDF alignment, secure development practices are ad hoc and inconsistent across teams. SSDF v1.1 provides a structured framework that maps development activities to security outcomes, enabling organizations to identify gaps, demonstrate compliance to auditors, and progressively improve their development security posture. The framework is particularly relevant for organizations subject to federal supply chain security requirements.

## Scope

Applies to all software development practices across the organization producing artifacts deployed to Kubernetes clusters. Covers all four SSDF task groups (PS, PW, PO, RV). Organizations not subject to federal requirements may use SSDF as a voluntary maturity benchmark.

## Enforcement

- Primary Enforcement Point: Development governance and compliance review processes.
- Implementation Surface: SSDF v1.1 task mapping (PS - Prepare the Organization, PW - Protect the Software, PO - Produce Well-Secured Software, RV - Respond to Vulnerabilities) for secure development lifecycle audit evidence, organizational self-assessment against SSDF tasks, gap analysis and remediation tracking, and periodic compliance review documentation.
- Owner Primary: Compliance / GRC
- Owner Supporting: DevSecOps / Release Engineering; Security Engineering
