# SAF-K8S-0504-007 - SPIFFE trust domain scoping and cross-cluster federation governance

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.4 - Zero Trust Architecture and Service Mesh
- Maturity: Advanced
- Classification: baseline

## Statement

SPIFFE trust domains must be explicitly configured and scoped to organizational boundaries, and cross-cluster workload identity federation must use explicit trust bundle exchange and enrolled federation relationships rather than implicit trust.

## Objective

Enable secure cross-cluster identity trust only where intended by governing trust-domain boundaries and explicit federation exchange.

## Rationale

Improperly scoped trust domains or implicit federation relationships allow workloads in one environment to impersonate identities from another. Separating federation governance from local identity issuance makes trust-boundary control independently auditable.

## Scope

Applies to all production Kubernetes clusters implementing zero trust networking. SPIRE server and agent deployments must follow high-availability patterns. Trust domain naming must align with organizational boundaries. Cross-cluster federation applies where workloads in separate clusters need to authenticate to each other.

## Enforcement

- Primary Enforcement Point: Trust domain and federation boundary governance.
- Implementation Surface: Trust domain configuration and naming, federation endpoint configuration, explicit trust bundle exchange, and approval of cross-cluster identity trust relationships.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Network Engineering
