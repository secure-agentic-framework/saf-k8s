# SAF-K8S-0403-030 - Environment variable secret injection exception governance

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations must maintain an exception registry for any approved environment variable secret usage with documented compensating controls and mandatory re-review dates.

## Objective

Ensure any rare approved use of environment variable secret injection is explicitly governed, time-bounded, and subject to compensating controls.

## Rationale

Exception governance is independently auditable from the enforcement of the default prohibition. This control isolates approval, compensating-control documentation, and re-review so teams can verify that deviations are rare, tracked, and actively governed.

## Scope

Applies to all approved deviations from the production prohibition on environment variable secret injection. Covers exception registry maintenance, documented compensating controls, approval metadata, and mandatory re-review dates. Default admission blocking remains governed by SAF-K8S-0403-029.

## Enforcement

- Primary Enforcement Point: Exception registry and re-review workflow for env-var secret usage.
- Implementation Surface: Exception registry entries, approval records, compensating control documentation, and re-review workflow records for approved environment-variable secret use.
- Owner Primary: Application Development
- Owner Supporting: Security Engineering; Platform Engineering
