# SAF-K8S-0603-006 - TUF-based secure software update systems

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.3 - Attestation, Provenance, and Cryptographic Assurance
- Maturity: Advanced
- Classification: baseline

## Statement

Organizations distributing software updates or artifact metadata through update systems must implement TUF (The Update Framework) with threshold and quorum trust for signing keys, HSM-based key storage for online signing roles, documented key rotation procedures, and protections against known update metadata attacks including rollback, freeze, mix-and-match, and endless data attacks.

## Objective

Secure the software update distribution channel against metadata-level attacks by implementing TUF's delegated trust model with proper key management and rotation.

## Rationale

Standard artifact signing protects individual artifacts but does not protect the metadata that describes which versions are current. Without TUF, an attacker who compromises the update metadata can serve stale (rollback), partial (mix-and-match), or indefinitely cached (freeze) artifact sets, causing clusters to run vulnerable or inconsistent versions. Threshold signing prevents a single compromised key from corrupting the entire update channel.

## Scope

Applies to organizations operating internal artifact distribution systems, Helm chart repositories, or custom update channels for Kubernetes operators and AI workload components. Does not apply to organizations exclusively consuming artifacts from third-party registries that implement their own TUF infrastructure, though verification of the third-party TUF root of trust is required.

## Enforcement

- Primary Enforcement Point: Update system metadata signing infrastructure and client-side TUF verification configuration.
- Implementation Surface: TUF (The Update Framework) metadata signing infrastructure with delegated trust roles (root, targets, snapshot, timestamp), HSM-based key storage for online signing roles (targets, snapshot, timestamp), offline HSM key storage for root keys with 3-of-5 quorum threshold signing, key rotation procedures and ceremony documentation, client-side TUF verification configuration, and protections against rollback, freeze, mix-and-match, and endless data attacks on update metadata.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; DevSecOps / Release Engineering
