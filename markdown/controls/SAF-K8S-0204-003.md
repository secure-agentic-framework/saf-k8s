# SAF-K8S-0204-003 - Container filesystem drift detection

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.4 - Runtime Threat Detection
- Maturity: Practitioner
- Classification: baseline

## Statement

Production workloads must be monitored for container filesystem drift, detecting and alerting on unexpected binary execution, library loading, or configuration file modifications that deviate from the original container image contents.

## Objective

Detect post-deployment tampering with container filesystems that may indicate malware installation, backdoor deployment, or attacker tool staging within running containers.

## Rationale

Containers are designed to be immutable after deployment; any runtime filesystem modification is inherently suspicious. Attackers who gain code execution inside a container frequently download additional tools, modify configuration files to disable security controls, or drop persistent backdoors. Without drift detection, these modifications go unnoticed even when the initial intrusion vector is eventually patched. Drift detection provides a high-signal, low-false-positive detection layer because legitimate containers rarely modify their own filesystems.

## Scope

Applies to all production workloads deployed from container images. Workloads that legitimately write to the container filesystem (e.g., applications with local caching or temporary file creation) must have drift detection rules tuned to distinguish between expected writable paths and unexpected binary or configuration modifications. Init containers and sidecar containers are also in scope.

## Enforcement

- Primary Enforcement Point: Runtime security tool configuration with container image baseline comparison and filesystem monitoring policies.
- Implementation Surface: Drift detection tool configuration (Falco, Tetragon, Sysdig drift control, Aqua drift prevention), readOnlyRootFilesystem enforcement, exclusion path definitions for legitimate writes (model cache, checkpoints, emptyDir), and binary execution alerting from writable paths.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Application Development
