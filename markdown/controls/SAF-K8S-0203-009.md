# SAF-K8S-0203-009 - Workload identity replacement for cloud API access

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.3 - Host OS and Kernel Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Cloud-hosted Kubernetes clusters must replace metadata-derived cloud credentials with workload identity federation where the platform supports it so pod workloads use dedicated short-lived identities instead of node instance credentials.

## Objective

Eliminate operational dependence on node metadata credentials by shifting workloads to dedicated workload identities for cloud API access.

## Rationale

Blocking metadata paths reduces exposure, but durable risk reduction requires removing the need for pods to use node instance credentials at all. This control isolates the workload identity replacement obligation so teams can independently verify that cloud API access uses federated pod identities rather than metadata-derived credentials.

## Scope

Applies to cloud-hosted Kubernetes clusters supporting workload identity federation features such as IRSA, EKS Pod Identity, GKE Workload Identity, and Azure Workload Identity.

## Enforcement

- Primary Enforcement Point: Workload identity configuration for pod credential issuance.
- Implementation Surface: Workload identity federation configuration replacing node instance credentials for cloud API access.
- Owner Primary: Cloud Infrastructure
- Owner Supporting: Platform Engineering; Security Engineering
