# SAF-K8S-0302-007 - Safe fsGroup and supplementalGroups volume ownership

- Domain: D03 - Workload and Pod Security
- Knowledge Area: 3.2 - Security Contexts and Capabilities
- Maturity: Foundational
- Classification: baseline

## Statement

Pods that use fsGroup or supplementalGroups must restrict group ownership changes to designated workload data directories and must not grant write access to sensitive paths, shared platform volumes, or host-backed filesystems outside the container's approved storage scope.

## Objective

Prevent filesystem group ownership settings from creating cross-workload tampering paths or unintended write access to sensitive storage.

## Rationale

fsGroup and supplementalGroups change the group ownership and write semantics of mounted volumes. Without explicit limits, those settings can grant container processes write access to shared or sensitive paths, enabling cross-tenant tampering, credential overwrite, or model and dataset corruption.

## Scope

Applies to workloads that mount persistent volumes, shared datasets, model caches, or other writable storage in production clusters.

## Enforcement

- Primary Enforcement Point: Admission policy rules and workload validation that restrict approved fsGroup and supplementalGroups use for mounted storage.
- Implementation Surface: Pod securityContext fsGroup and supplementalGroups fields, volume mount definitions, and volume permission analysis outputs.
- Owner Primary: Application Development teams
- Owner Supporting: Platform Engineering; Security Engineering
