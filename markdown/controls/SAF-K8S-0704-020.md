# SAF-K8S-0704-020 - Node-level cloud identity restriction for managed Kubernetes workloads

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.4 - Cloud Provider Security Integration
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations must restrict node-level instance profiles or managed identities to node bootstrap and infrastructure operations and must prevent pods from relying on node-scoped cloud identities for workload API access.

## Objective

Eliminate node-scoped cloud credentials as a pod access path so a compromised workload cannot inherit broad cloud permissions intended only for the Kubernetes node or platform infrastructure.

## Rationale

Node-level instance profiles and managed identities create a broad blast radius when pods can obtain or rely on them for cloud API access. Restricting those identities to node operations only and requiring workload-scoped credentials narrows authorization scope and reduces credential reuse across co-located workloads.

## Scope

Applies to managed Kubernetes worker nodes and any node-level instance profiles or managed identities that could otherwise be used by pods. Covers permission restriction, node-role review, and validation that workload access uses workload-scoped identities instead.

## Enforcement

- Primary Enforcement Point: Node-Scoped Cloud Identity Restriction
- Implementation Surface: Node instance profile or managed identity configuration, IAM permission scoping, node bootstrap role review, and validation that workloads do not use node-scoped identities for cloud API access.
- Owner Primary: Cloud Platform Engineering
- Owner Supporting: Platform Engineering; Security Engineering
