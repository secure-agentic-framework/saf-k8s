# SAF-K8S-0201-016 - Node audit log forwarding and centralized reviewability

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.1 - Kubelet and Node Configuration Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Node audit daemon settings and log forwarding must ensure kubelet access events are retained, forwarded to the centralized audit pipeline, and reviewable for investigation and alerting.

## Objective

Ensure node-level kubelet audit events reach centralized detection and investigation workflows.

## Rationale

Local audit rules are insufficient if node compromise, storage loss, or isolated logging prevents investigation. Centralized forwarding and reviewability make kubelet access telemetry operationally useful for detection, triage, and forensic analysis.

## Scope

Applies to all cluster nodes. Covers auditd or equivalent host audit rules for kubelet configuration and credential paths, audit pipeline forwarding, and recurring validation that those rules remain active. Managed Kubernetes services may satisfy this through provider-documented equivalent node audit telemetry.

## Enforcement

- Primary Enforcement Point: Node audit daemon retention and forwarding configuration and centralized audit pipeline ingestion.
- Implementation Surface: Host audit daemon settings, forwarding agents, SIEM or audit pipeline ingestion, and review workflows for node-level kubelet access telemetry.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; OS/Infrastructure
