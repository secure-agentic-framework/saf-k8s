# SAF-K8S-1003-005 - Technique-aligned detection engineering for Kubernetes AI attack scenarios

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.3 - Threat Modeling Methodologies
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must develop and maintain detection rules aligned to MITRE ATT&CK for Containers techniques relevant to Kubernetes AI workload attack scenarios so that prioritized adversary behaviors are covered by actionable detections.

## Objective

Ensure ATT&CK-relevant Kubernetes AI attack scenarios are translated into actionable detection logic rather than remaining only as analytical coverage mappings.

## Rationale

Coverage mapping alone does not improve operational security if identified adversary behaviors are never translated into usable detections. Technique-aligned detection engineering turns ATT&CK analysis into concrete SIEM, runtime, and workflow detections that can identify active attacks against AI workloads and supporting platform components.

## Scope

Applies to all Kubernetes clusters hosting AI workloads. Detection development must cover prioritized container and Kubernetes attack techniques relevant to AI training, inference, orchestration, and supporting platform components.

## Enforcement

- Primary Enforcement Point: Detection rule management platform and detection engineering review process.
- Implementation Surface: ATT&CK technique-to-detection mappings, SIEM and runtime detection rule development, AI workload-specific technique prioritization, rule coverage reviews for prioritized attack scenarios, and analyst workflows that maintain technique-aligned detections as ATT&CK content and AI platform architecture evolve.
- Owner Primary: Security Architecture
- Owner Supporting: Detection Engineering; Security Operations; AI/ML Engineering
