# SAF-K8S-0203-014 - Authenticated cloud metadata service mode enforcement

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.3 - Host OS and Kernel Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes nodes on cloud infrastructure must enforce IMDSv2 or equivalent authenticated metadata access requirements so insecure metadata service modes are not available on node instances or managed node pools.

## Objective

Ensure cloud instance metadata access requires provider-authenticated request patterns instead of legacy insecure modes.

## Rationale

Cloud provider metadata services expose instance identity and credentials. If insecure metadata modes such as IMDSv1 remain available, simple HTTP requests or SSRF paths can retrieve cloud credentials. Requiring IMDSv2 or equivalent authenticated metadata access is independently auditable from endpoint or hop-limit tuning because it governs the core metadata access protocol mode.

## Scope

Applies to Kubernetes nodes on AWS, GCP, Azure, and other cloud providers exposing instance metadata services. Covers node group, VM, or managed service metadata configuration that customers can set or must verify through provider documentation and platform APIs.

## Enforcement

- Primary Enforcement Point: Cloud infrastructure provisioning and managed service configuration governing metadata protocol mode on node instances or node pools.
- Implementation Surface: Infrastructure-as-code templates, node group or VM metadata options, IMDSv2 or equivalent provider-authenticated metadata settings, and managed service verification of metadata mode defaults.
- Owner Primary: Cloud Infrastructure
- Owner Supporting: Platform Engineering; Security Engineering
