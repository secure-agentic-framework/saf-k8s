# SAF-K8S-0505-003 - Internal service endpoint protection

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.5 - API Server and Service Exposure Protection
- Maturity: Practitioner
- Classification: baseline

## Statement

Internal services must use ClusterIP type by default, and headless services must be secured with network policies restricting direct pod access to authorized consumers.

## Objective

Minimize internal service exposure to the ClusterIP scope and ensure that headless services have compensating network policies preventing unauthorized direct pod access.

## Rationale

ClusterIP services are only reachable within the cluster, providing a baseline level of network isolation. However, headless services expose individual pod IPs directly, allowing targeted access that bypasses service load balancing and potentially service-level access controls. Without network policies on headless services, any pod in an allowed network segment can directly address individual pods by IP.

## Scope

Applies to all internal services in production Kubernetes clusters. Headless services used for StatefulSets or service discovery must have compensating network policies restricting which consumers can access the exposed pod endpoints.

## Enforcement

- Primary Enforcement Point: Kubernetes Service type defaults and network policies for headless service pod endpoints.
- Implementation Surface: Kubernetes Service type ClusterIP (default), NetworkPolicy objects restricting access to headless service (clusterIP: None) pod endpoints by pod label selectors, admission controller policies enforcing ClusterIP as default service type, and periodic audit scripts identifying services without covering network policies.
- Owner Primary: Platform Engineering
- Owner Supporting: Application Development
