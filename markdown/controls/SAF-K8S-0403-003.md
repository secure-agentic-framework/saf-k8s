# SAF-K8S-0403-003 - Approved secret injection pattern standards

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations must select approved secret injection patterns (volume mounts, CSI secret store volumes, init containers, and sidecar agents) based on a documented security assessment of each pattern's tradeoffs, and must maintain documented standards specifying which injection patterns are approved for each workload classification.

## Objective

Ensure that secrets are delivered to workloads through injection patterns that minimize exposure risk by restricting filesystem permissions, enabling update without pod restart, or fetching secrets dynamically with automatic rotation.

## Rationale

Volume-mounted secrets can be restricted by filesystem permissions and updated without pod restart. CSI secret store volumes fetch secrets directly from external stores without persisting them broadly in platform data stores. Init container and sidecar agent patterns enable dynamic secret fetching with automatic rotation. Selecting injection patterns without documented security assessment can negate the protections provided by encryption at rest and external secrets management. Environment variable prohibition is a separate obligation covered by SAF-K8S-0403-016.

## Scope

Applies to all workloads that consume secrets in production clusters. Covers Kubernetes-native Secret volume mounts, CSI secret store volumes, init container injection, and sidecar-based secret agents. Excludes environment variable injection prohibition and exception governance, which are covered by SAF-K8S-0403-016. Development and testing environments should follow the same approved patterns where feasible.

## Enforcement

- Primary Enforcement Point: Documented Secret Injection Pattern Standards per Workload Classification
- Implementation Surface: Pod spec volume mounts for Secret objects, CSI secret store volume drivers, Vault Agent sidecar and init container injection patterns, and documented standards specifying approved injection patterns by workload type.
- Owner Primary: Application Development
- Owner Supporting: Security Engineering; Platform Engineering
