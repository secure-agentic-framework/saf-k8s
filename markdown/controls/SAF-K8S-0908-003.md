# SAF-K8S-0908-003 - Model watermarking and fingerprinting

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.8 - Model Abuse and Extraction Prevention
- Maturity: Advanced
- Classification: ai-specific

## Statement

Production model serving infrastructure must integrate watermarking or fingerprinting mechanisms that embed verifiable markers in model outputs to enable detection of stolen or unauthorized copies of model outputs.

## Objective

Enable detection and attribution of stolen model outputs by embedding imperceptible but verifiable markers that survive downstream use.

## Rationale

Even with extraction prevention controls, determined adversaries may successfully copy model behavior through distillation or output scraping. Without watermarking or fingerprinting, organizations cannot prove that a competitor's model was derived from their proprietary model, limiting legal and contractual recourse. Output watermarks provide forensic evidence of model theft that persists in downstream applications.

## Scope

Applies to production inference endpoints serving models with significant intellectual property value on Kubernetes. Covers watermark embedding at the serving layer, fingerprint verification tooling, and marker durability testing. Excludes internal-only models where theft attribution is not a concern.

## Enforcement

- Primary Enforcement Point: Inference server output postprocessing configuration and watermark embedding middleware deployed as part of the model serving pipeline.
- Implementation Surface: Inference server output postprocessing configuration for watermark embedding (output watermarking for token sequence patterns in LLM output), model weight watermarking integrated into training pipelines, fingerprinting via adversarial example databases, radioactive data marking in training data pipelines, watermark verification key storage in Kubernetes Secrets or external secret stores, and external model output scanning and monitoring services for watermark detection on public platforms.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Legal
