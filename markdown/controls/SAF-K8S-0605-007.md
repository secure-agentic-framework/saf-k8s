# SAF-K8S-0605-007 - Admission webhook fail-closed enforcement and timeout bounds

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.5 - Admission Control
- Maturity: Practitioner
- Classification: baseline

## Statement

Security-critical admission webhooks must use a failure policy of Fail and must set timeout values that bound API server latency while preserving fail-closed enforcement for requests that cannot be safely evaluated.

## Objective

Ensure admission decisions fail securely without allowing webhook latency or misconfiguration to silently disable enforcement or destabilize the API server request path.

## Rationale

A webhook configured with failurePolicy: Ignore or with unbounded response times can create either a silent policy bypass or an API server degradation path that attackers can exploit. Fail-closed behavior paired with deliberate timeout bounds preserves enforcement intent and keeps admission failure modes predictable.

## Scope

Applies to all validating and mutating admission webhooks in production and staging Kubernetes clusters where webhook decisions enforce security-relevant policy.

## Enforcement

- Primary Enforcement Point: Kubernetes API server webhook configuration.
- Implementation Surface: ValidatingWebhookConfiguration and MutatingWebhookConfiguration resources, failurePolicy settings, timeoutSeconds configuration, and operational validation of fail-closed behavior for security-critical request paths.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; SRE / Operations
