# SAF-K8S-0907-007 - Model deployment authorization and namespace-scoped release control

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.7 - Feature Store Security and Data Access Controls
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Model download and deployment operations must be authorized so that only approved principals can pull specific models and deploy them into authorized namespaces after required promotion and policy checks have passed.

## Objective

Control which principals can release specific models into specific Kubernetes namespaces.

## Rationale

Without model deployment authorization, any principal with broad namespace access can deploy unapproved, untested, or malicious models into serving environments. Namespace-scoped release controls reduce the blast radius of compromised identities and enforce promotion-gate integrity before model rollout.

## Scope

Applies to model download and deployment workflows for Kubernetes AI serving environments. Covers registry pull authorization as consumed by deployment workflows, admission control, and namespace-scoped deployment authorization. Excludes direct dataset access controls for training data storage.

## Enforcement

- Primary Enforcement Point: Admission control and deployment pipeline policies governing model pull, release approval, and namespace-scoped deployment authorization.
- Implementation Surface: Model deployment authorization policies showing per-principal and per-namespace release controls, admission checks verifying required promotion state before deployment, and namespace-scoped deployment restrictions for serving resources.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Data Engineering; Security Engineering; Platform Engineering
