# SAF-K8S-1007-016 - GPU node hardware attestation, driver integrity, and taint verification

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.7 - Cluster Lifecycle and Asset Inventory
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must verify hardware attestation where required, GPU driver integrity, and GPU node taint configuration before new GPU nodes are allowed to schedule AI workloads.

## Objective

Ensure newly added GPU nodes are attested, run approved GPU drivers, and advertise the correct scheduling constraints before they join the trusted pool.

## Rationale

Baseline-hardening checks alone do not prove that a new GPU node is running trusted hardware state, approved GPU drivers, or correct taint configuration. Attestation, driver verification, and taint checks reduce the risk of compromised or misconfigured nodes entering sensitive AI compute pools.

## Scope

Applies to GPU-capable nodes added through scaling or onboarding workflows in clusters hosting AI workloads.

## Enforcement

- Primary Enforcement Point: Attestation and integrity verification steps within node onboarding automation.
- Implementation Surface: Hardware attestation workflows, GPU driver checksum verification, GPU taint validation, and scheduling-readiness gates for new GPU nodes.
- Owner Primary: Infrastructure Engineering
- Owner Supporting: Platform Engineering; Security Engineering
