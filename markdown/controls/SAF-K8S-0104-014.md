# SAF-K8S-0104-014 - Recurring CIS Kubernetes Benchmark scan execution

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.4 - CIS Benchmarks and Patch Management
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must be assessed against the CIS Kubernetes Benchmark using automated tooling on a recurring schedule, including the applicable managed-service benchmark profile where relevant, so that baseline hardening checks are executed consistently across customer-assessable cluster surfaces.

## Objective

Establish a repeatable Kubernetes hardening baseline by running recurring automated CIS Benchmark assessments against the applicable cluster scope.

## Rationale

Baseline hardening fails in practice when benchmark scans are run only ad hoc, skipped after upgrades, or limited to partial cluster surfaces. This control isolates the recurring execution obligation so teams can verify that benchmark checks actually run on the required cadence and scope.

## Scope

Applies to all production clusters. Covers recurring CIS Benchmark execution for control-plane, node, and policy sections that are customer-assessable, including managed-service benchmark profiles where applicable.

## Enforcement

- Primary Enforcement Point: Audit / Telemetry
- Implementation Surface: Automated CIS Benchmark scanning pipeline, scheduled jobs, managed-service benchmark profile selection, and post-upgrade validation workflow.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Governance Risk and Compliance
