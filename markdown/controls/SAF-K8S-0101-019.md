# SAF-K8S-0101-019 - API server AlwaysAllow prohibition

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must not enable AlwaysAllow authorization on the API server.

## Objective

Prevent authorization bypass on the Kubernetes API server by prohibiting the AlwaysAllow authorizer.

## Rationale

AlwaysAllow turns authorization into an unconditional permit decision and defeats policy-based access control on the control plane. Prohibiting it ensures API requests remain subject to explicit authorization controls such as Node and RBAC modes.

## Scope

Applies to all production Kubernetes clusters and to non-production clusters that host sensitive data, shared workloads, or provide a path to production. For managed services, applies to all customer-exposed API server authorization settings and compensating controls when provider-managed settings are fixed.

## Enforcement

- Primary Enforcement Point: API server authorization configuration.
- Implementation Surface: API server authorization mode settings, managed Kubernetes security settings, provisioning templates, and periodic assessment verifying AlwaysAllow is absent.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Cloud Platform Engineering
