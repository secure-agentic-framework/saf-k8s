# SAF-K8S-0304-004 - Ephemeral container security context enforcement

- Domain: D03 - Workload and Pod Security
- Knowledge Area: 3.4 - Secure Defaults and Resource Constraints
- Maturity: Foundational
- Classification: baseline

## Statement

Ephemeral debug containers must be subject to the same security context restrictions as regular containers, and clusters must enforce this through RBAC restrictions on the pods/ephemeralcontainers subresource and policy engine rules that deny ephemeral containers violating the namespace's Pod Security Standards level.

## Objective

Prevent ephemeral debug containers from bypassing the security context restrictions applied to regular containers, closing a privilege escalation path through the kubectl debug side door.

## Rationale

Ephemeral containers added for debugging bypass normal deployment pipelines and may not inherit security context restrictions, allowing privileged access through a side door. Because ephemeral containers are injected after initial pod admission, they can circumvent review processes applied to regular deployments. An attacker or careless operator using kubectl debug with elevated privileges can gain root access, mount host filesystems, or disable seccomp profiles within a running pod.

## Scope

Applies to all pods in production clusters whenever kubectl debug or equivalent tooling is used to inject ephemeral containers. Includes init containers promoted to ephemeral debug containers.

## Enforcement

- Primary Enforcement Point: RBAC rules restricting access to the pods/ephemeralcontainers subresource, Pod Security Admission enforcing PSS levels on ephemeral containers, and policy engine rules denying ephemeral containers with privileged settings or capabilities beyond the pod's PSS level.
- Implementation Surface: RBAC rules for pods/ephemeralcontainers subresource restricted to authorized debugging roles, Pod Security Admission and policy engine rules enforcing security contexts on ephemeral containers (privileged, allowPrivilegeEscalation, capabilities, runAsNonRoot, seccomp, AppArmor), and audit logging for ephemeral container creation events at high severity.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering
