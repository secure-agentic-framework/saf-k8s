# SAF-K8S-1004-005 - Adversarial ML threat taxonomy and structured classification

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.4 - AI and Supply Chain Threat Taxonomy
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must apply the NISTIR 8269 adversarial ML taxonomy and maintain adversarial ML attack pattern libraries so that Kubernetes AI threats are classified consistently and mapped to structured detection, triage, and risk analysis workflows.

## Objective

Standardize adversarial ML threat classification for Kubernetes AI environments so detections, triage decisions, and risk assessments use a consistent attack taxonomy.

## Rationale

AI-specific attacks such as evasion, model extraction, or poisoning are hard to compare and prioritize when teams classify them ad hoc. Structured adversarial ML taxonomy use creates a shared language for detection engineering, incident classification, and threat analysis, reducing ambiguity when new attack patterns emerge.

## Scope

Applies to security operations, threat intelligence, and detection teams responsible for classifying threats affecting Kubernetes-hosted AI workloads, models, and supporting pipelines.

## Enforcement

- Primary Enforcement Point: Threat classification standard, adversarial ML attack pattern library, and incident classification workflow.
- Implementation Surface: NISTIR 8269 taxonomy adoption, adversarial ML attack pattern library maintenance, structured threat classification in incident tooling, mapping of adversarial ML patterns to detections and controls, and analyst workflows for updating classifications as new AI attack techniques emerge.
- Owner Primary: Security Operations
- Owner Supporting: Threat Intelligence; Detection Engineering; Security Architecture
