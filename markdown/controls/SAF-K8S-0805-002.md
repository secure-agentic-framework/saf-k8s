# SAF-K8S-0805-002 - GPU allocation audit trail and workload identity tracking

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.5 - GPU Workload Auditing and Monitoring
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must maintain auditable records of GPU accelerator allocation tracking which pods used which specific GPU devices, the duration of allocation, the workload identity (service account, namespace, job name) associated with each allocation, and the GPU sharing mode in effect during the allocation period.

## Objective

Enable forensic investigation, cost attribution, and compliance auditing of GPU usage by maintaining a complete chain of custody for accelerator allocation that links physical GPU devices to authenticated workload identities and time windows.

## Rationale

GPU resources are high-value assets (thousands of dollars per hour for large-scale training clusters) and attractive targets for unauthorized use. Without allocation audit trails, organizations cannot determine which workload consumed GPU resources during a security incident, cannot attribute GPU costs to specific teams or projects, and cannot verify that regulated workloads were processed only on approved GPU hardware. Post-incident forensics require the ability to reconstruct which workload had access to which GPU at any point in time.

## Scope

Applies to all Kubernetes clusters with GPU workloads. Covers device plugin allocation events, pod scheduling records, GPU device-to-pod binding, and workload identity metadata. Audit records must be retained in accordance with organizational log retention policies. Managed GPU services must supplement provider billing records with Kubernetes-level allocation audit data.

## Enforcement

- Primary Enforcement Point: Kubernetes audit policy configuration for GPU resource events, device plugin allocation logging, GPU Operator event collection, and centralized log aggregation pipeline for GPU audit data.
- Implementation Surface: NVIDIA device plugin allocation logging (device ID, pod name, namespace, timestamp), Kubernetes audit policy configuration for GPU resource request and scheduling events, GPU Operator event collection, workload identity correlation (service account, namespace, job name, owner references, workload classification labels), GPU sharing mode recording at allocation time, allocation duration tracking with orphaned allocation detection, centralized SIEM integration for GPU audit records, and forensic query tooling for GPU allocation history reconstruction.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Operations; Observability Engineering; FinOps
