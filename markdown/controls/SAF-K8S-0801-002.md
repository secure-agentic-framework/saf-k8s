# SAF-K8S-0801-002 - MIG partitioning for hardware-enforced GPU isolation

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Advanced
- Classification: ai-specific

## Statement

Multi-tenant GPU clusters must use Multi-Instance GPU (MIG) partitioning where supported hardware is available, with MIG profiles configured to enforce hardware-level isolation between tenants and resource advertisements accurately reflecting partitioned GPU instances to the Kubernetes scheduler.

## Objective

Provide hardware-enforced memory and compute isolation between GPU tenants, preventing cross-tenant data leakage and ensuring that resource scheduling accurately reflects the isolated GPU partitions available to each workload.

## Rationale

Without MIG or equivalent hardware partitioning, multiple tenants sharing a GPU operate in the same fault domain with shared memory spaces. This enables cross-tenant GPU memory scraping, side-channel attacks through shared caches, and resource contention that can degrade model training integrity. Inaccurate MIG resource advertisement can cause scheduling failures or over-commitment.

## Scope

Applies to Kubernetes clusters with NVIDIA A100, H100, or later GPUs that support MIG, where multiple tenants or security domains share GPU nodes. Single-tenant dedicated GPU pools may document an exception. Managed Kubernetes GPU offerings must verify MIG configuration through provider controls.

## Enforcement

- Primary Enforcement Point: NVIDIA MIG manager configuration, GPU Operator MIG strategy settings, device plugin MIG resource advertisement, and admission control policies validating MIG partition requests.
- Implementation Surface: NVIDIA MIG Manager and nvidia-smi MIG CLI (nvidia-smi mig -i <gpu-id> -cgi <profile> -C), GPU Operator MIGConfig CRDs and MIG strategy settings, device plugin MIG resource advertisements (nvidia.com/mig-<profile> extended resources), Kubernetes ResourceQuota objects scoped to nvidia.com/mig-* resources per namespace, admission control policies (OPA/Gatekeeper, Kyverno) validating MIG profile requests, and MIG reconfiguration access controls restricting GPU quiescing to platform administrators.
- Owner Primary: ML Platform Engineering
- Owner Supporting: GPU Infrastructure Engineering; Security Engineering
