# SAF-K8S-0606-026 - Automated SCM security posture assessment before promotion reliance

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Source repositories must be continuously assessed with OpenSSF Scorecard or equivalent automated SCM posture tooling, and promotion decisions must not rely on repositories whose security posture fails defined acceptance criteria.

## Objective

Reduce reliance on insecure source repositories by enforcing automated SCM posture assessment as an input to promotion trust.

## Rationale

A repository with weak branch protection, unsafe workflows, or poor dependency hygiene can undermine deployment trust even when a specific artifact is fresh. Automated SCM posture assessment is independently auditable because it evaluates repository governance and workflow security rather than artifact age.

## Scope

Applies to source repositories whose outputs may be promoted to production. Covers continuous repository posture assessment and promotion decisions that depend on repository trustworthiness.

## Enforcement

- Primary Enforcement Point: SCM posture assessment tooling and promotion trust policy.
- Implementation Surface: Automated repository posture assessment tooling such as OpenSSF Scorecard, repository acceptance thresholds, and promotion controls that require passing SCM security posture results before repository outputs are trusted for release.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Development Teams
