# SAF-K8S-0101-013 - API server certificate rotation and validation

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must implement automated rotation and validation of API server certificates before expiry so that expired, self-signed, or otherwise invalid certificates do not remain in use on any API server communication path.

## Objective

Ensure API server certificate lifecycle controls prevent expiry, invalid trust chains, and stale certificates from weakening authenticated TLS protection.

## Rationale

Expired or invalid certificates create opportunities for man-in-the-middle attacks, break trust validation, and undermine otherwise-correct TLS configuration. Rotation and validation are a distinct lifecycle obligation from basic TLS enablement.

## Scope

Applies to all API server communication paths: client to API server (front-end), API server to etcd, API server to kubelet, API server to aggregated API servers, and webhook backends. For managed Kubernetes services, applies to customer-configurable TLS settings and to verification that provider-managed connections meet organizational TLS requirements.

## Enforcement

- Primary Enforcement Point: Certificate Lifecycle Management
- Implementation Surface: Certificate management infrastructure, renewal automation, managed-service certificate posture checks, and periodic certificate validity assessment.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; PKI/Certificate Management
