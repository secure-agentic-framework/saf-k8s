# SAF-K8S-0405-023 - Break-glass access audit logging coverage

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.5 - Identity Abuse Detection and Mitigation
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must ensure all break-glass access is fully audited so emergency privilege use remains attributable, reviewable, and available for post-use investigation.

## Objective

Preserve accountability for emergency access by ensuring all break-glass activity generates complete audit records.

## Rationale

Break-glass access is an intentional exception to normal controls. Complete audit logging is required so emergency actions can be reconstructed, reviewed, and investigated after use.

## Scope

Applies to all break-glass credentials, kubeconfigs, or certificates issued for production Kubernetes clusters, including managed and multi-tenant environments. Covers audit trails, post-use review evidence, automatic revocation, and expiration after emergency access use.

## Enforcement

- Primary Enforcement Point: Emergency access audit logging controls.
- Implementation Surface: Audit logging for break-glass activation and usage, log retention in central audit systems, and post-use access review evidence.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Identity and Access Management
