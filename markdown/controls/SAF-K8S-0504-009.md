# SAF-K8S-0504-009 - API-aware request contract validation

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.4 - Zero Trust Architecture and Service Mesh
- Maturity: Practitioner
- Classification: baseline

## Statement

Service-level access control must include API-aware security policies that validate request patterns against expected service contracts so malformed or unexpected operations are rejected before reaching backend logic.

## Objective

Validate request behavior against expected API contracts so allowed clients cannot abuse application-layer interfaces with unexpected request patterns.

## Rationale

Authorized traffic can still abuse services through unexpected request shapes or unsupported operations. Separating API contract validation from L7 authorization makes request-shape enforcement independently auditable.

## Scope

Applies to production clusters running a service mesh or CNI with L7 policy support. Priority should be given to services handling sensitive data, admin operations, or cross-trust-boundary communication. L7 policies require application protocol awareness and may not apply to non-HTTP workloads without protocol-specific support.

## Enforcement

- Primary Enforcement Point: API-aware request validation controls.
- Implementation Surface: API gateway or envoy policy enforcement validating requests against OpenAPI or expected service contracts, plus tests showing invalid request patterns are rejected.
- Owner Primary: Security Engineering
- Owner Supporting: Network Engineering; Application Development
