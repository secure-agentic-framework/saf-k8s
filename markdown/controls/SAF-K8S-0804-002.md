# SAF-K8S-0804-002 - Remote attestation for TEE integrity verification

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.4 - Confidential Computing for AI Workloads
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must implement remote attestation workflows that cryptographically verify TEE integrity, GPU Confidential Computing mode status, and platform configuration before loading sensitive models or training data into the enclave, with attestation failures preventing workload execution.

## Objective

Establish cryptographic proof that the TEE environment has not been tampered with before entrusting it with sensitive AI models or data, preventing workloads from executing in compromised or simulated enclave environments.

## Rationale

Without remote attestation, a workload has no way to verify that it is actually running inside a genuine TEE with active memory encryption rather than in a simulated or compromised environment that claims TEE status. An attacker who controls the host can present a fake TEE interface that appears functional but does not encrypt memory, enabling full access to loaded models and data. Remote attestation provides a hardware-rooted chain of trust that can be verified by external parties before data is released to the enclave.

## Scope

Applies to all confidential AI workloads using TEE protection where remote attestation is supported by the hardware platform (NVIDIA attestation service, Intel Trust Authority, AMD SEV-SNP attestation). Covers pre-deployment attestation before model loading and periodic re-attestation during long-running training jobs. This is an emerging control that may require vendor-specific attestation infrastructure.

## Enforcement

- Primary Enforcement Point: Attestation verification service integration, workload orchestration pre-checks, model serving pipeline attestation gates, and key release policies conditioned on successful attestation.
- Implementation Surface: NVIDIA Attestation SDK and attestation service for H100 CC mode, Intel Trust Authority attestation service, AMD SEV-SNP attestation reports, remote attestation verification service integration with workload orchestration, hardware root-of-trust measurement and attestation report signing, model loading pipeline attestation gates, key release policies conditioned on successful attestation verification, and periodic re-attestation scheduling for long-running training jobs.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; GPU Infrastructure Engineering; Key Management
