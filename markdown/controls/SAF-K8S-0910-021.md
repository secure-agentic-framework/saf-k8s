# SAF-K8S-0910-021 - Cross-cluster endpoint and workload authentication for AI communication

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.10 - Multi-Cluster and Federated AI Security
- Maturity: Advanced
- Classification: ai-specific

## Statement

Cross-cluster network communication for distributed AI workloads must authenticate workload-to-workload or cluster-to-cluster endpoints using strong federated identity so cross-cluster AI links cannot be spoofed by unauthorized endpoints.

## Objective

Prevent endpoint impersonation on cross-cluster AI communication paths by enforcing authenticated endpoint identity across federated clusters.

## Rationale

Encrypted channels alone do not prove that the remote endpoint is legitimate. Cross-cluster AI systems need authenticated workload or cluster identities so spoofed routing components, training workers, or pipeline endpoints cannot participate in distributed AI traffic flows.

## Scope

Applies to all cross-cluster network communication for AI workloads, including distributed training gradient exchange, multi-cluster inference routing, and cross-cluster pipeline data transfer. Covers endpoint or workload authentication for cross-cluster links. Excludes transport-channel encryption itself.

## Enforcement

- Primary Enforcement Point: Endpoint and workload authentication for cross-cluster communication links.
- Implementation Surface: Service mesh endpoint authentication, VPN or tunnel endpoint identity validation, SPIFFE-based authenticated endpoint identity for cross-cluster links, and certificate issuance, rotation, and validation for authenticated cross-cluster endpoints.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Security Engineering; ML Platform Engineering
