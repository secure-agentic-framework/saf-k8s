# SAF-K8S-0805-003 - GPU-based attack detection for cryptomining and memory scraping

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.5 - GPU Workload Auditing and Monitoring
- Maturity: Advanced
- Classification: ai-specific

## Statement

GPU monitoring systems must include detection rules for unauthorized cryptocurrency mining on training clusters, GPU memory scraping attempts that read residual data from previous workloads, and covert communication channels established through GPU shared memory, with automated alerting and response actions for confirmed detections.

## Objective

Detect and respond to GPU-specific attack patterns that exploit the unique characteristics of accelerator hardware, preventing resource theft, data exfiltration, and covert communication through GPU-specific channels.

## Rationale

GPUs are prime targets for cryptomining due to their parallel computation capabilities, and unauthorized mining on training clusters wastes resources costing thousands of dollars per hour while potentially delaying legitimate training jobs. GPU memory scraping exploits the lack of automatic memory clearing between workloads to read model weights, training data, or inference results from previous tenants. GPU shared memory (CUDA IPC) can be exploited as a covert communication channel that bypasses standard network monitoring. These GPU-specific attack vectors are invisible to CPU-focused security tools.

## Scope

Applies to all Kubernetes GPU clusters, with heightened priority for multi-tenant clusters and clusters processing sensitive data. Covers cryptomining signature detection, GPU memory access pattern monitoring, and CUDA IPC usage auditing. Detection rules must be tuned to the organization's GPU workload baseline to minimize false positives from legitimate high-utilization training workloads.

## Enforcement

- Primary Enforcement Point: GPU telemetry anomaly detection rules, CUDA IPC usage monitoring, workload behavioral analysis, and automated incident response integration (pod termination, node cordoning).
- Implementation Surface: GPU telemetry anomaly detection rules for cryptomining signatures (sustained >95% utilization without authorized training/inference labels), CUDA IPC usage auditing and shared memory (/dev/shm) monitoring, GPU memory access pattern monitoring for scraping detection (cuMemAlloc/cuMemcpy read-before-write patterns), runtime security tools (Falco, Tetragon) detecting known mining process names and CUDA kernel signatures, automated incident response integration (pod termination, service account GPU access revocation, node cordoning), and workload behavioral analysis against GPU utilization baselines.
- Owner Primary: Security Operations
- Owner Supporting: ML Platform Engineering; GPU Infrastructure Engineering; Incident Response
