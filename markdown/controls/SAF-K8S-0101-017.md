# SAF-K8S-0101-017 - API server audit log retention enforcement

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must retain API server audit logs for at least the minimum period defined by organizational policy and enforce that retention in downstream log storage.

## Objective

Ensure API server audit records remain available for the required investigation, compliance, and evidence window through enforced retention policies.

## Rationale

Even when audit logs reach a durable backend, they lose security and compliance value if they are aged out too quickly or deleted before the required retention window. Explicit retention enforcement preserves the audit trail for investigation and regulatory evidence needs.

## Scope

Applies to all production Kubernetes clusters and downstream storage systems that retain API server audit logs, including centralized logging platforms, SIEM repositories, and archival stores.

## Enforcement

- Primary Enforcement Point: Retention policy configuration on audit log storage and downstream aggregation systems.
- Implementation Surface: Retention settings on rotated log files, centralized log aggregation or SIEM storage policies, archival retention controls, and validation that API server audit logs are preserved for the required minimum period.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Compliance
