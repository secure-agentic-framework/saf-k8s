# SAF-K8S-0402-015 - Legacy service account token secret removal

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.2 - Service Accounts and Workload Identity
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes clusters must disable or remove all legacy static service account token secrets of type kubernetes.io/service-account-token.

## Objective

Eliminate persistent, non-expiring credentials by removing legacy static service account token Secrets, ensuring that stale agent identities cannot be exploited through credentials that never expire.

## Rationale

Legacy static service account tokens stored as Kubernetes Secrets never expire and are valid for any audience. A stolen legacy token provides indefinite cluster access until manually revoked. Removing these secrets enforces account disabling for unused agent identities and eliminates the risk surface from stale credentials. This control was decomposed from the original SAF-K8S-0402-005 which combined projected token configuration with legacy token removal.

## Scope

Applies to all clusters that have existing kubernetes.io/service-account-token type Secrets, whether auto-created or manually provisioned. All such Secrets must be identified using LegacyServiceAccountTokenTracking controller metrics (stable since Kubernetes 1.28) and scanning tools, then removed or migrated.

## Enforcement

- Primary Enforcement Point: Periodic scans for remaining kubernetes.io/service-account-token type Secrets, LegacyServiceAccountTokenTracking controller monitoring (serviceaccount_legacy_auto_token_uses_total, serviceaccount_legacy_manual_token_uses_total), and admission policies rejecting pods that mount legacy token Secrets.
- Implementation Surface: LegacyServiceAccountTokenTracking controller metrics, kubernetes.io/service-account-token type Secret scanning, admission policies blocking legacy token Secret mounts, and audit logs recording legacy token use events.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Application Development
