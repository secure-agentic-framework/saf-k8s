# SAF-K8S-0910-024 - Cross-cluster model provenance chain-of-custody preservation

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.10 - Multi-Cluster and Federated AI Security
- Maturity: Advanced
- Classification: ai-specific

## Statement

Model registry federation across clusters must preserve provenance chain-of-custody metadata for replicated model artifacts so downstream registries and clusters can trace replicated artifacts to their originating training, signing, and promotion workflow.

## Objective

Preserve artifact traceability across cluster boundaries by ensuring replicated model artifacts retain their provenance chain-of-custody metadata.

## Rationale

Cross-cluster replication can erode supply-chain assurance if provenance metadata is stripped, truncated, or no longer traceable after federation. Preserving chain-of-custody metadata ensures downstream clusters can validate where replicated model artifacts originated and what training, signing, and promotion workflow produced them.

## Scope

Applies to federated model registries and cross-cluster artifact replication topologies that replicate model metadata, signing identity, promotion gate results, and originating cluster lineage across clusters or regions. Excludes endpoint membership authorization and revocation governance for the federation itself.

## Enforcement

- Primary Enforcement Point: Provenance metadata preservation configuration and verification for replicated model artifacts.
- Implementation Surface: Registry replication settings that preserve provenance metadata, post-replication verification of preserved chain-of-custody fields, and downstream traceability checks for replicated model artifacts.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Platform Engineering
