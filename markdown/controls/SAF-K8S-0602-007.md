# SAF-K8S-0602-007 - Admission signature bypass and emergency break-glass governance

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.2 - Image Signing and Admission Enforcement
- Maturity: Practitioner
- Classification: baseline

## Statement

Any bypass or exemption from admission-time image signature verification must be explicitly documented, time-limited, and restricted to an audited emergency break-glass procedure.

## Objective

Constrain and audit any exceptional bypass of image signature verification so emergency operation does not become a persistent integrity gap.

## Rationale

Exception governance is a separate obligation from enforcement. Even with strong admission verification, undocumented or open-ended bypass mechanisms undermine the control by creating an alternate path for unsigned or tampered images.

## Scope

Applies to all production and staging Kubernetes clusters. All namespaces must be covered unless explicitly exempted in a documented policy (e.g., kube-system for provider-managed components). Emergency break-glass procedures must be audited and time-limited.

## Enforcement

- Primary Enforcement Point: Admission bypass policy, namespace exemption policy, and emergency break-glass governance workflow.
- Implementation Surface: Break-glass annotation mechanisms, exemption policies, auto-expiry controls, exception approval records, and audit logs for bypass use or emergency exemption activation.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
