# SAF-K8S-0103-006 - Profiling endpoint disablement for controller-manager and scheduler

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must disable profiling endpoints on the controller-manager and scheduler by setting --profiling=false on each component, ensuring that debug endpoints are not exposed on any control-plane component.

## Objective

Eliminate profiling debug surface on controller-manager and scheduler processes, complementing API server profiling disablement (SAF-K8S-0101-001) to achieve consistent control-plane hardening.

## Rationale

The controller-manager and scheduler each expose /debug/pprof/ profiling endpoints by default. These endpoints leak internal runtime state, including CPU profiles, heap dumps, and goroutine traces, and they can be abused for denial-of-service by triggering expensive profiling operations. SAF-K8S-0101-001 covers the API server. This control ensures the same hardening is applied to the remaining control-plane components.

## Scope

Applies to all production Kubernetes clusters. For managed Kubernetes services, verify through provider documentation that profiling is disabled on provider-managed controller-manager and scheduler processes, or document as a provider-managed control.

## Enforcement

- Primary Enforcement Point: Controller-Manager Configuration
- Implementation Surface: Controller-manager and scheduler static pod manifests or flag configuration (--profiling=false on both components), CIS Benchmark checks for profiling flags.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering
