# SAF-K8S-0401-001 - RBAC least-privilege design

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.1 - Role-Based Access Control (RBAC)
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must implement RBAC using Roles, ClusterRoles, RoleBindings, and ClusterRoleBindings designed according to least-privilege principles, granting only the minimum permissions required for each subject to perform its designated function.

## Objective

Ensure that every human user, service account, and group is granted only the Kubernetes API permissions strictly necessary for its role, reducing the blast radius of compromised credentials or insider misuse.

## Rationale

Overly permissive RBAC configurations are among the most common Kubernetes misconfigurations. When subjects hold unnecessary privileges such as wildcard verbs or cluster-wide resource access, a single compromised credential can lead to full cluster takeover, data exfiltration, or workload manipulation. Least-privilege RBAC design constrains each actor to its operational boundary.

## Scope

Applies to all production and pre-production Kubernetes clusters. Covers all RBAC objects including Roles, ClusterRoles, RoleBindings, and ClusterRoleBindings for human users, service accounts, and group subjects. Managed Kubernetes services must apply this control to all customer- configurable RBAC settings.

## Enforcement

- Primary Enforcement Point: RBAC manifest definitions, cluster provisioning templates, admission policies that reject overly permissive role definitions, and periodic RBAC configuration audits.
- Implementation Surface: Kubernetes RBAC API objects (Roles, ClusterRoles, RoleBindings, ClusterRoleBindings), namespace-scoped Role definitions, admission policies (OPA Gatekeeper, Kyverno) rejecting wildcard verb or resource grants, cluster provisioning templates and GitOps manifests, RBAC review tooling, and periodic RBAC configuration audit reports.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Identity and Access Management
