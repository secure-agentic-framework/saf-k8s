# SAF-K8S-0909-002 - Embedding pipeline integrity

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.9 - RAG Infrastructure Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Embedding pipelines that convert documents into vector representations for RAG indexes must verify the integrity and provenance of the embedding model, must validate input documents before processing, and must produce deterministic or reproducible embeddings that can be audited for consistency.

## Objective

Ensure that the embedding model used for RAG indexing is authentic and untampered, that input documents are validated before processing, and that embedding outputs are reproducible for integrity auditing.

## Rationale

The embedding model controls the semantic representation of all indexed documents. A substituted or compromised embedding model can manipulate which documents appear semantically similar, controlling retrieval results without touching the vector database. Unvalidated input documents can exploit parsing vulnerabilities or introduce adversarial content. Non-reproducible embeddings prevent integrity verification and make poisoning detection harder.

## Scope

Applies to all embedding pipelines that generate vector representations for RAG indexes on Kubernetes. Covers embedding model signature verification, input document validation and sanitization, and embedding reproducibility configuration. Excludes training-time embedding generation for model training.

## Enforcement

- Primary Enforcement Point: Embedding pipeline job configuration, model signature verification at pipeline startup, and input validation middleware in the document processing pipeline.
- Implementation Surface: Embedding model cryptographic signature verification at pipeline startup (cosign/notation), embedding model version pinning in pipeline configuration, ML-BOM entries for embedding models, input document format validation and size limits, document content sanitization (script/macro removal from HTML/PDF/Office), malware scanning before ingestion, deterministic embedding configuration (fixed random seeds, consistent batching), and embedding pipeline run logging (documents processed, model version, collection version).
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Data Engineering
