# SAF-K8S-1005-024 - Inference service quarantine and pipeline execution suspension with state preservation

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.5 - Incident Response for Kubernetes
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must define and document AI-specific containment procedures for quarantining compromised inference services and suspending ML pipeline execution with state-preservation considerations where applicable.

## Objective

Enable safe containment of incidents affecting inference services and ML pipeline controllers through targeted quarantine and suspension actions that preserve evidence and avoid corrupting serving or orchestration state.

## Rationale

Inference services and ML pipeline controllers introduce containment risks beyond standard workload isolation. Quarantining inference traffic can affect customer-facing availability, and suspending pipelines mid-run can corrupt state or lose evidence unless responders use service-aware containment procedures.

## Scope

Applies to Kubernetes clusters running model serving platforms and ML pipeline controllers. Covers inference service quarantine, traffic cutoff or scale-down actions, and pipeline execution suspension with state-preservation considerations.

## Enforcement

- Primary Enforcement Point: AI-specific incident response playbooks and service-specific containment procedures for inference and pipeline platforms.
- Implementation Surface: Inference service scale-to-zero or traffic-cutoff procedures, service quarantine workflows, pipeline suspension controls for Argo or equivalent orchestrators, checkpoint-preservation guidance, and runbooks for stateful AI service containment.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; AI/ML Engineering; SRE
