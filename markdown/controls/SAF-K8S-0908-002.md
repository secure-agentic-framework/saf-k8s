# SAF-K8S-0908-002 - Inference API information exposure controls

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.8 - Model Abuse and Extraction Prevention
- Maturity: Advanced
- Classification: ai-specific

## Statement

Inference APIs must be designed to minimize information disclosure by restricting output confidence scores, controlling access to logit and probability distributions, preventing feature leakage, and obfuscating model architecture details at the serving boundary.

## Objective

Reduce the information available to attackers through inference API responses, limiting the effectiveness of model extraction, inversion, and reverse engineering attacks.

## Rationale

Verbose inference API responses that include confidence scores, full probability distributions, or logit values provide attackers with the precise information needed to execute efficient model extraction and inversion attacks. Feature leakage in responses reveals model internals. Exposed architecture details (layer counts, parameter sizes, model family) enable targeted adversarial attacks. Restricting this information significantly increases the cost and difficulty of attacks without degrading legitimate use.

## Scope

Applies to all inference API response configurations on Kubernetes. Covers output field selection, confidence score precision, logit access controls, error message content, and API metadata that reveals model architecture. Excludes internal debugging endpoints that are network-isolated from external access.

## Enforcement

- Primary Enforcement Point: Inference API response serialization configuration, API gateway response transformation policies, and model serving endpoint configuration.
- Implementation Surface: Inference API response serialization configuration (top-K only or argmax-only output, confidence score precision rounding), API gateway response transformation policies removing logit/probability distributions, model architecture metadata stripping from API responses, generic error message configuration with server-side-only detailed logging, response time jitter injection, separate internal and external inference API configurations, and API response audit tooling verifying information minimization.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; API Platform Engineering
