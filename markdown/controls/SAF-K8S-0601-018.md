# SAF-K8S-0601-018 - Registry push-time container image vulnerability rescanning

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Container images must be rescanned at registry push time so newly built or externally introduced images receive current vulnerability analysis before promotion or deployment decisions are made.

## Objective

Maintain current vulnerability visibility for pushed container images after build completion by rescanning images in registries.

## Rationale

Build-time scanning alone does not cover images introduced outside standard CI paths or vulnerability disclosures that occur between build completion and registry promotion. Registry rescanning provides a current checkpoint at the artifact distribution layer.

## Scope

Applies to container registries and deployed workloads across all Kubernetes clusters, including runtime scanning of AI/ML images and continuous monitoring of native library exposure.

## Enforcement

- Primary Enforcement Point: Registry webhook scanning and registry-side rescan configuration.
- Implementation Surface: Registry webhook scanning, registry-native vulnerability scanning services, image push events, and rescan results for pushed images.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
