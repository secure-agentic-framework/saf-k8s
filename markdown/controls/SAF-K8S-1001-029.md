# SAF-K8S-1001-029 - SIEM ingestion health, delivery completeness, and source coverage monitoring

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must monitor SIEM ingestion pipelines for Kubernetes audit logs and AI workload security events to verify delivery completeness, forwarding health, and expected source coverage for the telemetry required by the SOC.

## Objective

Ensure that onboarded Kubernetes and AI security telemetry actually arrives completely and continuously in the SIEM.

## Rationale

Initial forwarding configuration does not guarantee ongoing observability. Pipeline failures, partial delivery, or missing sources can silently blind detection and response. Continuous ingestion health, delivery completeness, and source coverage monitoring preserves trustworthy visibility for the SOC.

## Scope

Applies to monitoring and verification of SIEM ingestion pipelines for Kubernetes and AI workload security telemetry. Covers delivery health, completeness checks, missing-source detection, and verification that expected telemetry feeds remain active. Excludes initial source onboarding and forwarding configuration itself.

## Enforcement

- Primary Enforcement Point: SIEM ingestion health monitoring and completeness verification.
- Implementation Surface: Log forwarding health dashboards, delivery completeness checks, missing-source detection, pipeline health alerts, and source coverage verification for the Kubernetes and AI security telemetry required by the SOC.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; GRC/Compliance
