# SAF-K8S-0201-015 - Node-level kubelet audit rule coverage

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.1 - Kubelet and Node Configuration Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Each cluster node must configure Linux audit rules or equivalent host audit telemetry to detect read and write access to kubelet configuration files, kubeconfig files, and PKI material, and recurring verification must confirm those audit rules remain active.

## Objective

Detect access to or modification of kubelet configuration and credential material directly on cluster nodes.

## Rationale

Host-level audit rule coverage provides a node-local detection record when privileged users, escaped workloads, or local compromise access kubelet configuration and credential paths. Verifying the rules remain active makes this detection control independently auditable from forwarding or retention workflows.

## Scope

Applies to all cluster nodes. Covers auditd or equivalent host audit rules for kubelet configuration and credential paths, audit pipeline forwarding, and recurring validation that those rules remain active. Managed Kubernetes services may satisfy this through provider-documented equivalent node audit telemetry.

## Enforcement

- Primary Enforcement Point: Node audit rule configuration and periodic verification of monitored kubelet paths.
- Implementation Surface: Linux audit rules or equivalent host telemetry for kubelet configuration files, kubeconfig files, and PKI paths, plus recurring rule validation.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; OS/Infrastructure
