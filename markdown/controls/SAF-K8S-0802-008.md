# SAF-K8S-0802-008 - GPU kernel module signing and Secure Boot enforcement

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.2 - GPU Driver, Library, and Toolkit Security
- Maturity: Advanced
- Classification: ai-specific

## Statement

GPU kernel module loading must be restricted to signed and approved modules through kernel module signing enforcement and Secure Boot validation on GPU nodes.

## Objective

Prevent unauthorized GPU kernel modules from loading by enforcing signed-module trust at the kernel and boot chain boundary.

## Rationale

Unauthorized kernel module loading can replace legitimate GPU drivers with backdoored versions that intercept GPU operations at the kernel level. Secure Boot and kernel module signing are distinct trust controls at the OS and boot boundary and should be audited independently from filesystem monitoring of driver binaries.

## Scope

Applies to all Kubernetes GPU nodes running NVIDIA kernel modules. Covers Secure Boot status, kernel module signing enforcement, and validation that only approved GPU kernel modules can load.

## Enforcement

- Primary Enforcement Point: Kernel Module Trust Enforcement
- Implementation Surface: Kernel module signing enforcement, Secure Boot configuration and attestation on GPU nodes, modinfo or module signature validation, and runtime detection of unauthorized module load events.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; GPU Infrastructure Engineering
