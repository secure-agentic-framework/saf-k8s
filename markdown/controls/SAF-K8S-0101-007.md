# SAF-K8S-0101-007 - API server request rate limiting and API Priority and Fairness

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Production Kubernetes clusters must configure API Priority and Fairness (APF) to classify and throttle API requests, ensuring that control-plane management traffic (leader election, node heartbeats, critical controllers) is prioritized over bulk workload operations. Clusters must define flow schemas that prevent any single tenant, namespace, or controller from monopolizing API server capacity. Legacy --max-requests-inflight and --max-mutating-requests-inflight flags must be reviewed and set appropriately where APF is not fully enabled.

## Objective

Protect API server availability and responsiveness under load by ensuring fair resource allocation across request categories and preventing denial-of-service from runaway controllers or high-volume workloads.

## Rationale

AI/ML workloads frequently generate high API server load through large-scale job submissions, frequent status updates from distributed training coordinators, and batch pod creation. Without rate limiting and prioritization, these bursts can starve critical control-plane operations (leader election, node heartbeat processing), leading to cascading failures. API Priority and Fairness provides a structured mechanism to classify requests into priority levels and apply fair queuing, replacing the coarse-grained legacy rate limits.

## Scope

Applies to all production Kubernetes clusters, especially those running large-scale AI/ML workloads. Covers APF FlowSchema and PriorityLevelConfiguration resources, as well as legacy rate limit flags. For managed Kubernetes services, verify provider APF defaults and configure customer-accessible flow schemas.

## Enforcement

- Primary Enforcement Point: API Server Configuration
- Implementation Surface: FlowSchema and PriorityLevelConfiguration custom resources, API server flags (--max-requests-inflight, --max-mutating-requests-inflight, --enable-priority-and-fairness), monitoring dashboards for APF queue depth and rejection rates.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; AI/ML Engineering
