# SAF-K8S-0102-015 - etcd certificate rotation testing and recovery validation

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters must test and document the etcd certificate rotation process, including recovery or rollback procedures, so that certificate replacement can be executed safely without prolonged control-plane disruption if rotation fails or partially completes.

## Objective

Ensure that etcd certificate rotation procedures are rehearsed and recoverable so certificate maintenance does not create avoidable outages during planned or emergency renewal events.

## Rationale

Even when certificate rotation is configured, organizations often fail when the procedure has never been rehearsed or when rollback steps are undefined. This control isolates testing and recovery validation so teams can independently prove that rotation failures can be detected, contained, and remediated.

## Scope

Applies to all self-managed Kubernetes clusters using etcd. Covers rotation test execution, documented recovery steps, and evidence that failed or partial rotations can be handled safely.

## Enforcement

- Primary Enforcement Point: Change Validation and Recovery
- Implementation Surface: tested rotation runbooks, maintenance rehearsal records, rollback procedures, and incident or drill evidence for certificate renewal failures.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering
