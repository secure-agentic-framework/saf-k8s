# SAF-K8S-0402-011 - Cross-environment static credential prohibition

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.2 - Service Accounts and Workload Identity
- Maturity: Advanced
- Classification: ai-specific

## Statement

Federated training and multi-cluster inference workloads must not rely on shared static credentials or replicated secrets for cross-environment access.

## Objective

Eliminate long-lived static credentials from cross-environment AI workloads to reduce the blast radius of credential compromise and simplify revocation across cluster and cloud boundaries.

## Rationale

Static credentials replicated between clusters or cloud environments create synchronization challenges, expand the blast radius of compromise, and cannot be quickly revoked when a cluster is compromised. Enforcing prohibition on static credentials drives adoption of short-lived federated mechanisms such as SPIFFE SVIDs and cloud STS tokens.

## Scope

Applies to all AI workloads that communicate across Kubernetes cluster or cloud provider boundaries, including federated learning participants, multi-cluster inference routers, and cross-environment pipeline orchestrators.

## Enforcement

- Primary Enforcement Point: Admission policies scanning for credential secrets referencing remote cluster endpoints, and audit scans confirming absence of replicated credential Secrets across cluster namespaces.
- Implementation Surface: OPA/Gatekeeper or Kyverno policies blocking creation of Secrets with cross-cluster credential patterns, periodic scans for orphaned credentials, CI/CD pipeline checks preventing static credential injection into multi-cluster configurations.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering, ML Engineering
