# SAF-K8S-0202-010 - Node default seccomp profile enforcement

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.2 - Container Runtime Security
- Maturity: Foundational
- Classification: baseline

## Statement

Nodes must enforce approved default seccomp profiles for container workloads, with policy mechanisms ensuring workloads inherit those seccomp protections by default unless a documented exception exists.

## Objective

Reduce container escape risk by constraining workload system-call access through default seccomp enforcement.

## Rationale

Default seccomp profiles reduce the kernel attack surface available to workloads before a runtime vulnerability is exploited. Enforcing seccomp inheritance by default is independently auditable from broader mandatory access control activation because it governs system-call mediation for container processes.

## Scope

Applies to production cluster nodes and the workloads scheduled onto them. Seccomp defaults and AppArmor or SELinux activation must be validated for supported operating systems, node pools, and runtime configurations, with exceptions documented and reviewed.

## Enforcement

- Primary Enforcement Point: Node seccomp default configuration and workload policy enforcement for seccomp profile inheritance.
- Implementation Surface: Default seccomp profile configuration, kubelet or runtime defaults, and admission or policy controls that require approved seccomp settings on workloads.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; OS/Infrastructure
