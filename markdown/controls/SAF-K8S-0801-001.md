# SAF-K8S-0801-001 - GPU device plugin security configuration and hardening

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Practitioner
- Classification: ai-specific

## Statement

GPU device plugins (NVIDIA device plugin, GPU Operator, GPU Feature Discovery) must be deployed with hardened security configurations, restricted RBAC permissions, validated container images, and secure communication with the kubelet to prevent unauthorized accelerator exposure or scheduling manipulation.

## Objective

Ensure that the device plugin framework exposing GPU accelerators to the Kubernetes scheduler operates with least-privilege permissions and tamper-resistant configurations, preventing unauthorized GPU allocation or kubelet-level exploitation.

## Rationale

GPU device plugins run as privileged DaemonSets with direct access to host hardware and the kubelet gRPC interface. A compromised or misconfigured device plugin can advertise false GPU capacity, redirect workloads to attacker-controlled devices, or grant unauthorized containers access to accelerator hardware. Without hardening, the device plugin becomes a high-value target for cluster-level privilege escalation.

## Scope

Applies to all Kubernetes clusters running GPU device plugins, including NVIDIA device plugin, GPU Operator, and GPU Feature Discovery components. Covers both self-managed and managed Kubernetes services where GPU node pools are provisioned. Exceptions require documented risk acceptance for development-only clusters with no multi-tenant workloads.

## Enforcement

- Primary Enforcement Point: Device plugin DaemonSet manifests, GPU Operator Helm values, RBAC policies for device plugin service accounts, and admission control policies restricting device plugin deployment.
- Implementation Surface: NVIDIA device plugin DaemonSet manifests and Helm values, GPU Operator CRDs and ClusterRole/ClusterRoleBinding RBAC policies, GPU Feature Discovery (GFD) DaemonSet configuration, kubelet device plugin registration gRPC interface, device plugin container image digest pinning and admission-time signature verification, /var/lib/kubelet/device-plugins/ socket directory permissions, GitOps-managed device plugin configuration (sharing mode, resource naming, health checks), and Dynamic Resource Allocation (DRA) ResourceClaim/DeviceClass/ResourceClaimTemplate objects.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Platform Engineering; Security Engineering
