# SAF-K8S-0603-004 - Cryptographic agility and post-quantum readiness

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.3 - Attestation, Provenance, and Cryptographic Assurance
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must select cryptographic algorithms for artifact signing and hashing that account for the long-lived nature of ML model artifacts, maintain a documented migration plan for transitioning to post-quantum signature schemes, and ensure hashing and signing coverage spans the full ML lifecycle including training inputs, model weights, inference outputs, and intermediate artifacts such as checkpoints and embeddings.

## Objective

Prepare the supply chain cryptographic infrastructure for algorithm migration, particularly for long-lived ML model artifacts whose signing validity may extend beyond the expected security lifetime of current algorithms.

## Rationale

ML model artifacts often have multi-year lifespans in production, far exceeding typical container image retention periods. If signing algorithms become compromised or deprecated during that lifespan, all provenance guarantees are invalidated. Without cryptographic agility, re-signing the full artifact corpus requires rebuilding trust chains from scratch. Gaps in signing coverage (e.g., unsigned checkpoints or embeddings) create supply chain blind spots exploitable by adversaries.

## Scope

Applies to all cryptographic signing and hashing operations across the ML supply chain, including container image signing, model signing, attestation signing, and SBOM signing. Covers both cloud-hosted and on-premises signing infrastructure. Migration plans must address timeline, algorithm selection, and backward compatibility.

## Enforcement

- Primary Enforcement Point: Signing infrastructure configuration and cryptographic policy governance review.
- Implementation Surface: Cryptographic algorithm selection configuration for artifact signing (ECDSA P-256, RSA-2048, SHA-256 minimum hashing), NIST-approved post-quantum signature algorithm support (ML-DSA, SLH-DSA), ML-KEM for key-establishment and envelope-encryption, dual-signing infrastructure for long-lived ML artifacts (>5-year retention), signing coverage spanning training data manifests, model weights, checkpoint hashes, embedding indices, and inference outputs, and documented post-quantum migration plans with timeline and algorithm candidates.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; DevSecOps / Release Engineering
