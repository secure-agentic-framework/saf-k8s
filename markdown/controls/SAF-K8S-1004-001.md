# SAF-K8S-1004-001 - ML threat taxonomy per CTA-2114 mapped to Kubernetes

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.4 - AI and Supply Chain Threat Taxonomy
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must adopt the CTA-2114 ML threat taxonomy as the formal classification framework for AI-specific threats, including Data Poisoning, Model Poisoning, Targeted Poisoning, Label Modification, Model Disclosure, Data Disclosure, Evasion, Oracle Attacks (extraction, inversion, membership inference), and Denial of Service via computational complexity, and must map each threat class to Kubernetes infrastructure attack surfaces, control points, and detection mechanisms.

## Objective

Establish a standardized, comprehensive threat classification for AI workloads that enables consistent risk communication, detection rule development, and control mapping across the Kubernetes AI infrastructure.

## Rationale

Without a formal AI threat taxonomy mapped to Kubernetes infrastructure, organizations describe AI threats inconsistently, leading to duplicate controls for some threat classes and no coverage for others. CTA-2114 provides an industry-standard classification that, when mapped to Kubernetes attack surfaces (container runtime, storage, network, GPU scheduling), ensures every AI threat class has identified infrastructure-level control points and detection strategies.

## Scope

Applies to all organizations operating AI workloads on Kubernetes. The taxonomy mapping must cover all CTA-2114 threat classes and must be referenced in threat modeling, detection rule development, and incident classification. Must be reviewed when CTA-2114 is updated or when new AI workload types are introduced.

## Enforcement

- Primary Enforcement Point: Threat modeling documentation standards, detection rule taxonomy tagging, and incident classification schema.
- Implementation Surface: CTA-2114 ML threat taxonomy adoption documentation, threat class-to-Kubernetes attack surface mapping tables (Data Poisoning, Model Poisoning, Targeted Poisoning, Label Modification, Model Disclosure, Data Disclosure, Evasion, Oracle Attacks, DoS via computational complexity mapped to container runtime, storage, network, and GPU scheduling surfaces), threat modeling documentation standards, detection rule taxonomy tagging in SIEM, and incident classification schema aligned with CTA-2114 threat classes.
- Owner Primary: Security Architecture
- Owner Supporting: AI/ML Engineering; Detection Engineering; GRC/Compliance
