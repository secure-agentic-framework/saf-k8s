# SAF-K8S-0803-005 - DPU and SmartNIC firmware cryptographic verification and Secure Boot integrity

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.3 - High-Performance Interconnect Security
- Maturity: Advanced
- Classification: ai-specific

## Statement

DPU and SmartNIC devices used for AI workload acceleration or network offload must maintain firmware integrity through cryptographic verification and Secure Boot validation of the DPU firmware chain.

## Objective

Ensure programmable offload devices boot only approved firmware and that unauthorized firmware modification is detectable before DPU or SmartNIC services influence AI workload traffic or storage paths.

## Rationale

DPUs operate below the host OS trust boundary with direct access to traffic and storage I/O. Cryptographic firmware verification and Secure Boot validation are separate trust anchors that ensure the DPU firmware chain itself has not been replaced or tampered with before the device begins enforcing policies or processing workload traffic.

## Scope

Applies to Kubernetes nodes equipped with DPUs or SmartNICs with programmable firmware. Covers firmware signature verification, Secure Boot attestation for the DPU boot chain, and monitoring for unauthorized firmware version changes.

## Enforcement

- Primary Enforcement Point: DPU Firmware Chain Integrity
- Implementation Surface: DPU firmware management platforms, Secure Boot configuration for DPU firmware chain, firmware cryptographic signature verification, and firmware version attestation or monitoring.
- Owner Primary: Network Engineering
- Owner Supporting: GPU Infrastructure Engineering; Security Engineering; Platform Engineering
