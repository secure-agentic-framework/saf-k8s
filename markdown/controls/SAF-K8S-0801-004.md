# SAF-K8S-0801-004 - vGPU virtualization security controls

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Hypervisor-based vGPU deployments must enforce vendor-specific security controls including vGPU profile isolation, resource allocation limits, hypervisor escape prevention, and validated driver compatibility between host and guest vGPU components.

## Objective

Ensure that virtualized GPU resources maintain isolation guarantees enforced by the hypervisor layer, preventing guest-to-host escapes and cross-VM GPU resource interference in virtualized Kubernetes environments.

## Rationale

vGPU implementations rely on hypervisor-mediated GPU access where a vulnerability in the vGPU manager or driver can enable hypervisor escape from a GPU-enabled VM. Misconfigured vGPU profiles can allow a single VM to consume all GPU memory or compute, causing denial of service to other tenants. Driver version mismatches between host vGPU manager and guest drivers create instability and potential security gaps.

## Scope

Applies to Kubernetes clusters running on virtualized infrastructure (VMware vSphere, KVM/QEMU, Citrix) with NVIDIA vGPU, Intel GVT-g, or equivalent GPU virtualization. Bare-metal GPU deployments using MIG or passthrough are covered by other controls. Cloud provider virtual GPU instances must verify provider security controls meet organizational requirements.

## Enforcement

- Primary Enforcement Point: Hypervisor vGPU manager configuration, vGPU profile assignments, host-guest driver compatibility validation, and VM security policy settings for GPU passthrough restrictions.
- Implementation Surface: Hypervisor vGPU manager configuration (NVIDIA GRID/vGPU, Intel GVT-g), vGPU profile assignments and resource allocation limits, host-guest GPU driver version compatibility matrices, hypervisor security patch management (VMware vSphere, KVM/QEMU, Citrix), VM security policy settings for GPU passthrough restrictions, and hypervisor escape vulnerability scanning for GPU virtualization components.
- Owner Primary: GPU Infrastructure Engineering
- Owner Supporting: Virtualization Engineering; ML Platform Engineering; Security Engineering
