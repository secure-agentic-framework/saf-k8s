# SAF-K8S-0904-017 - Pipeline definition signing and execution-time signature verification

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.4 - AI Pipeline Orchestration and Experimentation Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Pipeline definitions must be cryptographically signed and their signatures verified before execution or admission so unauthorized or tampered ML workflow definitions cannot run.

## Objective

Ensure only authenticated and untampered pipeline definitions are accepted for execution.

## Rationale

Pipeline definitions govern the entire ML workflow. Without cryptographic signing and enforcement of signature verification before execution, an attacker can substitute or tamper with workflow manifests and run malicious stages while appearing to use a legitimate pipeline process.

## Scope

Applies to ML pipeline definitions, templates, and workflow manifests stored and executed on Kubernetes-based orchestration platforms such as Argo Workflows and Kubeflow Pipelines. Covers definition signing, signature verification workflows, and pre-execution or admission enforcement. Excludes immutable version storage and rollback governance, which are addressed separately.

## Enforcement

- Primary Enforcement Point: Pipeline definition repository signing workflow and execution-time verification point.
- Implementation Surface: Signed pipeline manifests, provenance or signature artifacts, signature verification before execution or admission, and enforcement records showing unsigned or invalid definitions are rejected.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; ML Engineering
