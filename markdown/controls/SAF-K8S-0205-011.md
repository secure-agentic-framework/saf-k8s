# SAF-K8S-0205-011 - Node firewall audit and change governance

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.5 - kube-proxy and Node Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Node-level firewall rules must be audited periodically and changes to those rules must follow controlled change management so unauthorized or drifted firewall modifications are detectable and reviewable.

## Objective

Ensure node firewall policy remains governed, reviewable, and resistant to unauthorized change over time.

## Rationale

Even if an initial rule set is restrictive, unmanaged changes or configuration drift can reopen node communication paths. Periodic audit and controlled change management make firewall governance independently auditable from the rule content itself.

## Scope

Applies to all production cluster nodes. Cloud environments may use security groups or VPC firewall rules as the node-level firewall layer. On-premises environments may use iptables, nftables, or firewalld.

## Enforcement

- Primary Enforcement Point: Firewall audit workflows, change records, and periodic verification of node firewall policy state.
- Implementation Surface: Firewall verification scripts, change tickets, security group audit logs, NSG review workflows, and periodic attestation of node firewall rule state.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Cloud Infrastructure; Security Engineering
